## Tech Debt Accumulated (Non-blocking)

### From Phase 20 code review (Warning + Info severity)

After Phase 20 shipped, `/gsd-code-review 20` surfaced 3 Warnings and 5 Info findings against the QUAL closure code itself. WR-01 and WR-02 were fixed inline via `/gsd-code-review 20 --fix` (commits `275def8` and `cdb2fe5`). The following remain:

- **WR-03** — `time.Sleep(100ms)` as readiness signal in `internal/pool/regression_rel_pool_02_test.go:134`. The sleep predates v1.10.3 (introduced under Plan 17-02 / D-17-04). Reviewer's proposed fixes (add `started` channel to `blockingPromptClient`, or poll `Pool.Stats().Busy`) require structural test-helper changes that exceeded Phase 20's mechanical-refactor scope. Deferred to a future phase that explicitly owns pool-test readiness signalling.
- **IN-01** — Phase 20 QUAL-02 dedup is half-done: `cmd/otto-tray/tray.go` `applyState` still open-codes the `OTTO Gateway · {state}` header format that `tooltipForState` now produces.
- **IN-02** — `escapeApplescript` table-driven test missing CRLF (`\r\n`), leading/trailing strip, and strip-only cases.
- **IN-03** — Blocking-prompt loop uses `for range []*blockingPromptClient{bc0, bc1}` which allocates a 2-element slice solely to iterate twice.
- **IN-04** — `RunUntilSignal`'s defensive `select`-then-`close(forceCloseCh)` is dead code with the current `sync.Once` single-writer pattern (asymmetric with the `shutdownCh` close which IS load-bearing).
- **IN-05** — New `cmd/otto-tray/tooltip.go` has no companion test file.

### From integration check (advisory)

- **golangci-lint stale cache hygiene** — A cached lint entry pointing at a deleted `/tmp/sv-20-reviewfix-*` worktree can surface a phantom `gosec G703` on a cold `make ci` run. `golangci-lint cache clean` resolves it. Candidate for a future hygiene phase: bake `golangci-lint cache clean` into the lint target's pre-step so cold runs after worktree GC are reproducible.
