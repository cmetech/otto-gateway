## Notable observations

1. **D-12-02 ("residual-fix-by-tighten") is a useful established pattern.** Phase 12 surfaced 2 residuals under `1.26.3` (GO-2026-5039 net/textproto reached via `server.Run`; GO-2026-5037 crypto/x509 reached via `tools/kiro-shim`) and chose to tighten the `go` directive one patch (1.26.3 → 1.26.4) rather than documenting unreachability. This is the correct call for reachable production paths and is now recorded as a `pattern-established` in the SUMMARY frontmatter for future toolchain bumps.

2. **Go auto-toolchain semantics worked transparently.** D-12-03 leaned on `GOTOOLCHAIN=auto` so the dev box (running `go1.26.3`) transparently fetched `1.26.4` from `go.mod`. CI followed the same `actions/setup-go@v5 with go-version-file: go.mod` path. No manual operator action required; no documentation drift in `docs/operating.md`. This is a quiet win that should be re-used for v1.8+ Go bumps.

3. **Scope discipline was exceptionally clean.** Source-tree diff for the entire milestone: `go.mod | 2 +-, 1 file changed, 1 insertion(+), 1 deletion(-)`. Three commits total (T2 bump, T3 tighten, T5 summary + close), plus orchestrator's docs-only `402d673` + `a37327e`. No `go.sum` delta. No opportunistic refactors. This sets a high bar for future debt-reduction milestones.

4. **Deviation 2 (PATH augmentation for `make ci`) is an environmental gotcha worth surfacing.** The dev box's interactive shell `PATH` does not include `$(go env GOPATH)/bin`, so `make lint` fails with `golangci-lint: No such file or directory` unless invoked as `PATH=$HOME/go/bin:$PATH make ci`. Not a code defect; not in scope for v1.7. Could be addressed in v1.8 by a `Makefile` `ci:` recipe that auto-injects `$(go env GOPATH)/bin` to `PATH` — but that's tooling-config territory and would have been out of scope here. Worth a v1.8 grooming consideration.

5. **Single-phase milestone shape is healthy.** v1.7 (1 phase, 1 plan, 1 source-line change, ~30min execution, ~5 commits) is the cleanest milestone shape this project has shipped. The narrow-scope discipline that produced it (re-deferring 08.3.1 + Nyquist + Authenticode to v1.8 at milestone-open) should be the template for future debt-reduction milestones.

6. **No CI workflow drift during the bump.** `.github/workflows/ci.yml` was not touched by Phase 12 (as planned). The `Set up Go (from go.mod)` step picked up the bumped `1.26.4` transparently via `actions/setup-go@v5`. The `GOVULNCHECK_VERSION: v1.1.4` pin was unchanged. Zero CI-config debt accumulated.
