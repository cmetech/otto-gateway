## Milestone-level decisions (durable)

- **D-11-01: govulncheck failures routed to v1.7.** Phase 10's gate restoration unmasked Go stdlib CVEs (GO-2026-5039, -5037, -4982, -4980, -4971, -4947, -4946, -4870, …) that pre-existed v1.6 but were hidden because the lint step always failed first. The narrow-scope v1.6 envelope does not include vulnerability cleanup. v1.7 starting move: bump Go toolchain pin to a patched 1.25.x or 1.26.x release and re-run `govulncheck`.
- **D-11-02: Add gofumpt to existing pre-commit framework, not a new `make pre-commit` target.** The codebase already has `.pre-commit-config.yaml` with golangci-lint + shellcheck + gitleaks. Lowest friction; contributor-portable; one place for hooks. `make pre-commit` was considered as a fallback for operators who don't want to install the framework but rejected as unnecessary surface area.
- **D-11-03: Extract pre-commit hook bash to a shell delegate.** The plan's inline-bash YAML hook entry tripped `mapping values are not allowed` because of `: ` inside the install hint string. Extracted to `scripts/pre-commit-gofumpt.sh` with no behavioral change.
