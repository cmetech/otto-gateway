## Notable observations

- **WARNING — v1.7 govulncheck routing not yet promoted to rollup docs.**
  The Phase 10 closure surfaced unmasked Go stdlib CVEs (GO-2026-5039 series)
  that fail `make ci`'s `Vulnerability scan` step on every main push. This
  is correctly documented inside Phase 10 + Phase 11 artifacts and acknowledged
  in REQUIREMENTS.md's FMT-02 row, but is NOT listed under `PROJECT.md`
  §"Deferred to v1.7" (line 75) or `REQUIREMENTS.md` §"Future Requirements"
  (line 32). A reader scanning the milestone-rollup docs alone would miss it.
  Recommend a one-line addition to both rollups during the v1.6 milestone
  close ceremony or as the first item in `/gsd-new-milestone v1.7`.

- **WARNING — cosmetic checkbox drift in REQUIREMENTS.md `### Active`.** The
  per-REQ `- [ ]` boxes (lines 15-26) were not flipped to `- [x]` at close
  even though the downstream Traceability table marks all six rows Complete.
  Not a gap (the traceability table is the contract), but visually
  inconsistent. Phase 11's executor-scope discipline explicitly forbade
  editing REQUIREMENTS.md — so this was a known orchestrator follow-up;
  flagging it here so it doesn't carry into v1.7's REQUIREMENTS.md template.

- **Observation — `make ci` does NOT exit 0 end-to-end on `main` today.**
  This is the explicit FMT-02 carve-out, but worth surfacing: a literal
  reading of FMT-02's text ("`make ci` … holds end-to-end on `main`") is
  NOT met. The traceability table and the §Active text both acknowledge
  this with the `(govulncheck routed to v1.7)` qualifier. A v1.7 success
  criterion should be "FMT-02 carved-out clause closed — `make ci` exits 0
  end-to-end" so the carve-out is itself audited closed rather than
  permanently parked.

- **Observation — `.golangci.yml` migration completeness was a tail
  surprise.** Phase 10 Wave 4's deviation chain (action @v6 → @v7,
  v2.1.6 → v2.12.2, `ignoreSigs` → `extra-ignore-sigs`) revealed that the
  original v2-schema migration in `f3a70fc` was incomplete in three subtle
  ways no one noticed because `continue-on-error: true` masked the action's
  refusal to install. The lesson — "test the gate, not just the local
  binary" — is captured in `10-04-SUMMARY.md` §Deviations and is a
  durable artifact for future milestones that touch CI config.

- **Observation — Phase 11 cleanly avoided touching `.go` files.** The
  `git show --stat` verification confirms Phase 11's commits modified ONLY
  `.pre-commit-config.yaml` (+6 lines), `scripts/pre-commit-gofumpt.sh`
  (+27 lines, new), and `docs/operating.md` (+71 lines), plus planning
  artifacts. This means Phase 11 had zero opportunity to silently
  re-introduce any of the 49 baseline lint violations Phase 10 drained — a
  cleanly engineered isolation between the two phases.

- **Observation — Phase 11 SUMMARY correctly disclaims that
  `pre-commit run gofumpt --all-files` was not executed live** because the
  pre-commit framework is not installed on the executor's dev box. The
  fallback (direct invocation of `scripts/pre-commit-gofumpt.sh`) exercises
  the same script the framework would call, so the wiring is verified
  end-to-end; a fresh contributor's first `pre-commit install` is the next
  expected live verification point and is documented in
  `docs/operating.md` line 536.
