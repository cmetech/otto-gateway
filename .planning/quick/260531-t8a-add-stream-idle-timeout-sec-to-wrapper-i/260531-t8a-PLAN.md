# Plan — 260531-t8a

## Goal

Close the install-flow gap from 260531-ruv: `STREAM_IDLE_TIMEOUT_SEC` was shipped end-to-end (golden template, CLI flag, gateway parse) but the `init` flow's set_env_line registry was missed. New installs work because the template is copied verbatim; existing installs running `init --force` would not get the line preserved consistently because no managed-keys record exists.

## Scope (single atomic commit)

### `scripts/otto-gw` (bash)

1. **Flag parser** (around line 705 in `init_cmd()`): add `--idle-timeout SEC` and `--idle-timeout=SEC` → `arg_idle_timeout`. Default empty string when unset.
2. **Existing-value extract** (around line 745): add `existing_idle_timeout="$(extract_env_any "$dest" STREAM_IDLE_TIMEOUT_SEC 2>/dev/null || true)"` inside the `reinit=1` block.
3. **Resolution** (after CHAT_TRACE resolution around line 880): compute final `idle_timeout_value` and `idle_timeout_commented` with precedence: CLI flag > existing-file value > default "30" commented.
4. **set_env_line call** (line 937, after CHAT_TRACE): add `set_env_line "$dest" STREAM_IDLE_TIMEOUT_SEC "$idle_timeout_value" "$idle_timeout_commented"`.
5. **for-key list** (line 974): add `STREAM_IDLE_TIMEOUT_SEC` to the `env` subcommand's printed-key list.
6. **Usage text**: add `--idle-timeout SEC` under the `init flags` block of the usage help (around line 1029).

### `scripts/otto-gw.ps1` (PowerShell)

1. **Param block** (line 35): `[int]$IdleTimeout = -1` — already exists ✓ (sentinel for unset).
2. **Existing-value extract** (around line 600 in `Initialize-Gateway`): set `$existingIdleTimeout` from `$existing['STREAM_IDLE_TIMEOUT_SEC']`.
3. **Resolution** (after CHAT_TRACE resolution around line 728): compute `$idleValue` and `$idleCommented` with same precedence as bash.
4. **Set-EnvLine call** (line 773, after CHAT_TRACE): add `Set-EnvLine -FilePath $destPath -Key 'STREAM_IDLE_TIMEOUT_SEC' -Value $idleValue -Commented $idleCommented`.

### `scripts/.env.otto-gw.example`

Already has `# STREAM_IDLE_TIMEOUT_SEC=30` at line 33. No change needed.

## Resolution semantics

- CLI flag passed → value written **uncommented**.
- CLI flag NOT passed + existing-file value found → write **uncommented** (preserves user's prior tuning).
- CLI flag NOT passed + no existing value → write `30` **commented** (default surfaced, binary fallback still applies).
