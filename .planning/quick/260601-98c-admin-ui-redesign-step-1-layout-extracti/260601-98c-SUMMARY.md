# Quick 260601-98c: Admin UI Redesign Step 1 — Layout Extraction Summary

**One-liner:** Extracted the admin page into a shared base layout + per-page templates, added /about and /docs placeholder routes, and wired a pre-paint theme bootstrap with `data-theme` on `<html>` — pure structural scaffolding with zero visual restyle (current dark palette renders identically).

## Outcome

Step 1 of the admin UI redesign landed as planned. The single `index.html.tmpl` was split into a shared `base.html.tmpl` (header bar with wordmark + theme toggle, tab nav strip with Dashboard/About/Docs, pre-paint theme bootstrap inline in `<head>`) plus three per-page templates (`dashboard.html.tmpl` preserves all four original sections verbatim with `data-*` selectors intact; `about.html.tmpl` and `docs.html.tmpl` ship as "Coming soon" placeholders). `pageHandler` was renamed to `dashboardHandler` and now sets `TabActive: "dashboard"`; new `aboutHandler` and `docsHandler` use the same WR-05 buffer-then-write pattern and are wired at `GET /admin/about` and `GET /admin/docs`. CSS additions are purely additive: a no-op `[data-theme="light"]` override block proves the theme-token plumbing works end-to-end, plus minimal styles for `.otto-header`, `.otto-tab-nav` (with `#FAD22D` active underline), `.otto-theme-toggle`, and `.otto-coming-soon`.

Theme wiring is fully operational: the `<head>` inline script reads `localStorage.getItem('otto-theme')` and sets `document.documentElement.dataset.theme` BEFORE the stylesheet link paints (no FOUC). The header toggle button flips dark↔light and persists to localStorage.
