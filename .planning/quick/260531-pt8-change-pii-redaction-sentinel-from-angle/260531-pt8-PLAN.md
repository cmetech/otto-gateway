<objective>
Change the PII redaction sentinel shape from angle-bracketed (`<EMAIL_1>`) to
square-bracketed (`[EMAIL_1]`) in `internal/plugin/pii/modes.go`, update the
existing assertion strings in the pii package's tests to match, and add a
single regression test that fails if angle brackets ever reappear in
`ApplyMode` output for `replace` or `hash` modes.

Purpose: kiro-cli / Claude interpret `<EMAIL_1>` as the opening of an XML
tag and block waiting for the matching close tag, causing
`engine.ACP.Prompt()` to hang until the 120s client timeout. Diagnosed in
quick `260531-oox` via DEBUG markers. Square brackets match the Python
Presidio convention, stay ASCII, and don't trigger any tag interpretation.

Output: Updated `modes.go`, `modes_test.go`, `pii.go` doc comment,
`pii_test.go` assertions, plus one new regression test.
</objective>
