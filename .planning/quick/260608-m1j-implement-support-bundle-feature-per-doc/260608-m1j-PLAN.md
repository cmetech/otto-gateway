    Create `scripts/lib/redact.sh` as a sourceable bash library implementing the spec's redaction rules byte-for-byte. Required surface:
    - `redact_stream` — reads stdin, writes stdout. Applies in this order with `sed -E` (BSD-sed compatible — POSIX has macOS and Linux operators): (a) `Bearer [A-Za-z0-9._\-]+` -> `Bearer [REDACTED]`; (b) lines matching `^(AUTH_TOKEN|PII_HASH_KEY|PII_ENCRYPT_KEY)=` -> `<KEY>=[REDACTED]`; (c) `Authorization:[[:space:]]*\S+` -> `Authorization: [REDACTED]`; (d) case-insensitive `x-api-key:[[:space:]]*\S+` -> `x-api-key: [REDACTED]` (compose as two sed passes to avoid GNU-only `I` flag — match `[Xx]-[Aa][Pp][Ii]-[Kk][Ee][Yy]` instead).
    - `mask_env_value VALUE` — echoes `${VALUE:0:4}…(${#VALUE} chars)` (mirrors existing `print_env` mask format in scripts/otto-gw line ~1690). Empty input echoes empty.
    - `is_secret_key KEY` — returns 0 if uppercase KEY matches `*TOKEN*`, `*KEY*`, `*SECRET*`, `*PASSWORD*`, `*PASSPHRASE*` OR is in the explicit list AUTH_TOKEN / PII_HASH_KEY / PII_ENCRYPT_KEY. Use bash `[[ ${k^^} == *TOKEN* ]]` style; bash 4+ already required by scripts/otto-gw.

    Create `scripts/lib/redact.ps1` mirroring exactly the same three surfaces (`Invoke-RedactStream` as a pipeline filter, `Mask-EnvValue`, `Test-IsSecretKey`).
