Create `scripts/publish.sh` (new file, executable, mode 755) implementing the design exactly as specified. The spec is the source of truth — every section is normative; do not invent behaviour not in the spec, do not omit specified behaviour.

**Hard rules:**
- First line: `#!/usr/bin/env bash`
- Then: `set -euo pipefail`
- bash 3.2+ compatible (stock macOS). Specifically: no associative arrays (no `declare -A`), no `mapfile` / `readarray`, no `${var,,}` lowercasing (use `tr '[:upper:]' '[:lower:]'`), no `[[ ... ]]` regex shortcuts where `case` would do — `[[ ... ]]` test syntax itself IS available in bash 3.2 and is fine. Note: existing `scripts/otto-gw` uses `[[ ... ]]` freely; match that style. The 3.2 restriction is on bash 4+ FEATURES (assoc arrays, mapfile, `,,`), not on `[[ ... ]]`.
- Target size ≤300 lines.
- `shellcheck scripts/publish.sh` must be clean. Any unavoidable warning must be silenced with an inline `# shellcheck disable=SCxxxx` carrying a one-line justification.
- Style match `scripts/otto-gw`: long block comments above functions explaining intent, lower-snake-case function names, `local` for all function-local vars, `set -e`-safe (no bare command in conditional position that would mask exit).

**Function inventory (implement all 11; see spec Architecture table):**
1. `usage()` — print `-h` block from spec (the five Usage examples plus the flag table rendered as text) and exit 0.
2. `log_message()` — tee to stdout + `LOG_FILE` if set; if log write fails, warn to stderr and continue (spec Error Handling: "Logging failure … don't abort an upload because the log is unwritable").
3. `resolve_version()` — `-v` flag value if set; else `git describe --tags --always --dirty 2>/dev/null`; else exit 1 with `Error: version required (-v) or run from a git checkout. Run with -h for usage.`
4. `discover_release_set()` — build the 5-entry expected file list from VERSION: `otto_gateway-darwin-arm64-${VERSION}.tar.gz`, `otto_gateway-darwin-amd64-${VERSION}.tar.gz`, `otto_gateway-linux-amd64-${VERSION}.tar.gz`, `otto_gateway-windows-amd64-${VERSION}.zip`, `SHA256SUMS-${VERSION}.txt`. Per-file existence check inside `SOURCE_DIR`. Missing → print full list with `[ok]` / `[MISSING]` per row, then exit 2 with hint `run 'make package-all' to rebuild`.
5. `validate_local_manifest()` — for each archive, compute SHA256 using `shasum -a 256` (preferred; present on macOS + most Linux) with `sha256sum` fallback for Linux-without-shasum (mirror Makefile `package-checksums` pattern). Parse `SHA256SUMS-${VERSION}.txt` and cross-check each archive row. Mismatch → print offending rows in `<filename>: expected <sha>, got <sha>` format, exit 1 with hint `re-run 'make package-all'`. **Cache the computed SHA per file in a parallel-array structure** (two arrays indexed identically: filenames + hashes — no assoc arrays per bash 3.2) so phases 3 and 4 reuse them without recomputing.
6. `check_prereqs()` — given the resolved destination set (minio / artifactory / both), verify all prereqs and **collect every missing piece into a single error report** (operator must NOT have to re-run 4 times). For minio: `command -v mc`. For artifactory: `ARTIFACTORY_API_KEY` non-empty AND `$ARTIFACTORY_CERT_PATH` (default `../secrets/certs/client.pem`) readable AND `$ARTIFACTORY_KEY_PATH` (default `../secrets/certs/client-key.pem`) readable AND `command -v jq` AND `command -v curl`. Cert paths are resolved relative to the repo root (the script's invocation cwd is assumed to be repo root; encode this assumption by computing repo root via `git rev-parse --show-toplevel` when available, fall back to script's parent's parent).
   - Default-`-d` (omitted): if at least one destination's prereqs satisfied → narrow silently with one warning line naming what was skipped. If neither → exit 1 with the full collected report.
   - Explicit-`-d`: any missing prereq is a hard error → exit 1 with the report. Do NOT silently downgrade `-d both`.
   - `ARTIFACTORY_API_KEY` resolution order: `-k <key>` CLI flag → `$ARTIFACTORY_API_KEY` env → shell-profile fallback (mirror oscar's loop over `~/.zshrc` / `~/.bashrc` / `~/.profile` grepping for `export ARTIFACTORY_API_KEY=`).
7. `acquire_lock()` / `release_lock()` — `dist/.publish.lock` containing the PID.
   - Lock absent → write current PID, proceed.
   - Lock present AND `kill -0 <pid>` succeeds → exit 1 with `Error: another publish appears to be running (PID <pid>).`
   - Lock present AND PID dead → print `[warn] stale publish lock at dist/.publish.lock (PID <pid> not running) — reclaiming`, overwrite, proceed.
   - Trap `SIGINT`, `SIGTERM`, `EXIT` to call `release_lock` on any termination path. On INT/TERM specifically, also flush log, print `interrupted — partial state may exist` to stderr, exit 130.
8. `upload_to_minio()` — per file: `mc cp --attr sha256sum=<hash> "$file" "$MINIO_ALIAS/$BUCKET_NAME/" --insecure` (defaults `MINIO_ALIAS=myminio`, `BUCKET_NAME=images`). Capture exit code, duration (epoch math), size (`stat -f%z` macOS / `stat -c%s` Linux fallback), MB/s. Log filename / size MB / duration / MB/s / mc exit. On per-file failure, continue to next file; increment a minio-failure counter for the final exit code. Use the cached SHA from phase 1.7, **never recompute**.
9. `upload_to_artifactory()` — per file: `curl -H "X-JFrog-Art-Api:$KEY" -H "X-Checksum-Sha256:$SHA256" --cert "$CERT" --key "$KEY_PATH" --max-time 60 -T "$file" "${ARTIFACTORY_BASE_URL}/${ARTIFACTORY_PATH}/<filename>"` — capture curl exit code AND HTTP status (use `-w '%{http_code}' -o <tmp>` pattern). Constants are hardcoded per spec: `ARTIFACTORY_BASE_URL="https://artifactory.rosetta.ericssondevops.com/artifactory/sd-mana-tmo-prepaid-cdp-pristine-generic"`, `ARTIFACTORY_PATH="Infra/images"`. On 401/403 → log `[FAIL] artifactory: <file> — HTTP 401 auth rejected` and abort remaining Artifactory uploads for this run (don't hammer with bad key). On curl exit 35/58/60 (TLS handshake failures) → log with cert/key paths in the message and abort remaining Artifactory uploads. Other failures → continue to next file. On success → immediately call `verify_artifactory` for that file.
10. `verify_artifactory()` — `curl -sf -H "X-JFrog-Art-Api:$KEY" --cert <CERT> --key <KEY> "${BASE_URL}/api/storage/${ARTIFACTORY_PATH}/<filename>"`, pipe through `jq -r '.checksums.sha256'`, compare to local hash. Mismatch → log `[FAIL] artifactory verify: <filename>`; increment verify-failure counter.
11. `main()` — orchestrate Phases 1→5 from the spec:
    - Phase 1: parse flags → resolve version → resolve source dir to absolute path → discover release set → check destinations + prereqs → validate local manifest.
    - Phase 2: print the plan summary block exactly in spec format (`otto-gateway publish plan` with `version:`, `source:`, `destinations:`, `archives:`, `log:` rows). If `-n` (dry-run), print the plan, exit 0, **do not** acquire the lock and **do not** create the log file (the plan's `log:` row shows the path that WOULD be used).
    - Phase 3: if MinIO selected, log header `=== MinIO upload ===` and iterate `upload_to_minio` per file.
    - Phase 4: if Artifactory selected, log header `=== Artifactory upload ===` and iterate `upload_to_artifactory` (with inline verify) per file.
    - Phase 5: print the final report block (`upload summary` with `minio:`, `artifactory:`, `duration:`, `log:` rows; show only the lines for destinations that ran). Compute final exit code: 3 if any upload failure; else 4 if any verify failure; else 0. Log file is preserved on all paths except dry-run.

**Flags (getopts):** `-v <version> -d <dest> -k <api-key> -s <dir> -n -h`. Use `getopts "v:d:k:s:nh"` (no `--long` forms — spec uses short flags throughout; keep it bash 3.2 idiomatic). Unknown flag → exit 1 with `Error: unknown flag. Run with -h for usage.`. Bad `-d` value (not minio|artifactory|both) → exit 1 with `Error: -d must be minio|artifactory|both. Run with -h for usage.`. `-s` not a directory → exit 1 with `Error: -s <dir> is not a directory. Run with -h for usage.`. `-v` empty string → exit 1 with `Error: -v requires a non-empty value. Run with -h for usage.`.

**Log file:** `logs/publish_${VERSION}_$(date -u +%Y%m%dT%H%M%SZ).log` relative to repo root. Create `logs/` if absent. **Skip log creation entirely in dry-run** (spec Phase 2).

**Exit codes (spec):** 0 success / 1 bad flags or prereqs or manifest disagreement / 2 release set incomplete / 3 upload failed / 4 verify-only failed / 130 interrupted.

**Out of scope (do NOT implement):** install-side automation, interactive menus, per-file selection, retry/backoff, git tagging, version bumping, GitHub release publishing, real-upload CI job.

After writing the file: `chmod 755 scripts/publish.sh`. Run `shellcheck scripts/publish.sh` and resolve any warning (silence with inline justification only if unavoidable).
