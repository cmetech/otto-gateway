<objective>
Deliver the dead-slot detection + lazy synchronous re-spawn vertical slice for the warm pool (POOL-04), backed by a new push-based `acp.Client.Done()` exit signal (D-01). This is the Slice A end-to-end path: a slot dies, a per-slot exit-watcher observes it, the next `Pool.NewSession` Acquire detects the dead slot and synchronously spawns a replacement before returning the slot to the caller — or returns a wrapped error and drops the slot from `p.all` if re-spawn fails. Also flips the `POOL_SIZE` env default from 1 to 4 (POOL-01 Node parity per D-10) and produces the per-slot `Detail()` rows that plan 05-03's `/health/agents` consumes (D-15).

Purpose: Close POOL-01..04 without disturbing Phase 2's Codex M-3 slot-release race resolution or Phase 4's D-06 disconnect watchdog. Both prior contracts must continue to pass with `-race` and `goleak`.

Output: A `*acp.Client.Done()` accessor; a per-slot exit-watcher goroutine + Slot.dead flag; a `respawnSlot` + `removeSlot` helper pair on `*Pool`; a `Pool.Detail() []AgentSlot` rendering; an env-default flip in `internal/config/config.go`; matching tests covering dead-slot lazy re-spawn (happy + failure paths), watcher goroutine cleanup on `Pool.Close`, and the per-slot detail row shape.
</objective>
