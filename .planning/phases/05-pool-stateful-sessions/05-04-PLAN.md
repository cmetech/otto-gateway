    <!-- Revision: addresses 05-REVIEWS.md MEDIUM-2 (shim argument contract) -->
    **Shim argument contract (binding):** the shim treats `argv[1]` as the absolute path to the real `kiro-cli` binary, and `argv[2:]` as the args to pass through verbatim. Example invocation: `KIRO_CMD=/tmp/kiro-shim KIRO_ARGS="$(which kiro-cli) acp"` resolves to `kiro-shim <kiro-cli-path> acp` at exec time. The shim's FIRST action on startup MUST be to log the resolved command line as the literal first line of every trace file in the form `# kiro-shim invocation: <argv[1]> <argv[2:]>` followed by a newline. This line is a header, not a JSON-RPC frame, and downstream merging tools must skip lines beginning with `#`.
