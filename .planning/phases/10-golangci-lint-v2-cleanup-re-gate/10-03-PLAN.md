## Per-category decision record (LINT-03 evidence)

### gosec G703 (1 site — `internal/admin/admin.go:195` — **policy: prove safe + scoped exemption OR fix**)

The flagged line is `http.ServeFileFS(w, req, staticFS, filePath)` where
`filePath = chi.URLParam(req, "*")`. The taint analysis flags that
`filePath` derives from an HTTP request and reaches a `ServeFileFS` call.

`staticFS` is the embedded `embed.FS` rooted at
`internal/admin/static/`. `embed.FS.Open` rejects path traversal (`..`)
by spec since Go 1.16 — `http.ServeFileFS` documents the same. The
fileserver is safe by construction.

**Policy: documented `//nolint:gosec` exemption.** Fix has zero net
benefit; the taint analyzer cannot follow into stdlib. Add the directive
on the call line with a citation to the `embed.FS` spec:

```go
http.ServeFileFS(w, req, staticFS, filePath) //nolint:gosec // G703: embed.FS rejects '..' per Go 1.16+ spec; staticFS rooted at internal/admin/static/
```

### gosec G705 (2 sites — `internal/admin/sse.go:91`, `internal/admin/sse.go:104` — **policy: escape**)

Both are `fmt.Fprintf(w, `{"error":"... %s ..."}`, source)` where
`source := r.URL.Query().Get("source")`. The response Content-Type is
`application/json` (set on the line above each), so the XSS surface
is technically nil — a browser receiving `application/json` will not
parse it as HTML. BUT: an operator viewing the response via
`curl | less` or copy-pasting into a chat client could see arbitrary
embedded HTML; defense-in-depth is cheap here.

**Policy: fix by validating + escaping.** The `source` value is already
allowlisted against `h.deps.LogPathOrder` at line 88 (line 91 only fires
when the source is NOT in the allowlist — i.e. attacker-supplied), so
the JSON envelope must escape it. Two paths:

(a) **Preferred — switch to `json.Marshal` / `json.NewEncoder`:**
```go
w.Header().Set("Content-Type", "application/json")
w.WriteHeader(http.StatusBadRequest)
_ = json.NewEncoder(w).Encode(map[string]string{
    "error": "unknown source: " + source,
})
```
`json.Encode` quotes-escapes the source automatically; no taint reaches
the structural JSON. Same shape for line 104 with the missing-path
message. This kills both G705 sites in one edit and is the idiomatic Go
HTTP-JSON pattern.

(b) Fallback if (a) introduces ordering issues with the
`fmt.Fprintf` newline/trailing-byte behavior: keep `fmt.Fprintf` but
quote `source` via `strconv.Quote` (which `json.Marshal` uses
internally) before interpolation.

Pick (a) unless `go test ./internal/admin/...` regresses. Document
choice in commit message.

### bodyclose (1 site — `internal/admin/sse_test.go:277` — **policy: fix**)

The test does `resp, err := http.DefaultClient.Do(httpReq)` and pushes
`resp` to a channel without ever closing `resp.Body`. The downstream
receiver (`resp = <-respCh`) returns from the test without calling
`resp.Body.Close()`. Fix: add `defer resp.Body.Close()` at the receiver
site AFTER the channel-receive succeeds. The connection lifecycle is
already bounded by the `defer srv.Close()` higher up, but the linter
(and `golangci-lint`'s `bodyclose` analyzer) requires the explicit close.

### nilerr (1 site — `internal/adapter/anthropic/handlers_test.go:111` — **policy: fix**)

```go
if f.collectErr != nil {
    // No canned response but an error: surface via Stream.Result().
    ch := make(chan canonical.Chunk)
    close(ch)
    return &fakeRunHandle{stream: &fakeStream{chunks: ch, err: f.collectErr}}, nil
}
```

The handler synthesizes a run-handle that carries `f.collectErr` on the
stream, then returns `(handle, nil)`. The test surfaces the error
through the `Stream.Result()` channel as a deliberate design — the test
helper mimics the production engine path that streams errors. This is
NOT a bug.

**Policy: scoped `//nolint:nilerr` with rationale**, NOT a fix. Fixing
would mean returning `(nil, f.collectErr)` which would short-circuit the
test's stream-aware code path — the OPPOSITE of what the test verifies.

Pattern:
```go
return &fakeRunHandle{stream: &fakeStream{chunks: ch, err: f.collectErr}}, nil //nolint:nilerr // collectErr surfaced via Stream.Result() per fake engine contract
```
