## Per-category decision record (LINT-03 evidence)

Each category below names: policy, the specific baseline lines covered, and
rationale. Tasks 1-5 each carry one category; the records below are the
authoritative LINT-03 source-of-truth for these five categories (copied into
SUMMARY.md at phase close).

- **staticcheck (QF1001 — De Morgan's law refactor, 3 sites)** — **Policy: fix.**
  De Morgan rewrites are local-only and improve readability with zero behavior
  risk. Baseline lines: `internal/adapter/anthropic/handlers_test.go:865`,
  `internal/engine/collect_test.go:189`, `internal/plugin/logging_test.go:310`.
  No `//nolint` exemptions added.
- **unused (4 sites)** — **Policy: delete.** Truly dead code (a test type
  `fakePostHook` whose only methods are also reported unused, and one helper
  `newRequestIDFromReader` in `internal/plugin/request_id.go` that isn't on
  any call path). Baseline lines:
  `internal/plugin/chain_test.go:{49,54,59}` (the type + its 2 methods all
  fall together), `internal/plugin/request_id.go:160`. Verify each isn't
  referenced via reflection or test build-tag guard before deleting. No
  `//nolint` exemptions added.
- **revive redefines-builtin-id (3 sites, this plan only)** — **Policy: rename.**
  Shadowing built-ins (`min`, `cap`) makes the code harder to grep and risks
  accidental misuse in Go 1.23+ where the built-ins are heavily used. Baseline
  lines: `internal/acp/client_test.go:944` (`min`), `internal/admin/tail.go:77`
  (`cap`), `internal/admin/tail_test.go:123` (`cap`). Rename to
  domain-meaningful names (`minVal`/`minDur` for `min`; `capacity`/`bufCap`
  for `cap`). Wave 3 covers the remaining revive subcategories (exported
  stutters, unexported-return, godoc form).
- **gosec G301 (2 sites)** — **Policy: tighten to 0o750.** The dir-perms
  literal `0o755` in `cmd/otto-gateway/main.go:989` (chat-trace dir) and
  `internal/config/config.go:493` (env-template dir) become `0o750`. Owner
  read+write+exec and group read+exec is sufficient for both — these are
  operator-side directories on the deploy host, not world-readable surfaces.
  No `//nolint` exemptions added.
- **noctx (4 sites)** — **Policy: thread `context.Background()`.** Three test
  sites in `internal/admin/sse_test.go` (`{216,510,608}`) switch
  `httptest.NewRequest` → `httptest.NewRequestWithContext(context.Background(), …)`;
  one production site `tools/kiro-shim/main.go:77` switches `exec.Command` →
  `exec.CommandContext(context.Background(), …)`. Production site uses
  `context.Background()` because `main` does not yet receive a ctx through
  the wrapper invocation chain and bolting one in is beyond the lint fix
  scope (CLAUDE.md "Don't refactor beyond what the task requires"). No
  `//nolint` exemptions added.
