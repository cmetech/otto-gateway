<objective>
Remove the temporary non-blocking exemption from the CI lint step now
that the baseline is at zero, and prove the gate works by introducing a
deliberate lint violation in a throwaway branch and confirming CI fails.

This is the LINT-02 close-out: the lint step in `.github/workflows/ci.yml`
becomes a hard merge gate again, and a future regression that re-introduces
any of the 49 baseline categories will be caught at PR review time
instead of silently accreting more debt.

This plan has ONE checkpoint:human-verify because the negative-test
branch creation + push + CI inspection is a sequence the executor can
automate but the operator must visually confirm in the GitHub Actions UI
(the orchestrator cannot infer CI-run outcomes from local state).

Purpose: shut the gap that's been open since `f3a70fc`.
Output: Single commit removing `continue-on-error: true` + TODO; a
documented negative-test proving the gate fires.
</objective>

  <name>Task 2: Operator runs the negative-test on a throwaway branch + confirms CI fails the lint job</name>
  <what-built>
    Task 1 removed `continue-on-error: true` so the lint step now blocks
    merges. We need to PROVE the gate fires by deliberately introducing
    a lint violation on a throwaway branch, pushing it, and observing
    the CI lint job fail.
  </what-built>
  <how-to-verify>
    The executor will perform steps 1-5 below automatically (push the
    throwaway branch) but steps 6-8 require operator eyes on the GitHub
    Actions UI. After step 5 the executor pauses for operator approval.

    1. From the freshly-committed `main` (or the post-Wave-4 branch about
       to merge), the executor creates a throwaway branch:
       `git checkout -b lint-gate-negative-test-10-04`
    2. The executor introduces ONE deliberate violation. Pick a low-risk
       site, e.g. add an unused identifier to `internal/plugin/chain_test.go`:
       ```go
       // negative-test artifact — DO NOT MERGE
       func unusedNegativeTestFunc() {}
       ```
    3. The executor runs `~/go/bin/golangci-lint run --timeout=5m`
       locally and confirms it FAILS with `unused: func unusedNegativeTestFunc is unused`.
       If it doesn't fail, the violation choice was wrong — pick a
       different category (try `redefines-builtin-id` by shadowing
       `min` or `cap` in a test).
    4. The executor commits with subject
       `test(10-04): negative-test for CI lint gate — DO NOT MERGE`
       and pushes the throwaway branch.
    5. The executor opens a draft PR from the throwaway branch to `main`
       (`gh pr create --draft --title "[NEGATIVE TEST] LINT-02 gate verification" --body "DO NOT MERGE — verifying CI lint job fails on deliberate violation per Phase 10 LINT-02 acceptance."`).
    6. **OPERATOR STEPS:**
       a. Visit the PR in GitHub Actions.
       b. Wait for the `lint + test-race + arch-lint + govulncheck` job
          to complete.
       c. Confirm the `golangci-lint` step shows a red ✗ and the job's
          overall status is FAILED.
       d. Note the PR URL + the failing CI run URL.
    7. After confirming, the operator instructs the executor to clean up:
       - `gh pr close <number> --delete-branch` (closes PR and deletes the
         throwaway branch remote-side)
       - `git checkout main && git branch -D lint-gate-negative-test-10-04`
       - Confirm no orphan branches remain: `git branch | grep negative-test` returns empty
    8. The PR URL + CI run URL get pasted into the Phase 10
       10-04-SUMMARY.md as LINT-02 evidence.
  </how-to-verify>
