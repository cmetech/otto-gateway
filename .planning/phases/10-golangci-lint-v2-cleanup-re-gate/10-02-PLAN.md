## Per-category decision record (LINT-03 evidence)

### wrapcheck (9 sites — **policy: wrap with fmt.Errorf using %w**)

`.golangci.yml` `wrapcheck.ignoreSigs` already exempts `.Errorf(`, `errors.New(`,
`errors.Unwrap(`, `errors.Join(`, `.Wrap(`, `.Wrapf(`, plus `_test.go` excludes
wrapcheck entirely. Every remaining hit is production code returning an error
from either (a) an interface method we define (`PreHook.Before`, `PostHook.After`,
`SessionRegistry.Get`, `context.Context.Err`) or (b) an external-package call
(`syscall.Kill`, `bufio.Reader.ReadString`, `json.Marshal`). Both classes get
the same fix: wrap with `fmt.Errorf("<context>: %w", err)` so the caller can
still `errors.Is`/`errors.As` the underlying error.

Per-site context strings (kept short, no PII / no path leakage):
- `internal/acp/pool_pgid_unix.go:47` (`syscall.Kill`) → `"acp.pool.pgid: kill pgroup: %w"`
- `internal/adapter/anthropic/handlers.go:404` (`SessionRegistry.Get`) → `"anthropic.handlers: session lookup: %w"`
- `internal/adapter/ollama/handlers.go:368` (`SessionRegistry.Get`) → `"ollama.handlers: session lookup: %w"`
- `internal/adapter/openai/handlers.go:432` (`SessionRegistry.Get`) → `"openai.handlers: session lookup: %w"`
- `internal/admin/sse.go:177` (`ctx.Err()`) → `"admin.sse: ctx done: %w"`
- `internal/admin/tail.go:423` (`bufio.Reader.ReadString`) → `"admin.tail: read line: %w"`
- `internal/engine/engine.go:332` (`PreHook.Before`) → `"engine: prehook %s: %w"` (with hook name from existing local var; if no name var exists at line 332, plain `"engine: prehook: %w"`)
- `internal/engine/engine.go:350` (`PostHook.After`) → `"engine: posthook %s: %w"` (likewise)
- `internal/plugin/pii/summary.go:131` (`json.Marshal`) → `"pii.summary: marshal: %w"`

No `//nolint:wrapcheck` exemptions added.

### unparam (13 sites — **policy: split by intent**)

Three subcategories, each handled differently:

1. **Test-helper signature contracts (8 sites) — `//nolint:unparam` with rationale.**
   These helpers parameterize a value for symmetry with sibling helpers
   (different tests pass different bodies via the same helper signature) OR
   their return value is conventionally part of a helper-pair that the
   linter can't see across files. Forcing-fixing them either (a) breaks
   symmetry with the sibling helper that DOES vary the param, or
   (b) propagates to callers that don't need the change.
   Sites:
   - `internal/adapter/anthropic/handlers_session_test.go:107` (`doPostWithSid`, `body` always same) — sibling helpers in openai/ollama tests vary `body`; symmetry argument.
   - `internal/adapter/anthropic/sse_posthook_test.go:33` (`runSSEEmitterAndPostHooks`, result 0 unused) — sibling `runSSEEmitterAndPostHooks` in openai variant returns same shape and IS used; cross-package symmetry.
   - `internal/adapter/ollama/chat_trace_e2e_test.go:217` (`runNDJSONEmitterDirect`, result 0 unused) — helper-pair with `runNDJSONEmitterAndPostHooks`; symmetry.
   - `internal/adapter/ollama/handlers_test.go:860` (`buildFormatIntegAdapter`, result 1 unused) — adapter-pair return; ditto.
   - `internal/adapter/ollama/ndjson_posthook_test.go:32` (`runNDJSONEmitterAndPostHooks`, result 0 unused) — see anthropic/sse_posthook_test note.
   - `internal/adapter/openai/handlers_session_test.go:121` (`doChatCompletions`, `body` always same) — see anthropic equivalent.
   - `internal/adapter/openai/sse_posthook_test.go:31` (`runSSEEmitterAndPostHooks`, result 0 unused) — see anthropic equivalent.
   - `internal/plugin/pii/pii_test.go:60` (`freshHook`, `mode` always "replace") — the helper exists so future PII tests can vary mode (mask, hash, drop) without re-defining; pre-emptive symmetry for a documented future test surface (PII modes are a documented config axis).
   Pattern: `func helper(...) (...) { //nolint:unparam // helper-pair symmetry with <name>; <name>'s caller varies this arg`
   (when the rationale is symmetry-with-sibling) or `// future PII mode coverage` for the `freshHook` case.

2. **Production hot-path callers (3 sites) — fix.**
   - `internal/pool/pool_test.go:866` (`waitForSlotDead`, timeout always 200ms) — wait, this IS a _test.go file. **Re-classify as #1 (test-helper).** Use the same `//nolint:unparam // matches sibling timeout-helper signature` pattern.
   - `internal/session/reaper_test.go:18` (`eventually`, timeout always 1s) — same. `//nolint:unparam // eventually helper kept polymorphic for future-test reuse`.
   - `internal/plugin/pii/contextual.go:24` (`hasContextWithin`, `window` always 50) — **production code, fix.** This is a private helper in `internal/plugin/pii/contextual.go`. Drop the `window` parameter and inline the constant `50` (or promote to a `const nerContextWindow = 50` at package level if there's another reader). The diff stays small (one signature change + one call site).
   - `internal/plugin/pii/pii.go:259` (`(*PIIRedactionHook).acceptNERSpans`, `s` unused) — **production code, fix.** Remove the unused `s` receiver-method parameter; update the single call site. If `s` is the receiver and unused, leave the receiver (Go conventions); if `s` is a value parameter, drop it.

Final tally: 10 `//nolint:unparam` exemptions (all in `_test.go` files for
helper-pair symmetry, each with one-line rationale) + 3 production-code fixes
(re-counting: 2 from group 2 plus 1 re-classified from group 1 was wrong —
re-reading the baseline: `pool_test.go` and `reaper_test.go` ARE
exemptions, `contextual.go` and `pii.go` ARE fixes. So 2 fixes + 11
exemptions = 13. Re-classify cleanly in execution:
**2 production fixes** (`contextual.go`, `pii.go`) + **11 `//nolint:unparam`
exemptions** in test helpers (the 8 cross-package helper-pair items + the 2
timeout helpers + `freshHook`)).
