- `canonical.StopReason` enum with all six D-02 variants — `StopUnknown` (sentinel zero), `StopEndTurn`, `StopMaxTokens`, `StopMaxTurnRequests`, `StopRefusal`, `StopCancelled`. - Whitebox `TestCanonicalTypes_PresenceAndZeroValues` locks all four shapes against accidental regression — explicitly asserts `StopUnknown == zero` and that the six constants are distinct.

## Decisions Made

- **Split file layout** (`stop_reason.go` + `model.go` + `capabilities.go`) over a grouped `types.go`. Chose split because Phase 1's existing `chunk.go` already establishes one-file-per-concern granularity for canonical types; downstream waves will land more types and a grouped file would grow into a junk drawer.
- **`StopUnknown` first in the `iota` block.** D-02 explicitly requires forward-compat: unknown wire strings → `StopUnknown`. Making it the zero value means the readLoop-EOF teardown path in `stream.go` (which sets no StopReason) naturally lands there too; no extra code in Wave 3's `Stream.close` to handle the "no stopReason emitted" case.
- **`Name` field placed between `URI` and `Title` on `ResourceLinkBlock`** so the Go field order mirrors the wire-spec field order in `acp_wire_shapes.md` §6 (`uri, name, mimeType, title`). Cosmetic but aids reading the diff against the spec.
