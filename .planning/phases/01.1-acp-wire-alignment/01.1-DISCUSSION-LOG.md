## Area 2: API surface for handshake state

### PromptCapabilities exposure

| Option | Description | Selected |
|--------|-------------|----------|
| Accessor on *Client | client.PromptCapabilities() returns captured value; Initialize signature unchanged | ✓ |
| Return from Initialize | Initialize(ctx) (PromptCapabilities, error) — breaks main.go | |
| Both | Belt and suspenders; minor API bloat | |

**User's choice:** Accessor on *Client.
**Notes:** Non-breaking; thread-safe with mutex; future pool can serve from any acquired slot.

### AvailableModels exposure

| Option | Description | Selected |
|--------|-------------|----------|
| Accessor on *Client, mirroring caps choice | NewSession signature unchanged; client.AvailableModels() returns snapshot copy | ✓ |
| Return from NewSession (SessionInfo) | NewSession returns SessionInfo{SessionID, AvailableModels} — breaks main.go and pool | |
| Both | Bloat but flexible | |

**User's choice:** Accessor on *Client, mirroring caps choice.
**Notes:** Mirrors caps choice for consistency. Phase 5 pool delegates to slot[0].AvailableModels().

### Concurrency primitive for captured state

| Option | Description | Selected |
|--------|-------------|----------|
| sync.RWMutex around fields | Common Go idiom; matches existing streamMu pattern | ✓ |
| atomic.Pointer[T] per field | Lock-free reads; Go 1.19+ idiom; less obvious to Go novice | |
| Single sync.Mutex covering all client state | Simplest; coarsest locking; conflates with streamMu | |

**User's choice:** sync.RWMutex around fields.
**Notes:** New stateMu separate from existing streamMu. Readable for Go-novice review.

## Area 4: Defensive parsing implementation style

| Option | Description | Selected |
|--------|-------------|----------|
| Tolerant struct + map[string]any tail | Outer struct with json.RawMessage variant slot; helpers for fallback chains | ✓ |
| Custom UnmarshalJSON on sessionUpdateParams | Strongest type safety; all variance logic next to the type | |
| Parse to map[string]any, pluck fields manually | Concise; loses compile-time guarantees; many type assertions | |

**User's choice:** Tolerant struct + map[string]any tail.
**Notes:** Type-safe at the boundaries; map[string]any (actually json.RawMessage) only where shape is genuinely variant. Readable for Go novice; debuggable with stdlib json.

### Follow-up: CamelCase discriminator alias handling

| Option | Description | Selected |
|--------|-------------|----------|
| Normalize once, switch on snake_case | One helper `normalizeUpdateType`; one case per variant in the switch | ✓ |
| Multi-case in the switch | Two labels per case; no helper; verbose | |
| EqualFold comparison | Handles arbitrary casing; cascading ifs; less Go-idiomatic | |

**User's choice:** Normalize once, switch on snake_case.
**Notes:** Adding a new variant is one line; CamelCase coverage automatic. Explicit unit test for the normalizer.
