<objective>
Make the ACP client absorb the entire `session/update` notification variance surface (CONTEXT.md D-16..D-19) and fix the `session/request_permission` deadlock (D-20). Pair the implementation with two proofs: a table-driven variance matrix that walks the cross-product of all observed shape variations (D-22) and an end-to-end fake-server test that exercises mixed variants through the dispatcher in one session (D-23). Vertical slice: notification-parsing fix + permission-response fix + the fake that exercises them ship together.

Purpose: Phase 1 used flat `{type, content}` parsing and matched only `session/update` + `_kiro.dev/session/update`. Real kiro-cli emits the spec-compliant `{params:{update:{sessionUpdate:"agent_message_chunk", content:{type:"text", text:"..."}}}}` shape — Phase 1 would parse but discard the content. Worse, `session/request_permission` was handled as a notification that triggered a NEW `session/grant_permission` request — kiro-cli waits forever for a response to the original frame id and deadlocks Phase 2 on the first tool-using prompt. This plan ships both fixes plus the variance test that prevents future regressions.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| kiro-cli stdout -> Go client (JSON parse) | Untrusted-shape JSON crosses here; kiro-cli is a trusted binary but its versioning produces variance that defensive parsing must absorb |
| kiro-cli stdout -> permission request -> client response | The original frame ID must be echoed correctly; mishandling causes the subprocess to deadlock |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01.1-01 | Tampering | translate.go sessionUpdateParams unmarshal | mitigate | Use `json.RawMessage` for the variant payload + nil-guard every dereference; on top-level unmarshal failure log `c.cfg.Logger.Warn("acp: malformed session update", "err", err)` and drop the frame (preserve Phase 1 behavior) |
| T-01.1-02 | Denial of service | runaway JSON in body.Args (map[string]any) | accept | kiro-cli is trusted; framer already enforces a 1MB scanner buffer (Phase 1 framer.go); a malicious or runaway payload caps at that size and triggers a scanner error which terminates the client gracefully |
| T-01.1-03 | Information disclosure | logged unknown discriminator values (D-19 default arm) | mitigate | Log at Debug level (CONTEXT.md §Claude's Discretion picks Debug — kiro-cli versions may add new types; Debug is the kinder default and avoids log spam in production) |
| T-01.1-04 | Tampering | session/request_permission response envelope | mitigate | Per D-20 echo the inbound `frame.ID` exactly via `*frame.ID` deref; the rpcResponse marshal MUST use the same id type kiro-cli sent (numeric). No `default` arm on writeCh select per the WR-02 pattern — block until the response lands or the client closes |
| T-01.1-05 | Repudiation / state corruption | parse panic on missing fields | mitigate | Every dereference of a pointer field (e.g. `frame.ID`, optional sub-structs) is preceded by a nil check; missing fields fall through to the discriminator default arm rather than crashing the read loop |

The fake server in `fakeacp_test.go` deliberately emits malformed and oddly-shaped frames as test fixtures — that is the exercise surface, not a runtime threat.
