## Accomplishments

- **Spec-compliant initialize request (D-08):** `params` now serialises as `{protocolVersion: 1, clientInfo: {...}, clientCapabilities: {fs: {readTextFile: false, writeTextFile: false}, terminal: false}}`. The gateway declares it cannot service `fs/*` or `terminal/*` reverse RPCs — matching the brief's intent that Phase 1.1 is not a full ACP client.
- **Defensive promptCapabilities capture (D-09):** `Initialize` parses `result.agentCapabilities.promptCapabilities` (image / audio / embeddedContext booleans) and writes the translated `canonical.PromptCapabilities` value into `Client.caps` under `stateMu.Lock()`. Missing or empty result → zero struct (all false); never errors on absence.
- **State plumbing (D-06):** Added `stateMu sync.RWMutex`, `caps canonical.PromptCapabilities`, and `models []canonical.ModelInfo` to `*Client`. `models` is declared zero-valued and is exclusively touched in Plan 03 (NewSession); declaring it here finalizes the struct layout so Wave 3 lands as field-population only.
- **Accessor methods (D-05):** `PromptCapabilities()` takes an RLock and returns the captured struct by value (zero before Initialize). `AvailableModels()` takes an RLock and returns `append([]canonical.ModelInfo(nil), c.models...)` — a defensive copy so the caller can mutate without affecting client state. Returns nil before NewSession.
- **Whitebox test `TestInitialize_CapturesPromptCapabilities`:** Three sub-tests cover the three contract states (before Initialize / after Initialize with full caps / after Initialize with empty result). Each sub-test runs `goleak.VerifyNone(t)` on tear-down and passes under `-race`.

## Decisions Made

- **Field grouping for `*Client`.** Placed `stateMu / caps / models` as a single triple-line group at the bottom of the struct, with a comment above describing the shared concern. This mirrors the existing single-line-group pattern already used for `activeStream + streamMu`. Did NOT introduce a sub-struct (e.g., `handshakeState`) — premature abstraction with only two fields.
- **Wire-type twin (`promptCapabilitiesWire`) for canonical translation.** Adding JSON tags directly to `canonical.PromptCapabilities` would violate Phase 1 D-11 (canonical stays JSON-tag-free). Instead, declared `promptCapabilitiesWire` inside `client.go` with JSON tags, then field-copy into the canonical type when assigning to `c.caps`. Cheap (three bools), clean boundary.
- **Test approach: inline responder goroutine, not the blackbox fake.** The existing `newFakeACPServer` helper lives in `acp_test` (blackbox, D-18) and isn't reachable from a whitebox `package acp` file. The `mockRWC` + `readLineFromPipe` + `serverWriteJSON` machinery already in `client_test.go` is a smaller, more composable surface and was a better fit for three sub-tests with different one-shot responses. No new helpers added.
- **Empty-result handling: gate on `len(frame.Result) > 0` before unmarshal.** Encoding/json with an empty `RawMessage` returns "unexpected end of JSON input". The simple guard skips the unmarshal entirely when no body is present, letting the zero-value `canonical.PromptCapabilities` flow through under `stateMu.Lock()`. Matches D-09 "missing/empty → zero struct, no error".
