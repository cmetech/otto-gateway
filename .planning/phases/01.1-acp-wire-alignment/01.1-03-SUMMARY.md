# Phase 01.1 Plan 03: Session/New + Session/Prompt Wire Alignment Summary

**Wave 3 of Phase 1.1 — replaced the Phase 1 `sessionNewParams`/`sessionNewResult`/`promptParams` shapes with spec-compliant equivalents, populated `c.models` under `stateMu` from `result.models.availableModels[]`, taught the `wireBlock` text variant to use the `text` field (not `content`), wired resource_link `name` (with `path.Base(URI)` fallback per D-04), and surfaced the agent's `stopReason` as `canonical.StopReason` on `Stream.Result()`. Existing `NewSession`, `Prompt`, and `Stream.Result()` signatures are unchanged.**

## Accomplishments

- **D-02 / D-07 stopReason round-trip:** `parseStopReason` maps `end_turn / max_tokens / max_turn_requests / refusal / cancelled` to the matching `canonical.StopXxx` constant; unknown/empty wire strings land on `StopUnknown` (forward-compat). `Prompt` parses `frame.Result` into a `promptResult{StopReason string}`, runs it through `parseStopReason`, and passes the parsed value into `s.close(&FinalResult{StopReason: stop}, nil)` so callers receive it on `Stream.Result().StopReason`. Parse failure logs `Warn` and falls through — the prompt does NOT fail on an unrecognised stop reason (per D-02).
- **D-10 mcpServers:[] on the wire:** `sessionNewParams` gains `MCPServers []mcpServer` (`json:"mcpServers"`); the call site builds the slice via `make([]mcpServer, 0)` so JSON marshal emits an explicit `[]` rather than `null`. `mcpServer` is an empty struct — Phase 1.1 has no MCP server fields, but the type exists to be the slice element so the encoder gets a real array.
- **D-11 sessionId / id fallback:** `sessionNewResult` now carries BOTH `SessionID` and `ID` fields. `NewSession` resolves the returned id via `firstNonEmpty(r.SessionID, r.ID)`. When both are empty it returns `acp: session/new result: missing sessionId and id` (the protocol contract is broken — failing loudly is correct).
- **D-12 availableModels population:** New nested types `sessionNewResultModels{AvailableModels, CurrentModelID}` and `sessionNewResultModelEntry{ModelID, Name}` mirror the wire shape one-to-one. Inside `NewSession`, after the response unmarshal, each entry is translated into `canonical.ModelInfo{ID: entry.ModelID, Name: entry.Name}` and the resulting slice is assigned to `c.models` under `c.stateMu.Lock()`. The Plan 02 `AvailableModels()` accessor (which already returns a defensive snapshot copy) now actually returns the populated data.
- **D-13 promptParams Prompt+Content defensive duplicate:** `promptParams.Blocks` is gone; in its place sit `Prompt []wireBlock` and `Content []wireBlock`, both pointing at the same `translateBlocks(blocks)` slice. JSON marshal serialises each as its own array so the wire frame carries the data twice — matching Node's `acp-ollama-server.js:296-303` defensive behavior so older + newer kiro-cli versions both find the prompt.
- **D-14 wireBlock new shape:** the text variant ships the `text` field (was `content` in Phase 1); resource_link carries the required `name`; image carrier fields (`mimeType`, `data`) exist now so Phase 2's Ollama→ACP image producer can populate them without further struct churn. Per D-15, NO `canonical.BlockKindImage` is added — that's Phase 2's call.
- **D-04 resource_link Name fallback:** when `canonical.ResourceLinkBlock.Name` is empty, `translateBlock` derives the wire `name` via `net/url.Parse(URI)` → `path.Base(u.Path)` (or `u.Opaque` if Path is empty). An unparseable URI leaves the wire `name` empty without panicking. The fallback guards against `path.Base("")` returning `.` (which would emit `"."` on the wire — wrong).
- **D-07 Stream.close merge semantics:** Phase 1's `close()` REPLACED `s.result` with the passed-in pointer. Wave 3 changes this to a merge: the original `s.result` allocated in `newStream()` (carrying `SessionID`) and updated by `push()` (incrementing `ChunkCount`) is preserved; only non-zero fields from a caller-supplied `result` (today just `StopReason`) get copied onto it. `Result()` signature is unchanged: `(*FinalResult, error)`.
- **Whitebox tests (six new) under `-race` + `goleak`:**
  - `TestParseStopReason_MappingTable` — table-driven, seven rows covering all five documented values plus empty string and arbitrary "banana".
  - `TestTranslateBlock_TextUsesTextField` — marshals the produced `wireBlock` and asserts the rendered JSON contains `"text":"hello"`.
  - `TestTranslateBlock_ResourceLinkNameFallback` — three rows: explicit name passes through; empty name with `file:///foo/bar.txt` derives `bar.txt`; unparseable URI yields empty name without panic.
  - `TestNewSession_PopulatesAvailableModels` — scripted `session/new` response with a two-entry availableModels array; asserts `AvailableModels()` returns `[{ID, Name}]` in order.
  - `TestNewSession_FallsBackToResultID` — response carries only `id` (no `sessionId`); `NewSession` returns that id; `AvailableModels()` stays nil.
  - `TestPrompt_SurfacesStopReason` — scripted three-step dialogue (initialize → session/new → session/prompt) with two sub-tests: `end_turn` maps to `StopEndTurn`; `banana` maps to `StopUnknown` without erroring the prompt.

## Decisions Made

- **Stream.close() chose Option 1 from PATTERNS.md** (extend `FinalResult`, merge non-zero fields onto the existing `s.result`). The plan-checker note in agent context flagged that Phase 1's `close()` REPLACED `s.result` if a non-nil was passed, and `push()` accumulates `ChunkCount` on the existing pointer — so a literal "set if non-nil" would silently zero out `ChunkCount`. Read `stream.go` to confirm before implementing. The merge approach preserves `SessionID` (set in `newStream`) and `ChunkCount` (updated by `push`) while letting `StopReason` flow in from `Prompt`.
- **Helpers landed in `translate.go`, not a new `parse.go`** — kept the file-scoped layer consistency with Phase 1 D-01. Phase 1's translation helpers (`translateUpdate`, `translateBlock`, `translateBlocks`) all live in `translate.go`; adding `firstNonEmpty` and `parseStopReason` next to them is the lower-churn move. If Plan 04 needs an `internal/acp/parse.go` for additional helpers, splitting then becomes cheap.
- **`promptParams` ships TWO `[]wireBlock` fields, both pointing at the same slice** — rather than a single field tagged for two JSON names (which Go reflect doesn't support cleanly) or a custom marshaler. Both Go fields hold the same `translateBlocks(blocks)` slice reference; `encoding/json` walks each field independently and emits each as its own JSON array. Cheap, idiomatic, and exactly what the Node reference does.
- **Old `wireBlock.Content` field is FULLY removed (not just retagged)** — Phase 1 had `Content string \`json:"content,omitempty"\`` for the text variant. Wave 3 deletes the field rather than retagging it to `json:"text"`. Rationale: any leftover Phase 1 caller still referencing `wb.Content` becomes a compile error, surfacing the migration loudly. The negative grep in the plan's `<verification>` block confirms zero remaining references.
- **mcpServer is an empty struct (`type mcpServer struct{}`), not a placeholder type alias** — Phase 1.1 has no MCP server fields, but `make([]mcpServer, 0)` needs a real element type for the slice literal to compile. An empty struct is the cheapest representation; Phase 8+ can grow it without breaking the wire frame.
