<objective>
Close the Node-shim parity gap on Ollama's `format` field. Three coordinated changes land in one atomic slice:

1. **Wire-decode** the dropped `format` JSON value into `canonical.ChatRequest.Format` on both `/api/chat` and `/api/generate` (one shared `decodeFormat` helper), honoring D-02's accepted/rejected shape matrix.
2. **Steering hook** in a new `internal/plugin/jsonformat/` package, mirroring the `internal/plugin/pii/` layout from Phase 8 — a `PreHook` that appends a verbatim GEN_RULES block (and an optional JSON-schema description) to `req.System` when `req.Format != nil`. Registered in `chain.Pre` between `AuthHook` and `PIIRedactionHook` (D-07), gated by `JSON_FORMAT_STEERING_ENABLED` (default `true`, D-06).
3. **Fence-strip** at the Ollama non-streaming render layer: export the existing conservative `engine.stripFences` (D-08 option a) and call it from `handleChat` + `handleGenerate` when `req.Format != nil`, before the wire renderer sees the text.

Purpose: LangFlow flows that set `format:"json"` or `format:<schema>` and currently work against the Node shim must produce equivalent behavior when pointed at otto-gateway. The fix surface is the `canonical.ChatRequest.Format` seam (already present, dormant since Phase 3) plus the existing `stripFences` helper (already present, conservative). The new code is the wire-decode, the hook, and the render-layer wiring.

Output: `go build ./...` and `make ci` pass. A `format:"json"` request through `/api/chat` to a fake engine returning fenced JSON delivers unfenced JSON at the wire; the system prompt the model sees (verified via `ChatTraceHook` capture in tests) contains the verbatim GEN_RULES text. A no-`format` request through the same path is byte-identical to current behavior — no GEN_RULES, no fence-strip.

Streaming + `format:"json"` is explicitly out of scope for this plan (see CONTEXT.md Out-of-Scope §). The deferred-items file in this phase directory documents that.
</objective>
