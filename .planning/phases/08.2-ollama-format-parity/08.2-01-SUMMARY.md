# Phase 08.2 Plan 01: Ollama Format-Parity Summary

Closes the Node-shim parity gap on Ollama's `format` field via three coordinated changes: wire-decode `format` into `canonical.ChatRequest.Format`, a new `JSONFormatSteeringHook` PreHook that appends verbatim GEN_RULES text, and fence-strip at the non-streaming render layer.

## Decisions Made

**D-02: decodeFormat shape matrix** — "json" string → `Format{Type:"json"}`, schema object → `Format{Type:"json_schema", Schema:...}`, null/empty/omitted → nil, other string or non-object → 400. Wire-decode returns an error so handlers can return HTTP 400 without awkward sentinel values.

**D-07: chain insertion order** — JSONFormatSteeringHook goes between AuthHook and PIIRedactionHook. Auth short-circuit wins first (a 401 from an invalid token never reaches the steering hook). Steering runs before PII so the GEN_RULES block (which contains no PII patterns) is never accidentally redacted.

**D-08 option (a): export StripFences** — Renamed private `stripFences` in `engine/coerce.go` to exported `StripFences`. The conservative whole-trimmed-body match is load-bearing; only the identifier changed.

**Streaming fence-strip deferred** — `format:json` + streaming responses are not fence-stripped. The NDJSON emitter delivers chunks as they arrive; a whole-body match would require buffering the full stream, which changes the streaming contract. Tracked for a future phase.

**JSON_FORMAT_STEERING_ENABLED defaults true** — The Node shim applies GEN_RULES unconditionally. The gateway mirrors this to prevent behavioral drift for existing LangFlow flows after the cutover.
