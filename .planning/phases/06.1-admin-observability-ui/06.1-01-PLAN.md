## Phase Goal

**As an** OTTO Gateway operator, **I want to** open `http://localhost:11434/admin` in a browser **so that** I can see a styled dark-mode page showing current gateway status, uptime, version, and pool counts that auto-refreshes every 30 seconds — without manually polling `/health` or `/health/agents` JSON.

## Slice Scope (Slice A: Snapshot endpoint + minimal page shell)

This plan ships the thinnest possible end-to-end vertical slice of the admin UI:
- Backend: a new `internal/admin` package with `Handler(Deps) http.Handler`, `AdminSnapshot` types, snapshot aggregator endpoint, `embed.FS`-served HTML/CSS/JS.
- Frontend: minimal `index.html.tmpl` that renders ONLY the summary-strip section (status pill, uptime, version, pool counts, sessions count, live indicator) — the pool grid, sessions table, and log tail sections appear in Plans 02/03.
- JS: 30s polling loop (D-06) that fetches `/admin/api/snapshot`, updates the summary strip DOM in place, ticks the "Xs ago" counter every 1s, and turns the live-dot red after 3 consecutive failures.
- Wiring: `internal/server.Config` grows `AdminHandler http.Handler`; `NewFromConfig` mounts it on the OUTER router (auth-exempt per D-01, alongside `/health`); `cmd/otto-gateway/main.go` builds `admin.Deps` from existing in-process adapters and passes the handler in.
- Arch-lint: `.go-arch-lint.yml` gains an `admin` component allowed to depend on stdlib + chi (vendor) + version; explicitly NOT allowed to import `internal/engine`, `internal/pool`, or `internal/session` (admin uses its own consumer-defined `PoolDetailSource` / `RegistryStatsSource` interfaces).
- Non-interference smoke: new `internal/server/server_admin_test.go` asserts that mounting `/admin` does not alter route status or handler bodies for `/`, `/health`, `/health/agents`, `/api/version`, or any existing surface route (D-15/D-16).

**End-state user check:** `make build && OTTO_LOG=/tmp/otto-gateway.log ./bin/otto-gateway`; open `http://localhost:11434/admin`; see a dark-mode summary strip with status pill, uptime, version, pool counts; counter ticks every 1s; the strip updates within 30s without page refresh.

**Out of scope (deferred to later plans):**
- Pool slots grid + Active Sessions table (Plan 02)
- Log Tail SSE panel (Plan 03)
- Cross-compile binary-size gate (Plan 04)

**Task sizing note:** Task 1 is intentionally wide (full package scaffold — 10 files including the admin package source files, test files, asset files, and the arch-lint edit) so Plans 02/03 are pure additive — known tradeoff of MVP vertical-slice mode. Tasks 2 and 3 are narrow and surgical (2 files and 1 file respectively).
