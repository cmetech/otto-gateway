### Binary Size Budget Note (Not a Blocker)

Plan 04's must_have states: "Binary size delta vs the pre-Phase-6.1 baseline is ≤30KB." The actual delta is 2.03MB (7,427,426 → 9,566,210 bytes). The Plan 04 `done` condition provides an explicit escape hatch: "(or, if >30KB, the cause is investigated + documented + the budget is amended with developer approval at the Task 2 checkpoint)."

The 2.03MB delta is caused exclusively by adding `html/template` + `text/template` + `text/template/parse` stdlib packages. `go.mod` diff is empty — zero new external dependencies. No web fonts, no icon libraries, no third-party packages (exactly what RESEARCH Pitfall 5 warns against). The embedded assets total 41,066 bytes; Pitfall 5's intent was to keep assets under 30KB, not the stdlib-induced binary delta.

**Classification:** WARNING (not BLOCKER). The investigation + documentation required by the done condition is present in 06.1-04-SUMMARY.md. Developer approval was not formally recorded because the Task 2 human-verify checkpoint was never executed by a human operator. The human_verification section below includes confirmation of this item.
