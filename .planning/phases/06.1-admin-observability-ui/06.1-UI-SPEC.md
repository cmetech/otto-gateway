# Phase 6.1 — UI Design Contract

> Visual and interaction contract for the OTTO Gateway admin observability page. Generated by gsd-ui-researcher from CONTEXT.md decisions D-01..D-16; verified by gsd-ui-checker.

> **Hard constraint reminder:** OTTO Gateway ships as a single static Go binary with `CGO_ENABLED=0`. The admin UI ships via `embed.FS` as plain `html/template` + vanilla CSS + vanilla JS. NO React/Vue/jQuery, NO node toolchain, NO bundler, NO npm packages, NO web fonts (no runtime network deps). NO icon library — Unicode glyphs or inline SVGs only, used sparingly.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (custom dark-mode admin shell via `embed.FS`) |
| Preset | not applicable — no shadcn, no design-system toolchain (single-binary constraint) |
| Component library | none — hand-rolled plain HTML/CSS components |
| Icon library | none — Unicode glyphs (●, ▶, ⏸, ✕) and 1-2 inline SVGs only |
| Font | System font stack (no web font load): `system-ui, -apple-system, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif` for UI; `ui-monospace, "SF Mono", Menlo, Consolas, "Liberation Mono", monospace` for log tail + session IDs |

**Style anchor (look & feel reference, NOT a code dependency):** `oscar-adminui` general dark-mode admin shell — lean, simple, stylish. Single scrollable page. NOT a full dashboard product with sidebars/nav.

## Spacing Scale

Declared values (multiples of 4 except `xxs`):

| Token | Value | Usage |
|-------|-------|-------|
| xxs | 2px | Hairline offsets, inner badge padding y-axis |
| xs | 4px | Icon-to-text gaps, status-dot offsets |
| sm | 8px | Compact spacing inside cards (label-to-value), badge padding-x |
| md | 12px | Card inner padding, table cell padding |
| base | 16px | Default element spacing, section inner padding |
| lg | 24px | Section gaps (between the 4 stacked sections) |
| xl | 32px | Page top/bottom padding |

**Exceptions:** `12px` is non-standard for an 8-point scale but is intentional for the dense observability layout (table rows and pool-slot cards become cramped at 8px and waste vertical space at 16px). All other values are multiples of 4.

## Typography

Single typographic ramp — 4 sizes, 2 weights only. No italics. No web fonts.

| Role | Size | Weight | Line Height | Where used |
|------|------|--------|-------------|------------|
| Caption | 12px | 400 (regular) | 1.4 | Muted labels in cards ("LABEL", "MODEL", "LAST USED"), badge text, timestamps |
| Body | 14px | 400 (regular) | 1.5 | Default UI body, table cells, session-ID values, status-pill text |
| Subhead | 16px | 600 (semibold) | 1.4 | Section headings ("Pool Slots", "Active Sessions", "Log Tail"), card primary value (slot label) |
| Display | 20px | 600 (semibold) | 1.3 | Summary-strip numeric counts (pool alive/busy, sessions active), status pill in header |

**Log tail uses monospace `ui-monospace` stack** at 12px / 400 / 1.5 line-height. Log lines are uppercase level prefix + space + timestamp + space + msg.

**Letter spacing:** none custom — system defaults.

### Accent reserved for (explicit, exhaustive list — `#FAD22D` MUST NOT appear elsewhere):

1. The single status "live" dot in the top-right of the summary header strip when ALL polls are succeeding AND pool is healthy (overall "everything is fine" indicator)
2. The focused-state outline on the level-filter `<select>` and the grep `<input>` (2px outline; replaces default browser focus ring)
3. The "Last updated Xs ago" green-dot is `#0FC373`; the yellow accent is ONLY for the overall-healthy live dot — not for "last updated" status

### Color combination rules

- Never use `#FAD22D` (yellow) as a background-filling color — only as a border, dot, outline, or single-glyph foreground
- Card borders use `#4A4A4A` (1px) by default; dead-slot cards use `#FF3232` (2px); never both at once
- Badge backgrounds are 12% opacity of the semantic color over `#3A3A3A` (e.g. healthy badge bg is `rgba(15, 195, 115, 0.12)`); badge text is full-opacity of the semantic color. Pre-compute the resulting hex if the executor prefers opaque values.
- Status pill in summary strip is filled (not outlined) using the semantic color at 100% opacity with `#28243D` text — high contrast for at-a-glance read

### Dark mode only

No theme switching in v1. The page is dark-mode-only. Do NOT add `prefers-color-scheme: light` overrides.

## Layout & Interaction Contract

This section is informational for planner/executor — not part of the standard 6-dimension checker rubric, but locked here so the executor doesn't re-derive it.

### Page structure (D-03)

Single scrollable page. Max content width 1280px, centered with 32px (`xl`) top/bottom padding and 24px (`lg`) left/right padding. Four stacked sections, separated by 24px (`lg`) vertical gaps:

1. **Summary header strip** — full-width card. Horizontal flex row at ≥1024px (status pill | uptime | version | pool counts | sessions active | live-update indicator on the right). Wraps to 2 rows at 768-1023px. Stacks vertically <768px.
2. **Pool Slots grid** — section heading + 3-column CSS grid (`grid-template-columns: repeat(3, 1fr)`) at ≥1024px; 2-column at 768-1023px; 1-column <768px. Card padding 16px (`base`).
3. **Active Sessions table** — section heading + table card. Horizontal scroll inside the card if columns don't fit (<768px). Empty state replaces the table body when zero sessions.
4. **Log Tail panel** — section heading + control row (level dropdown, grep input, pause/resume button on the right) + fixed-height (480px desktop, 360px tablet, 240px mobile) scrollable log viewport with `overflow-y: auto`. Auto-scroll pinned to bottom while running; sticky scroll position while paused.

### Polling behavior (D-05, D-06, D-07)

- `GET /admin/api/snapshot` polled client-side at **30s fixed interval** (no UI control).
- First fetch fires on page `DOMContentLoaded`.
- Success: update the DOM in place (no full re-render), update the "Last updated Xs ago" tag, ensure live-indicator dot is green/yellow.
- Failure (network error, non-2xx, parse error): leave existing DOM in place, swap "Last updated…" tag to `Poll failed at {HH:MM:SS} — retrying` in `#FF8C0A`, show warning badge, continue polling. After 3 consecutive failures the live-indicator dot turns red (`#FF3232`); resets to green on next success.
- "Xs ago" counter ticks every 1s via a separate lightweight setInterval; the actual snapshot fetch remains on the 30s cadence.

### Log tail SSE behavior (D-08..D-12)

- `GET /admin/logs/stream` opened as `EventSource` on page load.
- On connection: backfill arrives (up to 500 lines from shared ring buffer per D-09), then live lines stream in.
- Auto-scroll behavior: while running, append each new line and scroll viewport to bottom. While paused, append to a hidden buffer; show `{N} new` badge in `#FF8C0A` next to the Resume button. On Resume, flush buffered lines and scroll to bottom.
- Level filter: client-side filter against parsed `level` field on each line. Hidden lines are not removed from the DOM — they get `display: none` so toggling levels back doesn't re-fetch.
- Grep filter: client-side regex compiled on input debounce (150ms). Invalid regex shows the "Invalid regex…" hint in `#FF8C0A` and does not apply the filter (last valid filter stays active). Empty grep = no filter.
- On EventSource error: show `Log stream disconnected — reconnecting…` row in `#FF8C0A`, EventSource auto-reconnects (browser default 3s); on reconnect, the row is removed and any backfill returned from the server's ring buffer is de-duplicated (line equality on `timestamp+msg`).

### Visual hierarchy at a glance (per Specifics: "obvious at a glance when something is wrong")

- **Dead pool slot:** 2px solid `#FF3232` border on the card (overrides default 1px `#4A4A4A`), red `Dead — respawning…` text in the body slot.
- **Failed poll:** orange badge in the summary strip's live-indicator zone; persists until next successful poll.
- **3+ consecutive failed polls:** live-indicator dot switches from yellow to red.
- **Down overall status:** the status pill in the summary strip switches from `HEALTHY` (green fill) to `DEGRADED` (orange fill) or `DOWN` (red fill); pill color is the dominant visual anchor at the top of the page.

### Responsiveness (Claude's discretion per CONTEXT.md)

| Breakpoint | Layout |
|------------|--------|
| ≥1024px (desktop, primary target) | 3-col pool grid, full horizontal summary strip, 480px log viewport |
| 768-1023px (tablet, graceful) | 2-col pool grid, 2-row wrapped summary strip, 360px log viewport |
| <768px (mobile, minimum-viable) | 1-col stack everywhere, sessions table horizontal-scrolls inside its card, 240px log viewport |

No mobile-first authoring required — desktop is the primary target for operators. Mobile graceful-degradation only.

### Component visuals

- **Card:** background `#3A3A3A`, border `1px solid #4A4A4A`, border-radius `6px`, padding `16px`. No drop shadow (flat dark mode aesthetic).
- **Status pill:** padding `4px 12px`, border-radius `9999px` (full pill), background = semantic color at 100%, text color `#28243D`, font 14px / 600.
- **Badge** (in-card status indicator): padding `2px 8px`, border-radius `4px`, background = semantic color at 12% opacity over card surface, text color = semantic color at 100%, font 12px / 600 uppercase tracking-normal.
- **Status dot:** 8px circle, semantic color background. The "live" overall-health dot in the summary header strip pulses (CSS `@keyframes` opacity 1.0 ↔ 0.6 over 2s). All other dots are static.
- **Input + Select:** background `#28243D` (inset into card), border `1px solid #4A4A4A`, border-radius `4px`, padding `6px 12px`, text `#FAFAFA`, focus outline `2px solid #FAD22D` (yellow accent reserved use #2).
- **Button (pause/resume):** background `#1174E6` (brandSecondary) when active, `#3A3A3A` with `#4A4A4A` border when default; text `#FAFAFA`, padding `6px 16px`, border-radius `4px`.
- **Table:** background transparent (sits on card), header row `#4A4A4A` bottom border (1px), body row 1px `#4A4A4A` bottom border, cell padding `12px` (`md`). Last row has no bottom border. No zebra striping (lean aesthetic).
