# Phase 6.1 Plan 03: Log Tail Panel via SSE Summary

**One-liner:** Shared single-tailer goroutine (500-line ring buffer, 250ms poll, rotation-safe) fans log lines to N SSE subscribers with lazy-start/stop lifecycle, backfill on connect, and a client-side log panel with level filter, regex grep, pause/resume, and reconnect deduplication.

## What Was Built

### Task 1 (TDD RED → GREEN): RingBuffer + Tailer — `tail.go` + `tail_test.go`

**internal/admin/tail.go:**

- `RingBuffer` — capacity-N FIFO ring; `Push` O(1) head-pointer overflow; `Copy` returns FIFO-ordered slice (nil when empty). Protected by `sync.Mutex`.
- `subscriber` — `C chan string` + `ctx context.Context` + `closed bool` (guards against send-to-closed-channel race).
- `Tailer` — `path string`, `logger *slog.Logger`, `ring *RingBuffer`, `subscribers []*subscriber`, `running bool`, `cancelRun context.CancelFunc`. All fields protected by `mu sync.Mutex`.
- `NewTailer(path, logger)` — does NOT start goroutine (D-09 lazy-start invariant).
- `Subscribe(ctx)` → appends subscriber, lazy-starts `go t.run(runCtx)` on first call.
- `Unsubscribe(sub)` → sets `sub.closed = true` under lock, closes `sub.C`, cancels run goroutine when subscriber count reaches 0.
- `Snapshot()` → `t.ring.Copy()` under lock (used for SSE backfill).
- `run(ctx)` → single goroutine; `os.Open` at EOF (`Seek(0, SeekEnd)` — D-10); `time.NewTicker(250ms)` poll; `os.Stat` + `os.SameFile` rotation detection (T-6.1-15); `bufio.Scanner` with 1MB buffer limit (Pitfall 2); on rotation/truncation/error: close + reopen at new EOF (never backfills historical content).
- `broadcast(line)` → holds `t.mu` during ring push AND subscriber sends; non-blocking `select { case sub.C <- line: default: /* drop */ }` (T-6.1-11).

**Race fix (Rule 1 — auto-fixed):** Original design snapshotted the subscriber slice header outside the lock, then iterated outside the lock. `Unsubscribe` closing `sub.C` while `broadcast` was selecting on it caused a send-to-closed-channel panic under `-race`. Fix: hold `t.mu` during all broadcasts; `sub.closed` flag set under the same mutex before closing the channel.

**internal/admin/tail_test.go** — 11 tests, all with `defer goleak.VerifyNone(t)`:
- `TestAdmin_RingBuffer_PushCopyFIFO` — FIFO order preserved
- `TestAdmin_RingBuffer_OverflowDropsOldest` — oldest dropped on N+1 push
- `TestAdmin_RingBuffer_EmptyReturnsNilOrEmpty` — len=0 when empty
- `TestAdmin_RingBuffer_DoubleFull` — 2N pushes → last N survive

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] send-to-closed-channel race in Tailer.broadcast**
- **Found during:** Task 1, TestAdmin_TailerBackfillSnapshot / TestAdmin_TailerSlowSubscriberDrops under `-race`
- **Issue:** `broadcast` snapshotted the subscriber slice header outside `t.mu` then iterated the snapshot outside the lock. `Unsubscribe` calling `close(sub.C)` while `broadcast` was executing `select { case sub.C <- line: }` caused a write-write race on the channel's internal state under the Go race detector.
- **Fix:** Hold `t.mu` for the entire `broadcast` body (ring push + all non-blocking sends). Set `sub.closed = true` under the same mutex before closing the channel. `broadcast` checks `sub.closed` before sending. Acceptable because sends are non-blocking (select/default) and subscriber count is small.
- **Files modified:** internal/admin/tail.go
- **Commit:** 0d1633b

**2. [Rule 1 - Bug] noFlushRecorder embedding promoted http.Flusher unexpectedly**
- **Found during:** Task 2, TestAdmin_SSEHandler_FlusherCastFailure (test timed out)
- **Issue:** `type noFlushRecorder struct { *httptest.ResponseRecorder }` uses embedding, which promotes `ResponseRecorder.Flush()`. The `w.(http.Flusher)` cast in `sseHandler` succeeded, causing `sseLoop` to block indefinitely instead of returning 500.
- **Fix:** Replaced with `noFlushResponseWriter` using delegation (explicit `Header`, `Write`, `WriteHeader` methods) — no `Flush` promotion. `httptest.ResponseRecorder.Flush()` is hidden.
- **Files modified:** internal/admin/sse_test.go
- **Commit:** 392e9fd

**3. [Rule 1 - Bug] Data race in TestAdmin_SSEBackfillAndLive on ResponseRecorder**
- **Found during:** Task 2, TestAdmin_SSEBackfillAndLive under `-race`
- **Issue:** Test goroutine called `rec.Result()` (reading headers) while the handler goroutine was writing headers via `w.Header().Set(...)` + `w.WriteHeader()`. `httptest.ResponseRecorder` does not synchronize concurrent access.
- **Fix:** Replaced the shared `flushRecorder{httptest.NewRecorder()}` pattern with a real `httptest.NewServer(mux)`. The HTTP client and server run on separate goroutines with full net/http synchronization. Headers are read from the live `*http.Response` returned by `http.DefaultClient.Do`.
- **Files modified:** internal/admin/sse_test.go
- **Commit:** 392e9fd

**4. [Rule 1 - Bug] Test timing: tailer opened file before test appended "hello"**
- **Found during:** Task 1, TestAdmin_TailerLazyStartStop_MultipleSubscribers
- **Issue:** `appendToFile` was called immediately after `Subscribe` without waiting for the tailer goroutine to open the file and seek to EOF. The append happened before `os.Open` completed, so the tailer started reading from a non-zero position and the line was missed.
- **Fix:** Added `time.Sleep(400ms)` after Subscribe to allow 1–2 poll ticks before appending.
- **Files modified:** internal/admin/tail_test.go
- **Commit:** 0d1633b

**5. [Rule 1 - Bug] TestAdmin_TailerMissingFileGracefulRetry timing**
- **Found during:** Task 1, TestAdmin_TailerMissingFileGracefulRetry
- **Issue:** After creating the file and immediately appending "late", the tailer had not yet opened the new file (it was still in its retry loop). The file was appended before `lastSize` was set to 0, so the next size comparison did not trigger a read.
- **Fix:** Added `time.Sleep(700ms)` between `os.Create` and `appendToFile` to allow the tailer to detect the new file, open it at EOF (lastSize=0), then observe the growth.
- **Files modified:** internal/admin/tail_test.go
- **Commit:** 0d1633b
