### Task 1 Auto-Fix (Rule 1 Bug): Static File Serving

During the automated smoke test, `GET /admin/static/css/admin.css` returned 404.

**Root cause:** chi.Mount("/admin", handler) does NOT rewrite `r.URL.Path` in the sub-router. The handler receives the full path `/admin/static/css/admin.css`, not the stripped `/static/css/admin.css`. The Plan 01 implementation used `http.StripPrefix("/static/", ...)` which only matched when the handler was called directly in unit tests (without the chi.Mount wrapper) — unit tests passed; real server returned 404.

**Fix:** Replaced `http.StripPrefix("/static/", http.FileServer(...))` with a closure that extracts `chi.URLParam(req, "*")` (always just `css/admin.css` or `js/admin.js` regardless of mount context) and calls `http.ServeFileFS`. This is mount-path-agnostic.

**Verification:** After fix, `GET /admin/static/css/admin.css` → HTTP 200, Content-Type: `text/css; charset=utf-8`, 15,039 bytes. `GET /admin/static/js/admin.js` → HTTP 200, Content-Type: `text/javascript; charset=utf-8`, 20,925 bytes.

**Commit:** `e54d76c`

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Static file serving 404 — chi Mount does not strip prefix from URL.Path**
- **Found during:** Task 2 agent-side automated smoke (curl /admin/static/css/admin.css → 404)
- **Issue:** Plan 01 documented this as "StripPrefix path correction: chi strips /admin prefix before forwarding" — this was empirically WRONG. chi.Mount("/admin", h) does not rewrite r.URL.Path; the sub-router sees /admin/static/css/admin.css. Unit tests bypassed the chi.Mount layer by calling h.ServeHTTP directly, so they passed. Live server returned 404.
- **Fix:** Replace `http.StripPrefix("/static/", http.FileServer(http.FS(staticFS)))` with `chi.URLParam(req, "*")` + `http.ServeFileFS` — mount-path-agnostic because chi always sets the wildcard URLParam to just the captured segment (css/admin.css).
- **Files modified:** `internal/admin/admin.go`
- **Commit:** `e54d76c`
