
## Tests

### 1. Dark-mode palette correctness
expected: page background is dark purple-navy `#28243D`; cards/panels are darker gray `#3A3A3A`; foreground text off-white `#FAFAFA`; muted text `#A0A0A0`; live indicator pulse uses purple `#AF78D2`; section headings are 16px semibold. Live indicator (top-right) is a small filled dot that turns red `#FF3232` after 3 consecutive `/admin/api/snapshot` failures.
result: passed

### 2. Summary strip + pool grid + sessions table auto-refresh every 30s
expected: "Last updated Xs ago" counter (top of page) ticks every 1s; within 30s of issuing a request (e.g., `POST /v1/chat/completions`), summary strip's pool-busy + sessions-active counts update in place without a page reload; pool grid cards and sessions table rows refresh accordingly; scroll position is preserved across refreshes; no full re-render flicker.
result: passed
note: Active Sessions table is correctly empty when requests don't include `X-Session-Id` — sessions are an opt-in stateful contract per the Plan 05-03 X-Session-Id branch in all three adapters. Verified live with `curl -H 'X-Session-Id: ...'` that the table populates as expected.

### 3. Dead-slot visual contract — 2px red border within 30s
expected: identify a kiro-cli child PID (`pgrep -f kiro-cli`) → `kill <pid>` → within ~30s of the next snapshot poll, the pool grid card for that slot flips to a 2px solid red `#FF3232` border and shows "Dead — respawning…" status text; the alive/dead badge changes from green to red at 12% opacity. The card is visually unmissable per UI-SPEC.
result: passed

### 4. Log tail SSE streaming + all four UI controls
expected: scroll to the Log Tail panel. With `OTTO_LOG=/tmp/otto-gateway.log` set and the gateway running, fire a few requests and watch new lines stream in within ~1s.

  - **Level filter:** dropdown to ERROR — only ERROR-level lines appear; switch to ALL — everything appears.
  - **Regex grep:** type a valid filter like `chat`; lines not matching are hidden. Type an invalid regex like `[`; orange validation hint appears next to the input; filter is bypassed (all lines shown).
  - **Pause:** click Pause → new lines stop appearing; a "N new" badge shows the count of buffered lines.
  - **Resume:** click Resume → buffered lines flush; badge clears; new lines stream normally.
result: passed

### 5. Log rotation via `mv` + `touch` does not crash the tailer
expected: with the log tail panel open and streaming, in a separate shell:
```bash
mv /tmp/otto-gateway.log /tmp/otto-gateway.log.1 && touch /tmp/otto-gateway.log
```
Then fire another gateway request. Within ~1s a new log line should appear in the tail panel. The tailer must NOT crash, NOT emit an error toast, NOT require a page reload. (Backed by `os.SameFile` rotation detection in `internal/admin/tail.go`.)
result: passed
