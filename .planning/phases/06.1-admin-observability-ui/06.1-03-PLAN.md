**As an** OTTO Gateway operator, **I want to** see new log lines stream live into the admin page's Log Tail panel — with level filter, regex grep, and pause/resume controls — **so that** I can triage a request issue end-to-end on one page without opening a separate `tail -f /tmp/otto-gateway.log` terminal.

    RingBuffer type:
    - `type RingBuffer struct { mu sync.Mutex; cap int; head int; data []string; full bool }`
    - `func NewRingBuffer(cap int) *RingBuffer` (cap > 0; cap == 0 → panic or return cap=1; document choice).
    - `func (r *RingBuffer) Push(line string)` — O(1); on overflow advances head pointer and overwrites oldest.

    Tailer type:
    - `type Tailer struct { path string; logger *slog.Logger; mu sync.Mutex; ring *RingBuffer; subscribers []*subscriber; running bool; cancelRun context.CancelFunc }`
    - `type subscriber struct { C chan string; ctx context.Context }` — the ctx field tracks the caller's context for natural lifetime correlation.
    - `func NewTailer(path string, logger *slog.Logger) *Tailer` — does NOT start the goroutine; only initializes the ring buffer.
    - `func (t *Tailer) Subscribe(ctx context.Context) *subscriber` — under lock: appends to subscribers, lazy-starts the run goroutine if `!t.running` (assign `t.cancelRun`; set `t.running = true`; `go t.run(runCtx)`); returns the subscriber.
    - `func (t *Tailer) Unsubscribe(sub *subscriber)` — under lock: removes sub from slice (linear scan; admin has at most a handful of subscribers); `close(sub.C)` to signal upstream consumers; if `len(t.subscribers) == 0 && t.running`, call `t.cancelRun()` and `t.running = false`. The run goroutine observes ctx.Done() and exits.
    - `func (t *Tailer) Snapshot() []string` — returns `t.ring.Copy()` under lock.

    Tailer run loop (the only place file I/O happens):
    - `func (t *Tailer) run(ctx context.Context)` — single goroutine; per RESEARCH §Pattern 4 lines 637-718.
    - Local state: `f *os.File`, `lastSize int64`, `scanner *bufio.Scanner`, `scanBuffer = make([]byte, 0, 64*1024)`.
    - `reopen()` inner func: closes old f if non-nil; `os.Open(t.path)`; on err, log DEBUG once per tick + return (f stays nil; retry next tick); on success, `Seek(0, io.SeekEnd)` to position at EOF (D-10 invariant: NEVER backfill historical); store size; new `bufio.Scanner(f)` with `scanner.Buffer(scanBuffer, TailerMaxLineBytes)`.
    - Call `reopen()` once at start; `ticker := time.NewTicker(TailPollInterval); defer ticker.Stop(); defer if f != nil { _ = f.Close() }`.
    - Main select loop: `case <-ctx.Done(): return` / `case <-ticker.C: pollAndForward()`.
    - `pollAndForward()`:
      - If `f == nil`, call `reopen()` and continue.
      - `os.Stat(t.path)` → on error, log DEBUG + reopen + continue. `f.Stat()` → if err OR `st.Size() < lastSize` (truncation) OR `!os.SameFile(fst, st)` (rotation via rename+recreate), reopen + continue.
      - If `st.Size() == lastSize`, continue (no growth).
      - For `scanner.Scan()` → `t.broadcast(scanner.Text())`. On `scanner.Err()` (e.g. `bufio.ErrTooLong`), log DEBUG + reopen.
      - At end of poll, recreate `scanner = bufio.NewScanner(f); scanner.Buffer(scanBuffer, TailerMaxLineBytes); lastSize = st.Size()` so the next tick gets a fresh scanner that hasn't reached EOF.

    - Under lock: `t.ring.Push(line); subs := t.subscribers` (snapshot the slice header so we can iterate outside the lock — copying the pointer slice is cheap; the underlying subscriber pointers are stable).
    - Release lock.
    - For each `sub` in subs: `select { case sub.C <- line: default: /* drop */ }`. Non-blocking is REQUIRED — a slow subscriber must not stall the tailer or block other subscribers.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → /admin/logs/stream (EventSource) | Long-lived TCP connection per connected operator. Reads only; no operator data crosses to the gateway. |
| gateway process → OTTO_LOG file | `os.Open` read-only; no exclusive locks; tailer shares the file with `scripts/otto-gw`'s `>>` redirection. |
| shared tailer goroutine → SSE subscriber chans | Non-blocking fan-out — a misbehaving subscriber cannot stall the shared tailer or block other subscribers (drop-on-full discipline). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-6.1-10 | Information disclosure | Log lines may contain DEBUG-level request paths, headers, PII | accept | D-01 explicit acceptance: trust at network layer. Phase 8's PIIRedactionHook is the right home for runtime PII scrubbing; admin tail just relays what the logger already writes. Operators with sensitive log lines should set `LOG_LEVEL=info` or higher in v1. |
| T-6.1-11 | Denial of service | Slow SSE subscriber blocks tailer | mitigate | Non-blocking `select { case sub.C <- line: default: /* drop */ }` in `Tailer.broadcast`; subscriber chan buffer = 16; TestAdmin_SSESlowSubscriberDrops verifies. |
| T-6.1-12 | Denial of service | Goroutine leak on disconnect race | mitigate | Per-request handler bound to `r.Context()`; `defer h.tailer.Unsubscribe(sub)` mandatory; `goleak.VerifyTestMain(m)` enforces in CI; TestAdmin_SSECtxCancelTeardown + TestAdmin_TailerLazyStartStop verify. |
| T-6.1-13 | Spoofing | Operator-supplied log line containing fake SSE control bytes (`\n\n`) | mitigate | `writeSSELine` splits multi-line payloads into multiple `data:` prefixes within ONE event so `\n` inside a log line cannot close a frame; TestAdmin_WriteSSELine_MultilinePayload verifies. |
