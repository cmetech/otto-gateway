# Phase 6.1 Plan 01: Admin Observability UI — Snapshot Endpoint + Minimal Page Shell Summary

**One-liner:** Dark-mode admin page skeleton served via embed.FS at `/admin` with 30s polling of unified `AdminSnapshot` JSON endpoint, D-14 CSS palette, consumer-defined interface boundary (TRST-04 preserved).

## What Was Built

### Task 1: Scaffold internal/admin package

Created the complete `internal/admin` package (10 files):

- **internal/admin/admin.go** — `Handler(deps Deps) http.Handler` constructor returning a `chi.Mux` with three routes: `GET /`, `GET /api/snapshot`, `GET /static/*`. Declares `PoolDetailSource` and `RegistryStatsSource` consumer-defined interfaces (TRST-04 invariant: admin never imports `internal/pool`, `internal/session`, or `internal/engine`).

- **internal/admin/assets.go** — `//go:embed templates/*.html.tmpl static` directive (bare `static` name, not `static/*` per Pitfall 1); `staticFS = mustSub(assetsFS, "static")` for correct http.FileServer path resolution; `pageTemplate` parsed once at init via `template.ParseFS`.

- **internal/admin/snapshot.go** — `AdminSnapshot`, `SnapshotPool`, `SnapshotSlot`, `SnapshotSess` types with snake_case JSON tags; `snapshotHandler` with nil-safe pool/registry guards, `Cache-Control: no-store`; pure `computeStatus()` function.

- **internal/admin/static/css/admin.css** — Full D-14 palette (all 11 color tokens + 6 spacing tokens + 3 radius tokens + 2 font tokens as CSS custom properties); summary strip, status pill, live dot, badge, card, table, log viewport styles; responsive breakpoints at 1024px and 768px.

- **internal/admin/static/js/admin.js** — IIFE; `fetchSnapshot()` polls `/admin/api/snapshot` every 30s; `renderSummary(snap)` updates all `data-*` DOM hooks; `tick()` 1s "Xs ago" counter; 3-fail-then-red live-dot logic; `OTTO_ADMIN_CONFIG.pollMs` drives interval.

- **internal/admin/handlers_test.go** — `TestAdmin_PageHandler`, `TestAdmin_StaticServes`, `TestAdmin_StaticServes_JS`, `TestAdmin_AssetsFSContains` (embed glob regression).

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] StripPrefix path correction for static file serving**
- **Found during:** Task 1 (TestAdmin_StaticServes returning 404)
- **Issue:** Plan action specified `http.StripPrefix("/admin/static/", ...)` but chi's `Mount("/admin", ...)` strips the `/admin` mount prefix BEFORE forwarding to the admin sub-router. So the sub-router sees `/static/css/admin.css`, not `/admin/static/css/admin.css`. Using the full path as the strip prefix caused the FileServer to never match any files.
- **Fix:** Changed to `http.StripPrefix("/static/", http.FileServer(http.FS(staticFS)))` — strips only the `/static/` portion that chi leaves in the path when forwarding from the `/static/*` route.
- **Files modified:** internal/admin/admin.go
- **Commit:** 8c49267

**2. [Rule 1 - Bug] Chi import missing from server_admin_test.go**
- **Found during:** Task 2 (compile error in stubRouteRegistrar usage)
- **Issue:** Test used `chi.Router` interface for the stub surface registrar but `chi` was not imported.
- **Fix:** Added `"github.com/go-chi/chi/v5"` to the import block.
- **Files modified:** internal/server/server_admin_test.go
- **Commit:** cc9cc56
