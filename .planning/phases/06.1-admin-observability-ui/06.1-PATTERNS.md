### `internal/admin/snapshot.go` (controller, request-response)

**Analog:** `internal/server/agents.go` (and secondarily `internal/server/health.go`)

**Imports pattern** (`agents.go` lines 1-7 + `health.go` lines 1-7):

**Typed-struct + JSON-tag wire-contract pattern** (`agents.go` lines 14-57 — typed structs with snake_case `json:"..."` tags; pointer fields for nullable):

```go
// AgentsResponse is the body returned by GET /health/agents (D-14).
// Shape is locked by D-14/D-15/D-16; consumer dashboards depend on the
// snake_case wire contract verbatim.
type AgentsResponse struct {
    Pool     AgentsPool     `json:"pool"`
    Sessions []AgentSession `json:"sessions"`
}

type AgentSlot struct {
    Label            string  `json:"label"`
    Alive            bool    `json:"alive"`
    Busy             bool    `json:"busy"`
    CurrentSessionID *string `json:"current_session_id"`  // pointer → null when idle
}

type AgentSession struct {
    ID       string    `json:"id"`
    Alive    bool      `json:"alive"`
    Busy     bool      `json:"busy"`
    LastUsed time.Time `json:"last_used"`
    Model    *string   `json:"model"`                    // pointer → null when unset
}
```

**Apply to `AdminSnapshot`:** same discipline. `*string` for `current_session_id` and `model`, snake_case JSON tags, slot/session field sets match the existing `server.AgentSlot` / `server.AgentSession` field-for-field so the JSON keys are isomorphic across `/health/agents` and `/admin/api/snapshot`.

**Nil-safe accessor + encode pattern** (`agents.go` lines 83-107):

```go
// agentsHandler handles GET /health/agents (D-14).
//
// Nil-safety:
//   - When s.poolDetail is nil, Pool defaults to {0,0,0,nil}.
//   - When s.registry is nil, Sessions stays nil (encodes as null per
//     stdlib JSON; consumers tolerate null in addition to []).
func (s *Server) agentsHandler(w http.ResponseWriter, r *http.Request) {
    resp := AgentsResponse{}
    if s.poolDetail != nil {
        slots := s.poolDetail.Detail()
        resp.Pool.Slots = slots
        resp.Pool.Size = len(slots)
        for _, sl := range slots {
            if sl.Alive {
                resp.Pool.Alive++
            }
            if sl.Busy {
                resp.Pool.Busy++
            }
        }
    }
    if s.registry != nil {
        resp.Sessions = s.registry.Detail()
    }
    w.Header().Set("Content-Type", "application/json")
    w.WriteHeader(http.StatusOK)
    if err := json.NewEncoder(w).Encode(resp); err != nil {
        // Cannot write error response after WriteHeader — just log it.
        LoggerFromCtx(r.Context(), s.logger).Error("agents encode", "err", err)
    }
}
```

**Apply to `snapshotHandler`:** mirror exactly. Guard each upstream source with a nil check before invoking `Stats()` / `Detail()`. Use `json.NewEncoder(w).Encode(resp)`. Log encode failures via `LoggerFromCtx` with a fallback to `h.logger` (the admin Handler's stored logger). Add one extra header that `agentsHandler` does NOT set: `Cache-Control: no-store` (per RESEARCH Anti-Patterns — pollers must not cache).

**Health-handler reference** (`health.go` lines 61-84) — same shape, fewer fields; reproduced here for the simplest-possible aggregator template:

```go
func (s *Server) healthHandler(w http.ResponseWriter, r *http.Request) {
    var ps PoolStats

```

**Status derivation:** RESEARCH §Pattern 2 defines `computeStatus(snap)` — `"ok"`/`"degraded"`/`"down"` based on Pool.Size/Alive ratios. Place it alongside `snapshotHandler` in `snapshot.go`.

### `internal/admin/sse.go` (controller, streaming)

**Analog:** `internal/adapter/anthropic/sse.go`

**Keepalive constant pattern** (`anthropic/sse.go` lines 19-24 — named const, NOT a magic literal, so tests can grep it):

```go
// PingInterval is the cadence at which the SSE emitter writes
// `event: ping` keepalive frames during idle stretches of the stream
// (W7 — named const so cadence is auditable via grep; tests reference
// this identifier directly so a literal change here flips the
// PingInterval_Constant test).
const PingInterval = 15 * time.Second
```

**Apply to `internal/admin/sse.go`:** declare `const SSEKeepaliveInterval = 15 * time.Second` (matches the Anthropic cadence per RESEARCH §Pattern 3). Tests grep this identifier.

**Flusher cast + streaming headers** (`anthropic/sse.go` lines 486-498):

```go
func runSSEEmitter(ctx context.Context, w http.ResponseWriter, run RunHandle, model string, logger *slog.Logger) error {
    flusher, ok := w.(http.Flusher)
    if !ok {
        // Caller (handlers.go) is responsible for translating this into
        // a JSON 500 envelope. We have NOT written any bytes yet so the
        // caller is free to call writeError.
        return errors.New("anthropic: response writer is not flusher")
    }

    w.Header().Set("Content-Type", "text/event-stream")
    w.Header().Set("Cache-Control", "no-cache")
    w.Header().Set("Connection", "keep-alive")
    w.WriteHeader(http.StatusOK)
    // ... emit initial frame ...
    ticker := time.NewTicker(PingInterval)
    defer ticker.Stop()
    return runSSEEmitterLoop(ctx, e, run, ticker.C)
}
```

**Apply to admin `sseHandler`:** same cast-and-check at entry; same three SSE headers; ADD `X-Accel-Buffering: no` (per RESEARCH Pitfall 3 — admin tail latency-sensitive vs. nginx default buffering). The admin handler does NOT call back to a caller for the 500 — it owns the response, so on flusher-cast failure call `http.Error(w, "streaming unsupported", http.StatusInternalServerError)` and return.

**Single-goroutine select loop** (`anthropic/sse.go` lines 556-578):

```go
// D-05: this is the SINGLE goroutine that owns the SSE writer. The
// select on (ctx.Done, tickerC, chunks) ensures no concurrent write
// across the three event sources.
func runSSEEmitterLoop(ctx context.Context, e *sseEmitter, run RunHandle, tickerC <-chan time.Time) error {
    chunks := run.Stream().Chunks()
    for {
        select {
        case <-ctx.Done():
            e.logger.Debug("anthropic: sse client disconnect", "session_id", run.SessionID())
            return fmt.Errorf("anthropic: sse ctx: %w", ctx.Err())

        case <-tickerC:
            if err := e.writeEvent("ping", pingEvent{Type: "ping"}); err != nil {
                return err
            }

        case c, ok := <-chunks:
            if !ok {
                return finalizeStream(e, run)
            }
            if err := e.applyChunk(c); err != nil {
                return err
            }
        }
    }
}
```

**Apply to admin sseHandler:** verbatim shape — `for { select { case <-r.Context().Done(): return / case <-ticker.C: ping / case line, ok := <-sub.C: forward } }`. The `sub.C` channel comes from `tailer.Subscribe(r.Context())`. Defer `tailer.Unsubscribe(sub)` BEFORE the select loop so the shared tailer's lazy-stop fires on disconnect (per D-09/D-11). Single-goroutine invariant: only this handler goroutine writes to `w`; the tailer never touches `w`.

**writeEvent framing pattern** (`anthropic/sse.go` lines 225-235 — single source of truth for SSE framing; ONLY method that touches `e.w`/`e.flusher`):

```go
// writeEvent emits one named SSE frame using the canonical
// `event: <name>\ndata: <json>\n\n` framing + Flush. This is the ONLY
// method that touches e.w / e.flusher (D-05 single-goroutine
// invariant; T-3.1-FRAMEERR single-source-of-truth for framing).
func (e *sseEmitter) writeEvent(eventName string, payload any) error {
    body, err := json.Marshal(payload)
    if err != nil {
        return fmt.Errorf("anthropic: marshal %s: %w", eventName, err)
    }
    if _, err := fmt.Fprintf(e.w, "event: %s\ndata: %s\n\n", eventName, body); err != nil {
        return fmt.Errorf("anthropic: write %s: %w", eventName, err)
    }
    e.flusher.Flush()
    return nil
}
```

**Apply to admin `writeSSELine`:** RESEARCH §Pattern 3 defines a slightly different framer because log lines are raw text (not JSON). Key difference: split multi-line payloads into multiple `data: ` prefixes within one event per SSE spec. Use the same `fmt.Fprintf(w, "event: %s\n", name)` + `data: %s\n` pattern; flush ONCE per frame.

### `internal/admin/tail.go` (service, event-driven)

**Analog:** No existing file-tailer in repo. RESEARCH §Pattern 4 supplies the full skeleton (lines 535-737 of RESEARCH.md) — that IS the contract for the executor. The pattern is:

- `Tailer` struct with `mu sync.Mutex`, `subscribers []*subscriber`, `ring *RingBuffer`, `running bool`, `cancelRun context.CancelFunc`
- `NewTailer(path, logger) *Tailer` — does NOT start the goroutine
- `Subscribe(ctx) *subscriber` — lazy-starts the run goroutine on first subscribe
- `Unsubscribe(sub)` — removes sub from slice; cancels run goroutine if last subscriber
- `Snapshot() []string` — returns ring buffer copy for backfill
- `run(ctx)` — single goroutine; opens at EOF; polls every 250ms; re-opens on rotation/missing; broadcasts via non-blocking send

**Closest in-repo discipline reference** is the Anthropic SSE single-goroutine + channel-fanout convention. The subscriber-slice + mutex + non-blocking send shape is also visible in `internal/session/registry.go` if the executor wants additional in-repo precedent for `mu sync.Mutex` + `subscribers []*X` patterns — but RESEARCH §Pattern 4 is the binding source.

**Critical invariants (RESEARCH §Pattern 4 design notes + D-09/D-10):**

1. Lazy start, lazy stop — first `Subscribe` starts the goroutine; last `Unsubscribe` cancels it.
2. Re-open on rotation at EOF (NEVER backfill historical content on rotation — D-10).
3. Non-blocking subscriber send (`select { case sub.C <- line: default: /* drop */ }`); slow subscribers drop lines, never backpressure the tailer.
4. `bufio.Scanner.Buffer(make([]byte, 0, 64*1024), 1024*1024)` — 1MB max line per RESEARCH Pitfall 2.
5. `os.Open` only — never `O_WRONLY`, never `O_APPEND`, never `O_EXCL`. Read-only, no locks.
6. `os.Stat` + `os.SameFile` for rotation detection on each poll tick.

```go
// internal/admin/admin.go (NEW)
package admin

import (
    "net/http"
    "github.com/go-chi/chi/v5"
)

// Deps bundles the inputs the admin Handler needs. Narrow consumer-defined
// interfaces (PoolDetail, Registry) keep the admin package free of any
// internal/pool or internal/session import — same boundary discipline
// used by server.PoolDetailSource / server.RegistryStatsSource.
type Deps struct {
    Logger     *slog.Logger
    Version    string
    Commit     string
    Start      time.Time
    PoolDetail PoolDetailSource    // structurally identical to server.PoolDetailSource
    Registry   RegistryStatsSource // structurally identical to server.RegistryStatsSource
    LogPath    string              // resolved from OTTO_LOG by cmd/otto-gateway/main.go
}

// Handler returns a chi.Router serving /admin and its sub-paths.
// Mount on the OUTER router (not under any SurfaceMount block) so the
// routes inherit the same auth-exempt posture as /health.
func Handler(deps Deps) http.Handler {
    h := &handler{deps: deps, tailer: NewTailer(deps.LogPath, deps.Logger)}
    r := chi.NewRouter()
    r.Get("/", h.pageHandler)
    r.Get("/api/snapshot", h.snapshotHandler)
    r.Get("/logs/stream", h.sseHandler)
    r.Handle("/static/*", http.StripPrefix("/admin/static/", http.FileServer(http.FS(staticFS))))
    return r
}
```

### `internal/admin/assets.go` (embed declaration)

**Analog:** NONE in this codebase — Phase 6.1 is the first `embed.FS` use. The pattern is taken directly from RESEARCH §Pattern 1 (lines 260-302 of RESEARCH.md), which is the canonical stdlib idiom:

```go
package admin

import (
    "embed"
    "html/template"
    "io/fs"
)

//go:embed templates/*.html.tmpl static
var assetsFS embed.FS

// staticFS roots the embed at "static/" so http.FileServer serves
// /admin/static/css/admin.css → static/css/admin.css inside the embed.
var staticFS = mustSub(assetsFS, "static")

// pageTemplate is parsed ONCE at package init. Re-parsing per request
// burns CPU for zero benefit — the template is baked into the binary
// and cannot change at runtime.
var pageTemplate = template.Must(
    template.ParseFS(assetsFS, "templates/index.html.tmpl"),
)

func mustSub(f fs.FS, dir string) fs.FS {
    sub, err := fs.Sub(f, dir)
    if err != nil {
        panic(err)
    }
    return sub
}
```

**Critical RESEARCH Pitfall 1:** the embed glob `static` (bare directory name) recursively embeds the subtree; `static/*` would only match top-level files. Use `//go:embed templates/*.html.tmpl static` exactly as shown.

## Shared Patterns

### Logger threading (per-request + fallback)

**Source:** `internal/server/middleware.go` lines 48-55 + usage in `internal/server/health.go` line 82, `internal/server/agents.go` line 105.

```go
// LoggerFromCtx retrieves the per-request logger stored by accessLog.
// Falls back to fallback if no logger is present in the context.
func LoggerFromCtx(ctx context.Context, fallback *slog.Logger) *slog.Logger {
    if l, ok := ctx.Value(loggerKey{}).(*slog.Logger); ok {
        return l
    }
    return fallback
}
```

**Apply to:** all admin handlers (`snapshotHandler`, `sseHandler`, `pageHandler`).

**Important caveat:** `LoggerFromCtx` and the `loggerKey{}` type are in package `server` and unexported. The admin package CANNOT call them directly. Options:
1. **(Preferred)** Stash `deps.Logger` on the admin handler struct and use it directly. The access-log middleware emits the request-id-tagged log entry for free; admin handlers don't need to extract the same logger from context.
2. Export `server.LoggerFromCtx` and the context key — adds public API surface for one consumer.

**Recommendation:** Option 1. The admin handler's encode-failure logs do NOT need the request_id; the request line emitted by `accessLog` already correlates failures.

### Encode-failure logging (cannot write error after WriteHeader)

```go
if err := json.NewEncoder(w).Encode(resp); err != nil {
```

### Auth-exempt routing posture

**Source:** `server.go` lines 193-206 (the outer-router block in `NewFromConfig`) + AUTH-03 (referenced in 06.1-CONTEXT.md canonical_refs).

**Pattern:** routes registered DIRECTLY on `s.router` (outside any `s.router.Route(prefix, func(r chi.Router) { r.Use(auth.Bearer); ... })` block) are auth-exempt. The existing exempt set is `/`, `/health`, `/health/agents`, `/api/version`.

**Apply to admin (D-01/D-07):** `s.router.Mount("/admin", cfg.AdminHandler)` — placed alongside the other `s.router.Get(...)` exempt registrations, NOT inside the `SurfaceMount` for-loop that wraps prefixes in auth middleware.

### Adapter-bridge pattern for engine-boundary discipline

**Source:** `cmd/otto-gateway/main.go` lines 423-490 (`poolStatsAdapter`, `poolDetailAdapter`, `registryStatsAdapter` — one-line bridges that satisfy the server's consumer-defined interfaces without making `server` import `internal/pool` or `internal/session`).

```go
// poolDetailAdapter wraps *pool.Pool to satisfy server.PoolDetailSource.
// pool.AgentSlot (plan 05-01) and server.AgentSlot (plan 05-03) are
// structurally identical wire types declared in separate packages to
// keep the engine boundary clean; this adapter does the per-row copy.
type poolDetailAdapter struct {
    pool *pool.Pool
}

func (p poolDetailAdapter) Detail() []server.AgentSlot {
    slots := p.pool.Detail()
    out := make([]server.AgentSlot, 0, len(slots))
    for _, sl := range slots {
        out = append(out, server.AgentSlot{
            Label:            sl.Label,
            Alive:            sl.Alive,
            Busy:             sl.Busy,
            CurrentSessionID: sl.CurrentSessionID,
        })
    }
    return out
}
```

**Apply to admin:** the admin package declares its OWN `PoolDetailSource` / `RegistryStatsSource` interfaces (mirroring server's), and the EXISTING `poolDetailAdapter` / `registryStatsAdapter` values in `main.go` satisfy BOTH the admin interfaces AND the server interfaces simultaneously (Go structural typing — the method sets are identical because the slot/session types are isomorphic across both packages). No new adapter is needed in `main.go`. If the admin package needs to re-shape the slot type to its own `SnapshotSlot` (per RESEARCH §Pattern 2), do the shape conversion INSIDE the admin handler — not in the adapter.
