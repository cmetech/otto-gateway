# Phase 6.1: Admin Observability UI ŌĆö Research

**Researched:** 2026-05-27
**Domain:** Go stdlib HTTP UI delivery (`embed.FS` + `html/template`) + log-file tailing + SSE fan-out
**Confidence:** HIGH

## Summary

Phase 6.1 ships a dark-mode admin page at `GET /admin` that surfaces existing `/health` + `/health/agents` data through a single static HTML/CSS/JS bundle compiled into the OTTO Gateway binary via `embed.FS`. The page polls a NEW typed aggregator endpoint `GET /admin/api/snapshot` (composing `pool.Stats() + registry.Stats() + poolDetail.Detail() + registry.Detail()` into one atomic `AdminSnapshot` struct) every 30 seconds, and consumes a NEW SSE stream `GET /admin/logs/stream` that fans out live lines from `/tmp/otto-gateway.log` to N subscribers via one shared single-tailer goroutine with a 500-line ring buffer.

Every dimension of the page (palette, copy, spacing, typography, layout, polling cadence, log-tail UX) is already locked by `06.1-UI-SPEC.md` and `06.1-CONTEXT.md` D-01..D-16. This research locks the **Go implementation contract** that the planner consumes: how `embed.FS` is structured, how the SSE handler mirrors `internal/adapter/anthropic/sse.go`'s D-05 single-goroutine invariant, how the tailer should be built with pure stdlib (no `nxadm/tail`, no `hpcloud/tail`, no `fsnotify`), how the snapshot struct composes existing types, and how the validation surface looks under nyquist_validation.

**Primary recommendation:** Build `internal/admin/` as a self-contained `chi.Router`-returning package with one Go file per concern (`admin.go` handler constructor, `snapshot.go` aggregator + types, `tail.go` shared file tailer + ring buffer, `sse.go` SSE handler mirroring the Anthropic emitter pattern, `assets.go` `embed.FS` declaration). Use **pure stdlib `os.Open` + size-tracking + `bufio.Scanner`** for the tailer (no third-party deps). Use **`html/template`** with one template file. The page is hydrated dynamically via JS; the template only bakes in `version`/`commit`/`commit_time` once at render. Cross-compile is preserved because `embed.FS` is a 1.16-vintage stdlib feature and ships zero cgo dependencies.

## User Constraints (from CONTEXT.md)

### Locked Decisions

**Auth posture (v1 only)**
- **D-01:** No auth on `/admin` or any of its sub-routes in v1. The page is served regardless of whether `AUTH_TOKEN` is set. Reason: lean delivery; the operator's network is assumed trusted (localhost / private VPC). Risk: anyone with HTTP access to the gateway can see pool slot labels, session IDs, last-used timestamps, and (via the log tail) every log line including DEBUG-level traces of request paths. Treated as acceptable for v1 because the same network already trusts `/health` and `/health/agents` (which are auth-exempt per AUTH-03). Worth revisiting if/when the gateway ships outside trusted networks.
- **D-02:** Auth is explicitly **deferred** as a follow-on. Phase 8 (plugin hook chain) is the right home for a per-route auth policy that could gate `/admin` behind the same `AuthHook` that protects `/v1/*` and `/api/*`. Not blocking v1.

**Page layout / information density**
- **D-03:** Single-page admin view with four stacked sections:
  1. Summary header strip
  2. Pool slots grid (3-column responsive)
  3. Active sessions table
  4. Log tail panel (SSE-driven)
- **D-04:** The `embeddings` field in `/health` JSON is left in place (zero-value) for wire backward compatibility but is NOT rendered on the admin page.

**Refresh / polling architecture**
- **D-05:** Single aggregator endpoint `GET /admin/api/snapshot` returns one unified JSON snapshot composing `pool.Stats()` + `registry.Stats()` + `poolDetail.Detail()` + the registry's session detail. Atomic snapshot taken at one instant on the server.
- **D-06:** Client-side polling at **30s fixed interval**.
- **D-07:** Snapshot endpoint is registered at the **router root** alongside `/health`. Auth-exempt per D-01.

**Log tail architecture**
- **D-08:** SSE-based live tail at `GET /admin/logs/stream`.
- **D-09:** **Shared single-tailer pattern** ŌĆö exactly ONE goroutine across the whole gateway tails the log file at a time. Maintains a 500-line in-memory ring buffer and fans out new lines to N subscribed SSE clients. Started lazily on first SSE connect; exits when the last subscriber disconnects.
- **D-10:** File access is strictly **read-only and non-blocking.** Open with `os.Open`. On read error (file rename / missing / permission revoked), close and re-open at the new file's EOF ŌĆö log rotation must not crash the tailer or backfill old content. Zero changes to the existing log-writing code path.
- **D-11:** SSE handler tears down cleanly on client disconnect.
- **D-12:** UI surface for log tail (level dropdown, regex grep, pause/resume, "newest" badge, color-coded by level, backfill last Ōēż500 lines on connect).

**Distribution / single-binary**
- **D-13:** Templates + CSS + JS shipped via `embed.FS` declared in a new `internal/admin/` package. Cross-compile from macOS to `linux/amd64` and `windows/amd64` with `CGO_ENABLED=0` verified as part of phase exit.

**Brand palette (D-14):** See UI-SPEC ┬¦Color ŌĆö palette tokens sourced verbatim from `oscar-adminui/src/layouts/UserThemeOptions.js` `customColors`.

**Non-interference contract**
- **D-15:** Surface gating via `ENABLED_SURFACES` does NOT affect `/admin` or `/admin/*`.
- **D-16:** Mounting the admin routes does NOT change the registration, ordering, or middleware chain of any existing `/api/*`, `/v1/*`, `/health`, `/health/agents` routes.

### Claude's Discretion

- Exact CSS class names, file layout inside `internal/admin/`, ring buffer struct shape, and SSE event framing details are implementation choices for the planner / executor.
- The 500-line ring buffer size, 30s poll interval default, and whether to wrap the log tail in a fixed-height scrollable region are reasonable starting points ŌĆö the planner can adjust if a more sensible default surfaces during research.
- All spacing/typography/copy/responsive-breakpoint dimensions are already locked by `06.1-UI-SPEC.md` (gsd-ui-checker-approved). The planner should treat that spec as a downstream input.

### Deferred Ideas (OUT OF SCOPE)

- **Auth on `/admin`** ŌĆö Postponed from v1 per D-01/D-02. Right home is Phase 8.
- **Operator-selectable refresh interval** ŌĆö D-06 locks it at 30s for v1.
- **Historical metrics / sparklines** ŌĆö Out of scope; would need a server-side metric store.
- **Request rate / throughput counters** ŌĆö Out of scope; would need new instrumentation in the engine.
- **Server-side log filtering** ŌĆö D-12 keeps level/grep filtering client-side.
- **Stripping the dead `embeddings` field from `/health`** ŌĆö D-04 leaves it for backward compat.

### Alternatives Considered (and rejected)

| Instead of | Could Use | Tradeoff | Verdict |
|------------|-----------|----------|---------|
| stdlib `os.Open + bufio.Scanner` poll-loop | `github.com/nxadm/tail` `[ASSUMED]` (community fork of `hpcloud/tail`) | `tail` handles rotation, truncation, follow-name semantics out of the box. But: adds external dep (project has only 2 deps today ŌĆö `chi/v5` and `goleak`), pulls in `fsnotify` (which has historically had Windows quirks), and the tailer use-case here is small enough (~150 lines) that a custom impl is auditable. | **REJECTED.** Minimalism + single-binary discipline. Pure stdlib also makes `gosec`/`govulncheck` reviews trivial. |
| stdlib | `github.com/fsnotify/fsnotify` | OS-native file event watching (inotify on Linux, kqueue on macOS, ReadDirectoryChangesW on Windows). | **REJECTED.** Adds dep with cross-platform quirks (Windows rename events are notoriously flaky). 250ms polling is fine for an admin-UI log tail; we're not building a high-frequency change-detector. |
| `html/template` | `text/template` | Faster (no context-aware escaping). | **REJECTED.** Page interpolates `version`, `commit`, and (if any future fields land) anything else operator-visible. Using `text/template` would invite XSS regressions if a future commit contains an HTML-special character. `html/template` overhead is negligible for one render per page load. |
| Inline `<style>` + `<script>` in the template | Separate `embed.FS` files served via `http.FileServer` | Inline ships ONE document; no second HTTP round-trip. | **MIXED.** UI-SPEC's Asset Inventory locks the separate-file layout (`internal/admin/static/css/admin.css`, `internal/admin/static/js/admin.js`). Keeping them as separate files allows browser caching across page reloads + makes the CSS/JS auditable as standalone artifacts. Accept the one-extra-request cost. |
| Server-Sent Events for log tail | WebSocket | Bidirectional comms. | **REJECTED.** SSE is unidirectional (server ŌåÆ client) which fits the log-tail use case exactly. The Anthropic adapter has already proven the SSE pattern works in this codebase. WebSocket would add the `gorilla/websocket` dep and complicate the auth-exempt routing. |

### Pattern 1: `embed.FS` with `fs.Sub` rooted on `static/`

**What:** Embed templates and static assets compiled into the binary; serve static assets via `http.FileServer(http.FS(staticFS))` so URLs are `/admin/static/css/admin.css` (not `/admin/static/static/css/...`).

**When to use:** Single-binary deliverables where you want one CDN-free artifact. Cross-compile-safe.

**Example:**

```go
// internal/admin/assets.go
package admin

import (
    "embed"
    "html/template"
    "io/fs"
    "net/http"
)

//go:embed templates/*.html.tmpl static
var assetsFS embed.FS

// staticFS roots the embed at "static/" so http.FileServer serves
// /admin/static/css/admin.css ŌåÆ static/css/admin.css inside the embed.
var staticFS = mustSub(assetsFS, "static")

// pageTemplate is parsed ONCE at package init. Re-parsing per request
// burns CPU for zero benefit ŌĆö the template is baked into the binary
// and cannot change at runtime.
var pageTemplate = template.Must(
    template.ParseFS(assetsFS, "templates/index.html.tmpl"),
)

func mustSub(f fs.FS, dir string) fs.FS {
    sub, err := fs.Sub(f, dir)
    if err != nil {
        // ParseFS at init would have already failed if the embed is
        // misconfigured; if mustSub panics, the binary is broken at compile time.
        panic(err)
    }
    return sub
}

// staticHandler is the http.Handler for /admin/static/*.
// http.FileServer + http.FS strip-prefix dance: chi's r.Handle("/static/*", ...)
// hands us the URL with /admin already stripped, so we additionally strip
// the /static/ leg.
func staticHandler() http.Handler {
    return http.StripPrefix("/static/", http.FileServer(http.FS(staticFS)))
}
```

### Pattern 2: Snapshot aggregator (mirroring `health.go` style)

**What:** Compose existing typed stats sources into one atomic JSON response. Nil-safe accessors, `json.NewEncoder(w).Encode(resp)`, error-logging on encode failure ŌĆö verbatim from the established `server.healthHandler` pattern.

**When to use:** Any new observability endpoint that aggregates data sources owned by other packages.

**Example:**

```go
// internal/admin/snapshot.go
package admin

import (
    "encoding/json"
    "net/http"
    "time"
)

// AdminSnapshot is the response body for GET /admin/api/snapshot.
// Composes pool + sessions detail into one atomic snapshot so the
// client doesn't see skew across two polls.
//
// Wire shape stability: JSON tags are the contract. Downstream JS
// can rely on snake_case keys; future fields go on existing sub-structs.
type AdminSnapshot struct {
    Status        string         `json:"status"`           // "ok" | "degraded" | "down"
    Version       string         `json:"version"`
    Commit        string         `json:"commit"`
    UptimeSeconds float64        `json:"uptime_seconds"`
    GeneratedAt   time.Time      `json:"generated_at"`     // RFC3339; client renders "Xs ago"
    Pool          SnapshotPool   `json:"pool"`
    Sessions      []SnapshotSess `json:"sessions"`
}

// SnapshotPool reuses the AgentsPool shape but inlines slot detail so
// the page renders the card grid + counts from ONE struct without an
// extra walk.
type SnapshotPool struct {
    Size  int             `json:"size"`
    Alive int             `json:"alive"`
    Busy  int             `json:"busy"`
    Slots []SnapshotSlot  `json:"slots"`
}

type SnapshotSlot struct {
    Label            string  `json:"label"`
    Alive            bool    `json:"alive"`
    Busy             bool    `json:"busy"`
    CurrentSessionID *string `json:"current_session_id"`
}

type SnapshotSess struct {
    ID       string    `json:"id"`
    Alive    bool      `json:"alive"`
    Busy     bool      `json:"busy"`
    LastUsed time.Time `json:"last_used"`
    Model    *string   `json:"model"`
}

// snapshotHandler renders the AdminSnapshot. Nil-safe ŌĆö when the pool
// or registry source is unset (degraded "KIRO_CMD unset" mode), the
// corresponding sub-tree renders zero/empty rather than panicking.
func (h *Handler) snapshotHandler(w http.ResponseWriter, r *http.Request) {
    snap := AdminSnapshot{
        Version:       h.version,
        Commit:        h.commit,
        UptimeSeconds: time.Since(h.start).Seconds(),
        GeneratedAt:   time.Now().UTC(),
        Pool:          SnapshotPool{Slots: []SnapshotSlot{}},
        Sessions:      []SnapshotSess{},
    }
    if h.poolDetail != nil {
        slots := h.poolDetail.Detail()
        snap.Pool.Slots = make([]SnapshotSlot, 0, len(slots))
        for _, s := range slots {
            if s.Alive { snap.Pool.Alive++ }
            if s.Busy  { snap.Pool.Busy++  }
            snap.Pool.Slots = append(snap.Pool.Slots, SnapshotSlot{
                Label: s.Label, Alive: s.Alive, Busy: s.Busy,
                CurrentSessionID: s.CurrentSessionID,
            })
        }
        snap.Pool.Size = len(slots)
    }
    if h.registry != nil {
        details := h.registry.Detail()
        for _, d := range details {
            snap.Sessions = append(snap.Sessions, SnapshotSess{
                ID: d.ID, Alive: d.Alive, Busy: d.Busy,
                LastUsed: d.LastUsed, Model: d.Model,
            })
        }
    }
    snap.Status = computeStatus(snap)

    w.Header().Set("Content-Type", "application/json")
    w.Header().Set("Cache-Control", "no-store")  // pollers must NOT cache
    w.WriteHeader(http.StatusOK)
    if err := json.NewEncoder(w).Encode(snap); err != nil {
        h.logger.Error("admin: snapshot encode", "err", err)
    }
}

// computeStatus derives the page-level "HEALTHY | DEGRADED | DOWN" pill.
//   - DOWN: pool unset OR every slot dead
//   - DEGRADED: at least one slot dead, at least one alive
//   - HEALTHY: all slots alive
func computeStatus(snap AdminSnapshot) string {
    if snap.Pool.Size == 0 || snap.Pool.Alive == 0 {
        return "down"
    }
    if snap.Pool.Alive < snap.Pool.Size {
        return "degraded"
    }
    return "ok"  // UI maps "ok" ŌåÆ "HEALTHY" pill
}
```

### Pattern 3: SSE handler mirroring `internal/adapter/anthropic/sse.go`

**What:** Per-request goroutine owns the `http.ResponseWriter` and `http.Flusher`. Single `select` loop watches `ctx.Done()`, a `time.Ticker` (15s keepalive), and the per-subscriber fan-out chan. No writes outside this goroutine ŌĆö the D-05 single-goroutine invariant.

**When to use:** Any SSE endpoint in this codebase. Established pattern, has goroutine-leak tests via `goleak`.

**Example:**

```go
// internal/admin/sse.go
package admin

import (
    "context"
    "fmt"
    "net/http"
    "time"
)

const (
    SSEKeepaliveInterval = 15 * time.Second // matches anthropic/sse.go:24
    SSEFanoutBuffer       = 16              // per-subscriber chan capacity
)

// sseHandler handles GET /admin/logs/stream.
//
// D-05 single-goroutine invariant: this goroutine is the SOLE writer to
// w + flusher. Backfill, live forwarding, and keepalive pings all flow
// through one select loop. No mutex needed on w.
func (h *Handler) sseHandler(w http.ResponseWriter, r *http.Request) {
    flusher, ok := w.(http.Flusher)
    if !ok {
        http.Error(w, "streaming unsupported", http.StatusInternalServerError)
        return
    }

    w.Header().Set("Content-Type", "text/event-stream")
    w.Header().Set("Cache-Control", "no-cache")
    w.Header().Set("Connection", "keep-alive")
    w.Header().Set("X-Accel-Buffering", "no") // disable nginx buffering if behind reverse proxy
    w.WriteHeader(http.StatusOK)

    sub := h.tailer.Subscribe(r.Context())
    defer h.tailer.Unsubscribe(sub) // unsubscribe also closes the shared tailer if last subscriber

    // Backfill: send the ring buffer contents IMMEDIATELY so the operator
    // has context before the live stream starts.
    for _, line := range h.tailer.Snapshot() {
        writeSSELine(w, "log", line)
    }
    flusher.Flush()

    ticker := time.NewTicker(SSEKeepaliveInterval)
    defer ticker.Stop()

    for {
        select {
        case <-r.Context().Done():
            h.logger.Debug("admin: sse client disconnect")
            return
        case <-ticker.C:
            writeSSELine(w, "ping", "")
            flusher.Flush()
        case line, ok := <-sub.C:
            if !ok {
                return // tailer shut down (last subscriber departed during a race)
            }
            writeSSELine(w, "log", line)
            flusher.Flush()
        }
    }
}

// writeSSELine emits one SSE frame: `event: <name>\ndata: <payload>\n\n`.
// The payload is the raw log line; data is NOT JSON-encoded ŌĆö log lines
// are already either text or JSON (slog JSON handler) and are forwarded
// verbatim. The client parses the level field from JSON lines on its end.
//
// Multi-line payloads (if a log line contains \n) MUST split into multiple
// data: prefixes per SSE spec. defensive: stdlib log lines never embed \n
// except in panic stack traces, which the recoverer middleware catches
// and emits as JSON ŌĆö but the splitter handles them anyway.
func writeSSELine(w http.ResponseWriter, eventName, payload string) {
    if eventName != "" {
        _, _ = fmt.Fprintf(w, "event: %s\n", eventName)
    }
    // SSE spec: each data: line is one logical line; \n in payload
    // splits into multiple data: prefixes within ONE event.
    if payload == "" {
        _, _ = fmt.Fprint(w, "data:\n\n")
        return
    }
    for _, segment := range splitLines(payload) {
        _, _ = fmt.Fprintf(w, "data: %s\n", segment)
    }
    _, _ = fmt.Fprint(w, "\n")
}
```

### Pattern 4: Shared single-tailer with ring buffer + fan-out

**What:** One goroutine across the whole gateway tails the log file via stdlib poll. Maintains a 500-line ring buffer. `Subscribe(ctx)` returns a chan that receives new lines AFTER subscribe; `Snapshot()` returns the current buffer contents for backfill. Started lazily on first `Subscribe`; goroutine exits when last subscriber unsubscribes.

**When to use:** Any time multiple HTTP clients need to observe the same append-only file without each opening their own file handle.

**Example skeleton:**

```go
// internal/admin/tail.go
package admin

import (
    "bufio"
    "context"
    "log/slog"
    "os"
    "sync"
    "time"
)

const (
    RingBufferLines  = 500              // D-09 default; planner may tune
    TailPollInterval = 250 * time.Millisecond // poll cadence for size-delta read
)

// Tailer fans new log lines from path to N subscribed channels.
// Concurrency: one writer goroutine (run()) + N reader goroutines
// (SSE handlers). subscribers + ring buffer guarded by mu.
type Tailer struct {
    path   string
    logger *slog.Logger

    mu          sync.Mutex
    ring        *RingBuffer // last RingBufferLines lines
    subscribers []*subscriber
    running     bool
    cancelRun   context.CancelFunc
}

type subscriber struct {
    C    chan string
    done chan struct{}
}

// NewTailer constructs a tailer rooted at path. It does NOT start
// polling until the first Subscribe.
func NewTailer(path string, logger *slog.Logger) *Tailer {
    return &Tailer{
        path:   path,
        logger: logger,
        ring:   NewRingBuffer(RingBufferLines),
    }
}

// Subscribe returns a channel that receives every new line read AFTER
// this call. Subscribers SHOULD also call Snapshot() to backfill.
// Caller MUST call Unsubscribe when ctx is done OR when the subscriber
// is dropped ŌĆö leaking subscribers blocks the fan-out goroutine.
func (t *Tailer) Subscribe(ctx context.Context) *subscriber {
    t.mu.Lock()
    defer t.mu.Unlock()
    sub := &subscriber{
        C:    make(chan string, SSEFanoutBuffer),
        done: make(chan struct{}),
    }
    t.subscribers = append(t.subscribers, sub)
    if !t.running {
        // Lazy start: first subscriber spins up the tailer goroutine.
        runCtx, cancel := context.WithCancel(context.Background())
        t.cancelRun = cancel
        t.running = true
        go t.run(runCtx)
    }
    return sub
}

// Unsubscribe removes sub from the fan-out. If sub was the last
// subscriber, the shared tailer goroutine is cancelled.
func (t *Tailer) Unsubscribe(sub *subscriber) {
    t.mu.Lock()
    defer t.mu.Unlock()
    for i, s := range t.subscribers {
        if s == sub {
            t.subscribers = append(t.subscribers[:i], t.subscribers[i+1:]...)
            close(sub.C)
            break
        }
    }
    if len(t.subscribers) == 0 && t.running {
        t.cancelRun()
        t.running = false
    }
}

// Snapshot returns a copy of the current ring buffer contents for backfill.
func (t *Tailer) Snapshot() []string {
    t.mu.Lock()
    defer t.mu.Unlock()
    return t.ring.Copy()
}

// run is the single tailer goroutine. It opens the file at EOF, polls
// for size growth, reads new bytes through bufio.Scanner, pushes lines
// into the ring buffer AND fans out to subscribers (non-blocking ŌĆö a
// slow subscriber drops lines rather than backpressuring the tailer).
//
// On read error (file rename / missing / truncate), close and re-open
// at EOF on next tick. NEVER backfill historical content on re-open
// ŌĆö D-10 invariant.
func (t *Tailer) run(ctx context.Context) {
    var (
        f          *os.File
        lastSize   int64
        scanner    *bufio.Scanner
        scanBuffer = make([]byte, 0, 64*1024)
    )

    reopen := func() {
        if f != nil {
            _ = f.Close()
            f = nil
        }
        nf, err := os.Open(t.path)
        if err != nil {
            // log file does not exist yet (start before scripts/otto-gw),
            // or path is wrong ŌĆö log once per minute, keep retrying.
            t.logger.Debug("admin: tailer cannot open log", "path", t.path, "err", err)
            return
        }
        // Seek to END so we never backfill historical content (D-10).
        sz, err := nf.Seek(0, 2) // io.SeekEnd
        if err != nil {
            _ = nf.Close()
            return
        }
        f = nf
        lastSize = sz
        scanner = bufio.NewScanner(f)
        scanner.Buffer(scanBuffer, 1024*1024) // 1MB max line len
    }

    reopen()
    ticker := time.NewTicker(TailPollInterval)
    defer ticker.Stop()
    defer func() {
        if f != nil { _ = f.Close() }
    }()

    for {
        select {
        case <-ctx.Done():
            return
        case <-ticker.C:
            if f == nil {
                reopen()
                continue
            }
            // Detect rotation: stat the path, if size shrunk or inode
            // changed, re-open. Inode check via os.SameFile.
            st, err := os.Stat(t.path)
            if err != nil {
                t.logger.Debug("admin: tailer stat failed", "err", err)
                reopen()
                continue
            }
            fst, ferr := f.Stat()
            if ferr != nil || st.Size() < lastSize || !os.SameFile(fst, st) {
                reopen()
                continue
            }
            // Read new bytes if size grew.
            if st.Size() == lastSize {
                continue
            }
            for scanner.Scan() {
                line := scanner.Text()
                t.broadcast(line)
            }
            if err := scanner.Err(); err != nil {
                t.logger.Debug("admin: tailer scan", "err", err)
                reopen()
                continue
            }
            // bufio.Scanner returns false at EOF; we need a fresh scanner
            // on next iteration so it doesn't stay in "ended" state.
            scanner = bufio.NewScanner(f)
            scanner.Buffer(scanBuffer, 1024*1024)
            lastSize = st.Size()
        }
    }
}

// broadcast pushes a line into the ring buffer AND each subscriber chan.
// Subscribers with a full buffer DROP the line ŌĆö never block the tailer.
func (t *Tailer) broadcast(line string) {
    t.mu.Lock()
    t.ring.Push(line)
    subs := t.subscribers
    t.mu.Unlock()
    for _, sub := range subs {
        select {
        case sub.C <- line:
        default:
            // Drop: this subscriber is slow. The tailer keeps moving.
            // Operator may briefly see gaps in the SSE stream ŌĆö acceptable
            // tradeoff vs. blocking the shared tailer.
        }
    }
}
```

**Critical design notes:**
1. **Lazy start / lazy stop:** Subscribe spins up the tailer goroutine; Unsubscribe of the last subscriber stops it. Prevents goroutine + file handle leaks when no operators are watching.
2. **Backpressure:** Subscriber chans have a finite buffer; full buffer = drop the line for that subscriber. The shared tailer NEVER blocks on a slow client.
3. **Rotation handling:** `os.Stat` + `os.SameFile` on every tick detects rotation. On detection, close + re-open at the new file's EOF. NEVER backfill old content ŌĆö D-10 invariant.
4. **Empty path / missing file:** `os.Open` failure logs once per tick (DEBUG level). The tailer retries on every tick until the file appears. No crash.
5. **Concurrent subscribers:** `t.subscribers` is a slice protected by `t.mu`. Snapshot of the slice taken under lock before fan-out; the broadcast loop iterates the snapshot without holding the lock so a concurrent Subscribe/Unsubscribe doesn't deadlock.

### Anti-Patterns to Avoid

- **Re-parsing the template per request.** Parse once in `assets.go` (`var pageTemplate = template.Must(template.ParseFS(...))`). Per-request parse burns CPU; the binary contents are immutable.
- **Opening the log file in each SSE handler.** D-09 mandates ONE goroutine. Per-request opens explode file-handle count under N operators + leak goroutines on disconnect-race.
- **Holding the tailer mutex across the file read.** The mutex protects only the subscriber slice + ring buffer push. File I/O happens OUTSIDE the lock so a slow disk read doesn't block Subscribe/Unsubscribe.
- **Blocking on a slow subscriber chan.** `select { case sub.C <- line: default: /* drop */ }` is mandatory ŌĆö never `sub.C <- line` directly. Dropping is the right call for live-tail UX.
- **Backfilling historical log content on rotation.** D-10 says re-open at EOF, NOT at byte 0. Otherwise rotation floods every operator with the entire old log.
- **JSON-encoding log lines in the SSE handler.** Lines are already in their final wire form (text or JSON-line). Re-encoding adds CPU and changes the on-the-wire bytes the client expects.
- **Adding middleware to `/admin/*`** that's not on `/health`. D-15/D-16 require non-interference; any new middleware must be opt-in for this prefix only.
- **Forgetting `Cache-Control: no-store` on `/admin/api/snapshot`.** Without it, the browser may serve a stale snapshot from cache and the 30s poll silently becomes a 0s poll of stale data.
- **Using `text/template` instead of `html/template`.** XSS regression risk if any operator-supplied data ever lands in the template ŌĆö `html/template` is XSS-safe by default.

## Common Pitfalls

### Pitfall 1: `embed` `//go:embed` directive scope
**What goes wrong:** `//go:embed templates/*` matches files NOT subdirectories by default. Using `//go:embed templates/* static/*` to capture both top-level dirs and their contents requires careful pattern. Subdirectories under `static/` (css, js) need a recursive glob: `//go:embed static`.
**Why it happens:** Glob semantics in `embed` differ from shell glob ŌĆö `*` matches files only.
**How to avoid:** Use `//go:embed templates/*.html.tmpl static` ŌĆö naming the bare directory `static` recursively embeds it; the explicit `.html.tmpl` glob narrows the template match.
**Warning signs:** `fs.Sub` succeeds but `template.ParseFS` returns "pattern matches no files", or `http.FileServer` returns 404 for files you can see in the directory.
`[CITED: pkg.go.dev/embed ┬¦Directives]`

### Pitfall 2: `bufio.Scanner` default buffer limit
**What goes wrong:** Default `bufio.Scanner.Buffer` is 64KB. Stack traces or large JSON log lines exceed this and `Scanner.Err()` returns `bufio.ErrTooLong`; the line is lost forever.
**Why it happens:** Default is conservative for arbitrary input; log lines can be much longer than typical lines.
**How to avoid:** `scanner.Buffer(make([]byte, 0, 64*1024), 1024*1024)` ŌĆö initial 64KB, grow to 1MB max. Lines >1MB are still dropped but the gateway logs no such monsters.
**Warning signs:** Operator reports "missing log lines" or `tailer scan: bufio.Scanner: token too long` in DEBUG logs.

### Pitfall 3: SSE behind a reverse proxy buffers responses
**What goes wrong:** nginx (and others) buffers responses; SSE clients see lines arrive in 4KB-or-larger chunks rather than line-by-line.
**Why it happens:** Default proxy_buffering is on.
**How to avoid:** Set `X-Accel-Buffering: no` response header. Most proxies (nginx, kong, traefik) respect it. The Anthropic SSE handler currently does NOT set this ŌĆö phase 6.1's handler SHOULD because admin tail-streaming is more latency-sensitive than chat streaming.
**Warning signs:** SSE works locally but lines clump in production.

### Pitfall 4: Missing `http.Flusher` cast
**What goes wrong:** `w.(http.Flusher)` cast fails when the handler is wrapped by middleware that swaps in a non-flushable writer (chi's `middleware.NewWrapResponseWriter` IS flushable; some custom middlewares aren't).
**Why it happens:** Middleware chains can substitute the writer; the cast is not guaranteed.
**How to avoid:** Cast and check at handler entry; return `http.Error(w, ..., 500)` on failure. Mirrors anthropic/sse.go:487-498 exactly.
**Warning signs:** Lines accumulate in the writer's internal buffer; client sees nothing until handler returns.

### Pitfall 5: `embed.FS` files not stripped from binary
**What goes wrong:** Binary size grows by the size of the embedded files. For a typical admin slice (~10KB template + ~5KB CSS + ~5KB JS = 20KB), this is negligible. But: large fonts (we use system fonts per UI-SPEC) or large images (we use Unicode glyphs) would balloon binary size.
**Why it happens:** `embed.FS` literally embeds the bytes ŌĆö there's no compression by default.
**How to avoid:** UI-SPEC mandates no web fonts and no icon library. Stick to that. Total admin bundle should stay <30KB. Verify post-build with `ls -la bin/otto-gateway` ŌĆö binary should grow by Ōēż30KB vs. baseline.
**Warning signs:** Binary exceeds the 25MB target in BLD-04.

### Pitfall 6: Test isolation ŌĆö tailer goroutine leaks across tests
**What goes wrong:** If `TestSSEHandler_BasicForward` returns without unsubscribing, the shared tailer goroutine keeps running. `goleak.VerifyNone(t)` flags this.
**Why it happens:** SSE handler tests use `httptest.NewRecorder` which doesn't trigger `r.Context().Done()`; the handler's select loop never exits.
**How to avoid:** Use `httptest.NewRequestWithContext(ctx, ...)` with a cancelable ctx; cancel ctx after assertions complete. Combine with `goleak.VerifyTestMain` at package init.
**Warning signs:** `TestMain` failure with "goroutine leak" pointing at `admin.(*Tailer).run`.

### Pitfall 7: Log file path resolution divergence
**What goes wrong:** `scripts/otto-gw` defaults `OTTO_LOG=/tmp/otto-gateway.log`. The Go binary does NOT currently read `OTTO_LOG`. The admin package needs the path BUT: the gateway when run directly via `go run` writes to stdout, not the file.
**Why it happens:** Logging is shell-redirection-based, not Go-managed.
**How to avoid:** Admin Deps struct includes a `LogPath string` field. Wiring in `cmd/otto-gateway/main.go` populates it from `os.Getenv("OTTO_LOG")`, defaulting to `/tmp/otto-gateway.log`. If the file doesn't exist (e.g., when running via `go run`), the tailer logs DEBUG once per tick and the UI shows "Waiting for log activityŌĆ”" forever ŌĆö acceptable degraded mode.
**Warning signs:** Operator runs `go run ./cmd/otto-gateway` directly (no scripts/otto-gw), opens /admin, and gets a perpetual "Waiting for log activityŌĆ”" panel.

### Pitfall 8: Snapshot endpoint races vs. pool shutdown
**What goes wrong:** Operator hits `/admin/api/snapshot` during graceful shutdown; `pool.Detail()` runs on a partially-closed pool and panics.
**Why it happens:** `*pool.Pool.Detail()` is concurrency-safe under `p.mu.Lock` but the shutdown path may close stat-tracking channels.
**How to avoid:** Existing pool.Detail() handles nil slots defensively (line 41-43 of detail.go). No new mitigation needed; the snapshot handler inherits pool's safety guarantees.
**Warning signs:** Panic in `server.Shutdown` window during operator-driven testing.

### Network-Layer Trust (v1 explicit acceptance)
