<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| context.AfterFunc goroutine → ACP pipe | Watchdog goroutine calls Cancel which writes to kiro-cli stdin via sendNotification |
| fake-ACP wire | JSON-RPC frames read by fakeACPServer.serve() from the pipe; session/cancel is an untrusted notification in production (kiro-cli is trusted in dev) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-08 | Denial of Service | Watchdog goroutine leak per request if AfterFunc fires and Cancel blocks | mitigate | Cancel is best-effort fire-and-forget (sendNotification non-blocking); watchdog goroutine exits immediately after Cancel returns; goleak validates in TestWatchdog tests |
| T-04-09 | Spoofing | session/cancel notification carrying wrong sessionId | mitigate | TestIntegration_CancelFrame asserts f.lastCancelSID == passed sid; engine watchdog captures sid from its Run() closure so the ID is the one from NewSession |
| T-04-10 | Repudiation | Watchdog fires Cancel for a session that already completed normally (stop() returned false) | accept | Cancel is idempotent best-effort notification; kiro-cli ignores cancel for unknown/completed sessions; documented in StopWatchdog() accessor comment |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new external packages — test-only files using existing go.uber.org/goleak and stdlib |
</threat_model>
