Test 2: TestWatchdog_StopPreventsCancel_OnNormalCompletion
Verify the Codex-strengthened ordering: stop() FIRST, then cancelFn(), then assert no Cancel. This proves stop() actually prevents the goroutine (not just that it returned true).
Steps:
1. Construct fakeACP with newSessionID: "watchdog-stop-sid" and chunksToEmit: one text chunk.
2. newTestEngine(t, ack) + Run with background ctx (no cancel).
3. Drain run.Stream().Chunks() to completion (range until closed).
4. run.Stream().Result() to unblock the stream.
5. FIRST: call the stop function: `if stop := run.StopWatchdog(); stop != nil { stopped := stop(); _ = stopped }`. stop() may return true (goroutine was never started, most common) or false (goroutine was in flight — still idempotent per Pitfall 4).
6. THEN: create a separate cancelCtx just to confirm the AfterFunc goroutine is truly done — but we cannot call cancelFn on the original background ctx. Instead, assert that no cancelCalls occurred. Since the original ctx is context.Background() (never canceled), the watchdog goroutine cannot fire anyway in this case. The test instead proves that run.StopWatchdog() returns a non-nil func (the watchdog was registered) and the func can be called without panic.
7. Assert len(fakeACP.cancelCalls) == 0.
8. goleak.VerifyNone(t).

Note on the Codex ordering concern: The original pattern "call stop() then assert no Cancel" is strongest when we can artificially cancel the ctx AFTER stop(). However, since the background ctx is never canceled, we get the same proof: the watchdog goroutine is deregistered (stop() returns true), and even if we canceled the ctx NOW, the goroutine would not fire. If testing with a cancelable ctx, the correct sequence is: drain stream → stop() → cancelFn() → wait brief deadline → assert cancelCalls empty. Implement this stronger variant using a context.WithCancel:

```go
ctx, cancelFn := context.WithCancel(context.Background()); defer cancelFn()
run, _ := eng.Run(ctx, req)
for range run.Stream().Chunks() {}
_, _ = run.Stream().Result()
// FIRST stop the watchdog
if stop := run.StopWatchdog(); stop != nil { stop() }
// THEN cancel the ctx — should NOT trigger Cancel(sid) since stop() already ran
cancelFn()
// Brief wait then assert
time.Sleep(50 * time.Millisecond) // this one sleep is justified: AfterFunc goroutine has 50ms to (not) execute
ack.mu.Lock()
calls := ack.cancelCalls
ack.mu.Unlock()
if len(calls) > 0 {
    t.Errorf("watchdog: unexpected Cancel calls after stop() + cancelFn(): %v", calls)
}
```
This matches the Codex recommendation: stop() → cancel ctx → assert no Cancel.
