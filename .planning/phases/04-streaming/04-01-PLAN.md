Lay the structural foundation that all subsequent Phase 4 plans build on:
(1) Fix the wire.Stream bool → *bool zero-value bug so absent stream fields default to true (Node parity per D-03 / Pitfall 1 in RESEARCH.md).
(2) Add the engine-owned disconnect-cancel watchdog via context.AfterFunc in engine.Run() with a StopWatchdog() teardown accessor (D-06 in CONTEXT.md).

Doing them together in Wave 1 prevents a dependency race and ensures the atomic coupling required by Pitfall 5 (the wire type change must land before the handlers.go branch that reads it).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| JSON decoder → wire.Stream *bool | Client-supplied stream field parsed by encoding/json; nil is valid (absent) |
| context.AfterFunc goroutine → ACP pipe | Watchdog goroutine writes to kiro-cli stdin via sendNotification; must not touch HTTP ResponseWriter |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Spoofing | engine watchdog: AfterFunc goroutine calls Cancel after normal stream close (stop() race) | accept | Cancel is idempotent best-effort; kiro-cli ignores cancel for unknown/closed sessions; stop() returning false is the documented race and is safe (RESEARCH.md Pitfall 4) |
| T-04-02 | Tampering | wire.Stream *bool nil-deref in streamEnabled | mitigate | streamEnabled guards `s == nil` before `*s`; nil is valid and means stream:true |
| T-04-03 | Denial of Service | watchdog goroutine leak per request if StopWatchdog() never called | mitigate | goleak.VerifyTestMain in engine/testmain_test.go catches this in CI; Plan 03 watchdog_test.go exercises normal-completion path |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new external packages in this phase — stdlib only (context.AfterFunc is Go 1.21+ stdlib); no install tasks |
</threat_model>
