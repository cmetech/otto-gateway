WHY THIS MUST BE ONE PLAN (review finding):
- FACT 1: engine.Collect routes through engine.Run (collect.go:33). Once the AfterFunc watchdog is added to Run, every stream:false request also registers a watchdog. If stop() is not called in Collect after natural stream completion, every non-streaming response fires a spurious session/cancel when r.Context() ends. This is a semantic bug now and a real slot-recycling bug in Phase 5 (ACPClient doc says "Phase 5 adds slot-release on Cancel").
- FACT 2: Adding Run() to ollama.Engine breaks main.go:168 (engineForAdapter = a.engine) because *engine.Engine.Run returns *engine.Run not ollama.RunHandle — Go rejects this. An ollamaEngineAdapter + ollamaRunHandleAdapter shim (mirroring the existing openai/anthropic shims at main.go:388-427) must be added in the SAME commit.
- FACT 3: Adding StopWatchdog() to openai.RunHandle and anthropic.RunHandle breaks the existing openaiRunHandleAdapter and anthropicRunHandleAdapter shims (which only have Stream() + SessionID()). StopWatchdog() must be forwarded to h.run.StopWatchdog() in all three *RunHandleAdapter structs.

Wave 1 leaves the codebase green at every level: go build ./..., go vet ./..., go test -race ./... all exit 0.

Purpose: All three streaming surfaces (Ollama NDJSON in Plan 02, OpenAI/Anthropic ratification in Plan 04) and all three non-streaming Collect paths are spurious-cancel-free from the moment the watchdog is introduced.
