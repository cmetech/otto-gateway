Create internal/adapter/ollama/ndjson.go (package ollama):
The file has four exported-or-package-level pieces:

1. ndjsonChatLine struct — intermediate /api/chat NDJSON line (PATTERNS.md Intermediate line struct, RESEARCH.md Pitfall 7). Fields: Model string json:"model", CreatedAt string json:"created_at", Message ollamaChatResponseMessage json:"message", Done bool json:"done". This struct is for done:false frames only.

2. ndjsonGenerateLine struct — intermediate /api/generate NDJSON line. Fields: Model string json:"model", CreatedAt string json:"created_at", Response string json:"response", Done bool json:"done".

3. emitNDJSONChunk(w http.ResponseWriter, flusher http.Flusher, c canonical.Chunk, model string, isChat bool, cancelFn context.CancelFunc) error — emits one done:false line for the chunk. For ChunkKindText: populate Message.Content or Response. For ChunkKindThought AND isChat==true: populate Message.Thinking (D-04 CONTEXT.md). For ChunkKindThought AND isChat==false: drop silently (D-04 generate path). Unknown kinds: drop silently with no state change. On json.Marshal error or fmt.Fprintf error: call cancelFn() (D-07) then return the wrapped error. After each successful write: flusher.Flush(). The write sequence is: json.Marshal → fmt.Fprintf(w, "%s\n", body) → flusher.Flush(). Note: fmt.Fprintf for NDJSON uses "%s\n" (single newline), NOT "data: %s\n\n" which is SSE format.

4. runNDJSONEmitter(ctx context.Context, cancelFn context.CancelFunc, w http.ResponseWriter, run RunHandle, model string, isChat bool, start time.Time, logger *slog.Logger) error — the main emitter function (PATTERNS.md select-loop + RESEARCH.md Pattern 1):
   a. Assert http.Flusher BEFORE any write — if not Flusher, return errors.New("ollama: response writer is not flusher") WITHOUT writing headers. Caller (handlers.go) checks for this before headers are sent and calls writeError.
   b. Set w.Header().Set("Content-Type", "application/x-ndjson") and w.Header().Set("Cache-Control", "no-cache"). Then w.WriteHeader(http.StatusOK). Order is critical (Pitfall 2).
   c. Core select-loop over run.Stream().Chunks():
      - case <-ctx.Done(): logger.Debug with "ollama: ndjson client disconnect" and "session_id" attr. Return fmt.Errorf("ollama: ndjson ctx: %w", ctx.Err()). Do NOT call cancelFn or Cancel here — the watchdog (D-06) handles Cancel.
      - case c, ok := <-chunks: if !ok { return finalizeNDJSON(w, flusher, run, model, isChat, start, logger) }; if emitErr := emitNDJSONChunk(...); emitErr != nil { return emitErr } (cancelFn already called inside emitNDJSONChunk on write error per D-07).
   d. The function is the SOLE goroutine that touches w and flusher (single-goroutine invariant, RESEARCH.md Pitfall 8).

5. finalizeNDJSON(w http.ResponseWriter, flusher http.Flusher, run RunHandle, model string, isChat bool, start time.Time, logger *slog.Logger) error:
   a. Call run.Stream().Result(). On error: logger.Debug + return wrapped error (D-05 truncate — no done:true line written).
   b. Call run.StopWatchdog() and invoke the returned stop func to prevent the watchdog goroutine from firing on normal completion (Pitfall 3).
   c. Build the final done:true line. For isChat: call chatResponseToWire(nil, start, model) then set out.Done = true, out.DoneReason = mapStopReason(final.StopReason) if final != nil. For generate: call generateResponseToWire(nil, start, model) similarly. Marshal and write with fmt.Fprintf(w, "%s\n", body), flusher.Flush().

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP response body → client | NDJSON lines written to ResponseWriter; cannot be recalled once flushed |
| kiro-cli stdout → canonical.Chunk → ndjson.go | Chunk content from kiro-cli becomes JSON in the NDJSON line |
| write-error path | Broken pipe from client triggers cancelFn() which propagates to the engine watchdog |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-04 | Information Disclosure | Mid-stream error: engine/ACP error text in client response | mitigate | D-05 in CONTEXT.md: finalizeNDJSON debug-logs terminal errors and stops emission — no error JSON line sent to client (matches openai/sse.go finalizeSSE pattern) |
| T-04-05 | Tampering | NDJSON line injection via chunk text containing literal newline | accept | encoding/json.Marshal escapes \n and \r inside strings; the "%s\n" format only adds one trailing newline after the marshaled object; no raw string bypass possible |
| T-04-06 | Denial of Service | Client flooding with streaming requests that disconnect immediately | accept | context.WithCancel + watchdog ensures kiro-cli is notified; each disconnect costs at most one goroutine (AfterFunc) that exits quickly; goleak gate validates no accumulation |
| T-04-07 | Denial of Service | Pool-of-1 slot deadlock if slot not released after cancel | mitigate | Chat_DisconnectSmoke E2E asserts the slot is reusable after a disconnect; full slot-release-on-cancel hardens in Phase 5 |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new external packages this plan — all stdlib + existing go.mod dependencies |
</threat_model>
