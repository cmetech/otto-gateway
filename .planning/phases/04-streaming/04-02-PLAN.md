```go
func runSSEEmitter(ctx context.Context, w http.ResponseWriter, run RunHandle, model string, logger *slog.Logger) error {
    flusher, ok := w.(http.Flusher)
    if !ok { return errors.New("openai: response writer is not flusher") }
    w.Header().Set("Content-Type", "text/event-stream")
    w.Header().Set("Cache-Control", "no-cache")
    w.Header().Set("Connection", "keep-alive")
    w.WriteHeader(http.StatusOK)
    chunks := run.Stream().Chunks()
    for {
        select {
        case <-ctx.Done():
            logger.Debug("openai: sse client disconnect", "session_id", run.SessionID())
            return fmt.Errorf("openai: sse ctx: %w", ctx.Err())
        case c, ok := <-chunks:
            if !ok { return finalizeSSE(e, run) }
            if err := e.applyChunk(c); err != nil { return err }
        }
    }
}
```
For NDJSON: Content-Type is "application/x-ndjson"; no Connection header; finalizeNDJSON replaces finalizeSSE; emitNDJSONChunk replaces applyChunk; cancelFn is an additional parameter (D-07).

Create internal/adapter/ollama/ndjson.go (package ollama):
The file implements four package-level pieces:

1. ndjsonChatLine struct — intermediate /api/chat line (done:false frames only, per RESEARCH.md Pitfall 7):
   Fields: Model string json:"model", CreatedAt string json:"created_at", Message ollamaChatResponseMessage json:"message", Done bool json:"done".

2. ndjsonGenerateLine struct — intermediate /api/generate line:
   Fields: Model string json:"model", CreatedAt string json:"created_at", Response string json:"response", Done bool json:"done".

3. emitNDJSONChunk(w http.ResponseWriter, flusher http.Flusher, c canonical.Chunk, model string, isChat bool, cancelFn context.CancelFunc) error:
   - For canonical.ChunkKindText: build ndjsonChatLine (isChat) or ndjsonGenerateLine (!isChat) with done:false, marshal to JSON, write `fmt.Fprintf(w, "%s\n", body)`, flusher.Flush().
   - For canonical.ChunkKindThought AND isChat==true: build ndjsonChatLine with Message.Thinking set (D-04); marshal and write.
   - For canonical.ChunkKindThought AND isChat==false: drop silently (D-04 — /api/generate has no thinking field).
   - For unknown chunk kinds: drop silently.
   - On json.Marshal error or fmt.Fprintf error: call cancelFn() (D-07 — adapter signals write failure; watchdog observes derived ctx cancel and fires Cancel(sid)); then return wrapped error.
   - Write sequence: json.Marshal → fmt.Fprintf(w, "%s\n", body) → flusher.Flush(). NDJSON uses single newline, NOT "data: %s\n\n" (that is SSE format).

4. runNDJSONEmitter(ctx context.Context, cancelFn context.CancelFunc, w http.ResponseWriter, run RunHandle, model string, isChat bool, start time.Time, logger *slog.Logger) error:
   a. Assert http.Flusher BEFORE any write — if !ok, return errors.New("ollama: response writer is not flusher") WITHOUT writing headers. Caller checks this error before headers are sent.
   b. Set w.Header().Set("Content-Type", "application/x-ndjson") and w.Header().Set("Cache-Control", "no-cache"). Then w.WriteHeader(http.StatusOK). Order critical (Pitfall 2).
   c. Core select-loop over run.Stream().Chunks():
      - case <-ctx.Done(): logger.Debug "ollama: ndjson client disconnect" with "session_id" attr. Return fmt.Errorf("ollama: ndjson ctx: %w", ctx.Err()). Do NOT call cancelFn or Cancel — the watchdog (D-06) handles Cancel.
      - case c, ok := <-chunks: if !ok { return finalizeNDJSON(w, flusher, run, model, isChat, start, logger) }; if err := emitNDJSONChunk(w, flusher, c, model, isChat, cancelFn); err != nil { return err } (cancelFn already called inside emitNDJSONChunk on write error).
   d. The function is the sole goroutine touching w and flusher (single-goroutine invariant — Pitfall 8).

5. finalizeNDJSON(w http.ResponseWriter, flusher http.Flusher, run RunHandle, model string, isChat bool, start time.Time, logger *slog.Logger) error:
   a. Call run.Stream().Result(). On error: logger.Debug + return wrapped error (D-05 truncate — no done:true line written on error).
   b. Call run.StopWatchdog() and invoke stop func — prevents watchdog goroutine from firing on normal completion (Pitfall 3):
      `if stop := run.StopWatchdog(); stop != nil { stop() }`
      Inline comment: "D-06 teardown: prevent watchdog from emitting spurious Cancel after natural stream completion."
   c. Build final done:true line. BEFORE calling chatResponseToWire/generateResponseToWire with a nil first argument, CONFIRM in render.go that these helpers nil-guard their `*canonical.ChatResponse` parameter (do not dereference it unconditionally). If they DO nil-guard: call chatResponseToWire(nil, start, model) (isChat) or generateResponseToWire(nil, start, model) (!isChat), set Out.Done = true, set Out.DoneReason = mapStopReason(final.StopReason) if final != nil. If they DO NOT nil-guard (would panic on nil): instead build a zero-value &canonical.ChatResponse{Model: model} (and use the emitter's accumulated text if the helper needs message content) and pass that pointer. Either way, never pass an argument the helper will nil-dereference. Marshal and write fmt.Fprintf(w, "%s\n", body), flusher.Flush().

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP response body → client | NDJSON lines written to ResponseWriter; cannot be recalled once flushed |
| kiro-cli stdout → canonical.Chunk → ndjson.go | Chunk content from kiro-cli becomes JSON in the NDJSON line |
| write-error path (D-07) | Broken pipe from client triggers cancelFn() from emitNDJSONChunk, which propagates via derived ctx to the engine watchdog |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-06 | Information Disclosure | Mid-stream error: engine/ACP error text in client response | mitigate | D-05: finalizeNDJSON debug-logs terminal errors and stops emission — no error JSON line sent to client (matches openai/sse.go finalizeSSE truncation pattern); Anthropic is explicitly exempt from D-05 (its spec mandates event:error) |
| T-04-07 | Tampering | NDJSON line injection via chunk text containing literal newline | accept | encoding/json.Marshal escapes \n and \r inside strings; no raw string write path |
| T-04-08 | Denial of Service | Client flooding with streaming requests that disconnect immediately | accept | context.WithCancel + watchdog ensures kiro-cli is notified; each disconnect costs at most one AfterFunc goroutine that exits quickly; goleak gate validates no accumulation |
| T-04-09 | Denial of Service | Pool-of-1 slot deadlock after cancel | mitigate | Chat_DisconnectSmoke E2E asserts slot reusable after disconnect (health alive count unchanged + follow-up request succeeds); full slot-release-on-cancel hardens in Phase 5 |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new external packages — all stdlib + existing go.mod |
</threat_model>
