RESOLVED: Default-streaming semantics for OpenAI and Anthropic (Codex HIGH concern):
- OpenAI: wire.Stream is `bool json:"stream,omitempty"`. Absent field → false → non-streaming. This is CORRECT — OpenAI's public spec says stream defaults to false when omitted. SC2's "defaults to streaming" language refers to the intended dominant use case (Pi-SDK always sends explicit stream:true). The bool-is-safe because Pi-SDK never omits the field. No *bool change needed for OpenAI.
- Anthropic: wire.Stream is `bool json:"stream,omitempty"`. Absent field → false → non-streaming. This matches the Anthropic public spec exactly: "stream defaults to false on messages.create." No change needed or desired for Anthropic.

RESOLVED: Anthropic mid-stream error vs D-05 (Codex MEDIUM concern):
Anthropic's finalizeStream (internal/adapter/anthropic/sse.go) emits `event: error` on Result() error. This is NOT a violation of D-05. D-05 applies only to surfaces with no spec-mandated error frame. Anthropic's spec explicitly documents `event: error` as the terminal frame on stream errors; the @anthropic-ai/sdk expects it. Anthropic is explicitly exempt from D-05. This exemption is documented in the Plan 01 action comment for the anthropic/sse.go teardown addition.
