**Analog (composite — two proven sites):**

2. Per-surface non-streaming sibling for `writeError` signature + status code:
   - Ollama: `handlers.go:147-150` → `writeError(w, http.StatusUnauthorized, shortCircuitMessage(resp))` (flat envelope, no `errorType`)

**Template — Anthropic non-streaming source (copy this shape):**

```go
// Source: internal/adapter/anthropic/collect.go:66-73
// Phase 8 SC1 short-circuit: if a PreHook (e.g., AuthHook) returned
// a synthesized response, return it verbatim. The chunk-based
// aggregator below would otherwise see an empty stream and drop
// the hook's user-facing message — handleMessages then detects
// StopReason == canonical.StopError and renders the Anthropic
// error envelope.
if shortCircuit := run.ShortCircuitResponse(); shortCircuit != nil {
    // Stop the watchdog for parity (it was nil on a short-circuit
    // Run, but the call is nil-safe).
    if stop := run.StopWatchdog(); stop != nil {
        stop()
    }
    return shortCircuit, nil
}
```

**Adapted for streaming — Ollama (`handleChat` + `handleGenerate`):**

```go
// Source: pattern from collect.go:66-73; status from handlers.go:147-150.
// Phase 08.1 INTEG-01: PreHook short-circuit must be caught BEFORE
// runNDJSONEmitter opens NDJSON headers, otherwise the bad-bearer
// case emits a benign empty stream instead of 401. Mirrors the
// non-streaming short-circuit branch at handlers.go:147-150.
if sc := run.ShortCircuitResponse(); sc != nil {
    // Watchdog is nil on the short-circuit path (engine.go:150);
    // guard the deref. Mirrors collect.go:69-71.
    if stop := run.StopWatchdog(); stop != nil {
        stop()
    }
    writeError(w, http.StatusUnauthorized, shortCircuitMessage(sc))
    return
}
```

**Adapted for streaming — OpenAI (`handleChatCompletions`):**

```go
// Phase 08.1 INTEG-01: PreHook short-circuit before runSSEEmitter
// opens SSE headers. Mirrors non-streaming branch at handlers.go:165-168.
if sc := runHandle.ShortCircuitResponse(); sc != nil {
    if stop := runHandle.StopWatchdog(); stop != nil {
        stop()
    }
    writeError(w, http.StatusUnauthorized, errAuthentication, shortCircuitMessage(sc))
    return
}
```

**Adapted for streaming — Anthropic (`handleMessages`):**

```go
// Phase 08.1 INTEG-01: PreHook short-circuit before runSSEEmitter
// opens SSE headers. Mirrors non-streaming branch at handlers.go:220-223
// and collect.go:66-73 (same surface, non-streaming path).
if sc := runHandle.ShortCircuitResponse(); sc != nil {
    if stop := runHandle.StopWatchdog(); stop != nil {
        stop()
    }
    writeError(w, http.StatusUnauthorized, errAuthentication, shortCircuitMessage(sc))
```

**Critical invariants (verbatim from non-streaming siblings — do NOT drift):**
1. **Variable name** matches the local in scope — `run` for Ollama (`handlers.go:188`), `runHandle` for OpenAI and Anthropic. Mismatch = compile fail.
2. **`writeError` signature differs per surface** (Pitfall 5 — bug-prone). Ollama is 3-arg flat envelope; OpenAI + Anthropic are 4-arg with `errAuthentication`.
3. **Hard-coded `http.StatusUnauthorized`** — D-04, no derivation.
4. **`StopWatchdog` nil-guard mandatory** — D-03, Pitfall 4. `stop != nil { stop() }` is the canonical guard from `collect.go:69-71`.
5. **`return` immediately after `writeError`** — no fall-through to the emitter.

## Anti-Pattern Reminders (verbatim from RESEARCH.md)

Future contributors reading PLAN files should be reminded these are forbidden:

- **Do NOT check `resp.StopReason == canonical.StopError` after a chunk-loop drain on streaming paths.** The chunk channel is empty (closed) on short-circuit per `engine.go:231-239`; draining it just to read `FinalResult.StopReason` races the watchdog and emits headers prematurely. Use `ShortCircuitResponse()` BEFORE any header write.
- **Do NOT push the check into the emitter.** `runSSEEmitter` / `runNDJSONEmitter` runs AFTER the handler decides to open the stream. The check must precede that decision.
- **Do NOT make `eng.Run` return an error on short-circuit.** That would change the engine contract and break Anthropic's existing `CollectAnthropicChat`-via-`ShortCircuitResponse` pattern at `collect.go:66`.
- **Do NOT add a `StatusCode int` field to `canonical.ChatResponse`.** Speculative (no non-401 producer exists in v1); violates Phase 2 D-11 "canonical types stay narrow."
- **Do NOT refactor Anthropic's non-streaming path to use `StopReason == StopError`** to "harmonize" with Ollama/OpenAI. Two patterns coexist intentionally — that's the load-bearing decision D-06 documents in this file.
- **Do NOT collapse `08.1-PATTERNS.md` into a sentence in some other doc.** Future hook authors need to find this decision at the call site they'd modify.
