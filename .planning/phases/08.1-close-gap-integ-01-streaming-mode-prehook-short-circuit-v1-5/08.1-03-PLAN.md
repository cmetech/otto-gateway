<objective>
Close WARNING-01 from the v1.5 audit by making the brief §3.12 canonical trust-gate sequence explicit in both the `Makefile` and `.github/workflows/ci.yml` per D-16. Currently the sequence is `gofumpt -d → go vet → go build → golangci-lint run → govulncheck → go test -race ./... → go test -run Example ./... → property tests → cross-compile`, but `Makefile` is missing explicit `fmt-check` (the read-only gofumpt variant), `vet`, `build`, and `examples` targets, and `.github/workflows/ci.yml` covers vet implicitly via golangci-lint's `govet` linter without an explicit step that future contributors can read at a glance.

The fix is documentation-of-existing-discipline rather than a capability gap: golangci-lint's `govet` linter and `go test ./...` already implicitly cover the missing pieces. The audit finding is that the brief's explicit sequence isn't visible in the canonical files — fixing the visibility prevents future audits from re-flagging the same concern.

Purpose: Restore brief §3.12 visibility in both local-dev (Makefile) and CI (workflow) so future contributors don't re-question why steps are missing. Aligns dev-side and CI-side behavior; reduces drift risk per RESEARCH §"Ask 5" delegation recommendation.

Output: `make ci` runs the full canonical sequence end-to-end and exits 0 on a clean checkout; CI workflow has explicit gofumpt-check, vet, build, and Example-test steps logged in canonical order. No code-side changes; no test-side changes; no behavior change for any client.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI runner → source tree | GitHub Actions runs the workflow against the checked-out code. Trust-gate sequence is what gates merges. Visibility of the sequence to future contributors is the audit finding being closed. |
| Local dev shell → Makefile | Developer runs `make ci` before pushing. Same sequence; CI is the second line of defense, not the first. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08.1-CI-DRIFT | Tampering (silent erosion of trust-gate coverage) | Makefile + .github/workflows/ci.yml | mitigate | Per D-16 + brief §3.12: explicit canonical sequence visible in both files. Future contributors who'd otherwise re-question why steps are missing now see the explicit list and the brief §3.12 reference. Delegation pattern (CI → make) prevents the two from drifting. |
| T-08.1-FALSE-GREEN | Tampering (linter passes mask runtime issues) | golangci-lint's govet sub-linter vs an explicit `go vet ./...` | accept | The brief §3.12 sequence calls for an explicit go vet step; golangci-lint's govet covers the same ground but a separate step is clearer in CI logs and matches the brief. No functional change. |
| T-08.1-FMTCHECK-FALLBACK | Tampering | gofumpt unavailable on CI runner | mitigate | The Makefile fmt-check target falls back to `gofmt -d`; the CI workflow installs gofumpt explicitly via `go install` in a step before the format check. Failure surfaces as an "Install gofumpt" step failure, not as a silent skip. |
