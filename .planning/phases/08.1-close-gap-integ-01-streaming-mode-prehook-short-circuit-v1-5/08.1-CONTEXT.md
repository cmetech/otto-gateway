**Net-new work in Phase 08.1:**

1. **INTEG-01 fix (BLOCKER).** The streaming branch in each of the three
   surface adapters writes SSE/NDJSON headers + 200 OK **before** checking
   whether the PreHook chain short-circuited. When `AuthHook` rejects a bad
   bearer on a `stream:true` request, the engine returns a `RunHandle` whose
   chunks channel is empty and whose `FinalResult.StopReason == StopError`;
   the emitter sees zero chunks, maps `StopError` through
   `mapStopReason`/`mapFinishReason` which lack a `StopError` case, and
   falls through to `"stop"` (Ollama, OpenAI) or `nil` (Anthropic). Client
   receives a benign-looking empty 200 stream. **No 401 path currently
   exists on the streaming surface.** Closes the streaming side of
   PLUG-02, PLUG-04, STRM-01, STRM-02, ANTH-01, ANTH-02.

   `ShortCircuitResponse() *canonical.ChatResponse` (adapter.go:72) and uses

   `openai.RunHandle` lack the method entirely. Phase 08.1 adds it to both
   so all three streaming branches share one pattern.

3. **WARNING-01 (CI trust-gate sequence).** `.github/workflows/ci.yml` and
   the Makefile target omit explicit `go vet` + `go build` steps. They are

## Implementation Decisions

### Streaming short-circuit response shape

- **D-01: Pre-header 401 + native JSON envelope on all three surfaces.**
  When `runHandle.ShortCircuitResponse()` returns non-nil, the streaming
  handler writes `writeError(w, http.StatusUnauthorized, ...,
  shortCircuitMessage(sc))` BEFORE any SSE/NDJSON headers are written and
  returns. Symmetric across all three surfaces; identical to the
  non-streaming short-circuit branch (`handlers.go:147` Ollama,
  `handlers.go:165` OpenAI, `handlers.go:220` Anthropic). Reuses each
  adapter's existing `shortCircuitMessage()` helper. Pi-SDK receives a
  real 401; @anthropic-ai/sdk receives the canonical Anthropic error
  envelope on its non-stream HTTP error path; LangFlow receives a
  JSON error.

- **D-02: Check point â€” after `eng.Run`, before
  `runSSEEmitter`/`runNDJSONEmitter`.** Inside each adapter's streaming
  branch:
  ```go
  runHandle, err := eng.Run(streamCtx, req)
  if err != nil { /* existing 500 path */ }
  if sc := runHandle.ShortCircuitResponse(); sc != nil {
      runHandle.StopWatchdog()()              // D-03
      writeError(w, http.StatusUnauthorized, ..., shortCircuitMessage(sc))
      return
  }
  if emitErr := runSSEEmitter(...); ...
  ```
  One concentrated branch per handler. Mirrors the non-streaming
  `if resp != nil && resp.StopReason == canonical.StopError { ... }`
  check site. NOT pushed into the emitter (the emitter is stream-only).
  NOT signaled via engine.Run error (that would change the engine
  contract and break Anthropic's existing
  `CollectAnthropicChat`-via-ShortCircuitResponse pattern).

- **D-03: Call `runHandle.StopWatchdog()()` in the short-circuit branch.**
  Engine.Run spawned the D-06 disconnect watchdog via
  `context.AfterFunc(streamCtx, ...)` even on the short-circuit path
  (`SessionID()` returns empty). The `defer cancelFn()` already tears
  the watchdog down, but explicitly calling `StopWatchdog()` makes the
  "ACP was never engaged" contract visible at the call site and avoids
  a benign `Cancel(emptySID)` debug-log line. Symmetric with normal
  stream completion in the emitter (which already calls
  `StopWatchdog()` per Phase 4 D-06 / `04-CONTEXT.md` Pattern 2).

- **D-04: Hard-code `http.StatusUnauthorized`. No status derivation.**
  AuthHook is the only v1 PreHook short-circuit producer; it always
  means 401. The non-streaming branches in all three adapters already
  hard-code 401 as a literal. Staying symmetric keeps the two paths
  identical. When a non-401-producing hook lands (rate-limit â†’ 429,
  content-mod â†’ 403), revisit streaming + non-streaming together;
  adding a `StatusCode int` field to `canonical.ChatResponse` now
  would be speculative and violates the Phase 2 D-11 "canonical types
  stay narrow" invariant.

### Adapter symmetry for ShortCircuitResponse()

- **D-05: Add `ShortCircuitResponse() *canonical.ChatResponse` to
  `ollama.RunHandle` and `openai.RunHandle` interface declarations.**
  Mirrors `anthropic.RunHandle` (`adapter.go:54-72`). `*engine.Run`
  already implements this method (Anthropic proves it structurally);
  the change is the two interface lines + the structural-typing shim
  wiring (D-07).

- **D-06: Two discriminator patterns coexist intentionally.**
  - **Non-streaming path** (Ollama, OpenAI): check
    `resp.StopReason == canonical.StopError` on the value returned
    by `eng.Collect`. Cheap â€” the response is already in hand.
  - **Non-streaming path** (Anthropic): call
    `runHandle.ShortCircuitResponse()` inside
    `CollectAnthropicChat` (`collect.go:66`). This idiom predates
    Phase 08.1 and stays.
  - **Streaming path** (all three): call
    `runHandle.ShortCircuitResponse()`. The chunks channel is
    empty on short-circuit; draining it just to read
    `FinalResult.StopReason` is wasteful and races the watchdog.
  - **Don't collapse to one pattern in 08.1.** The dual pattern is
    documented in `08.1-PATTERNS.md` (new) and in each adapter.go
    doc comment on `ShortCircuitResponse()`. Future hook authors
    pick by code path, not by surface.

- **D-07: Forward `ShortCircuitResponse()` through main.go's shim
  adapters.** `cmd/otto-gateway/main.go` wraps `*engine.Run` with
  per-surface adapter types (`ollamaRunHandleAdapter`, analogous
  OpenAI one if present). Add one method per shim:
  ```go
  func (a ollamaRunHandleAdapter) ShortCircuitResponse() *canonical.ChatResponse {
      return a.run.ShortCircuitResponse()
  }
  ```
  Mirrors Anthropic's `realRunHandle.ShortCircuitResponse` in
  `internal/adapter/anthropic/integration_test.go:142`. Researcher
  verifies whether OpenAI uses a similar shim or passes
  `*engine.Run` directly; either way the wiring is trivial.

- **D-08: Add stub `func (...) ShortCircuitResponse() *canonical.ChatResponse { return nil }`
  to every existing `fakeRunHandle` / mock in ollama + openai test
  files.** Mirrors `anthropic/handlers_test.go:161`. Forces the
  contract to be visible at every test site; compiler-enforced
  correctness across `tests/e2e/`, `internal/adapter/{ollama,openai}/
  *_test.go`. Researcher to enumerate the exact files needing the
  stub (likely ndjson_test.go, handlers_test.go, sse_test.go per
  surface).

### E2E test packaging for streaming bad-bearer

- **D-09: Extend `TestE2E_BadBearer_AllThreeSurfaces` in-place with

     - OpenAI: `{error:{type:"...", message:"...", ...}}`.
     - Ollama: `{error:"..."}`.

       message:"..."}}`.
  4. Body contains zero SSE/NDJSON markers â€” no `data:`, no
     `event:`, no NDJSON line with `done:true`. Strongest catch
     for header-leak regressions.
