### Pattern 1: Streaming short-circuit check (CANONICAL template)

**What:** A 4-line guard between `eng.Run(...)` and the emitter call. One per streaming handler.
**When to use:** Any future PreHook short-circuit producer (rate-limit, content-mod, semantic-cache). Same site, same shape.

**Example (Ollama handleChat — extracted to template form):**
```go
// Source: existing non-streaming pattern at internal/adapter/ollama/handlers.go:147-150;
// adapt to the streaming branch starting at line 185.
streamCtx, cancelFn := context.WithCancel(ctx)
defer cancelFn()

run, err := eng.Run(streamCtx, req)
if err != nil {
    writeError(w, http.StatusInternalServerError, err.Error())
    return
}

// INTEG-01 fix — Phase 08.1 D-01..D-04.
if sc := run.ShortCircuitResponse(); sc != nil {
    // D-03: stop the watchdog. nil-safe per engine.go:150 — when a PreHook
    // short-circuited Run, stopWatchdog is nil because no ACP session opened.
    if stop := run.StopWatchdog(); stop != nil {
        stop()
    }
    writeError(w, http.StatusUnauthorized, shortCircuitMessage(sc))
    return
}

start := time.Now()
if emitErr := runNDJSONEmitter(streamCtx, cancelFn, w, run, wire.Model, true, start, a.cfg.Logger, req); emitErr != nil {
    a.cfg.Logger.Debug("ollama: ndjson chat emitter error", "err", emitErr)
}
```

### Pattern 2: Dual-discriminator decision table (08.1-PATTERNS.md content)

| Path | Discriminator | Why |
|------|---------------|-----|
| Non-streaming Ollama/OpenAI (`handleChat`, `handleGenerate`, `handleChatCompletions`, `handleCompletions`) | `resp.StopReason == canonical.StopError` on `eng.Collect`-returned `*canonical.ChatResponse` | Response is in hand; cheap; matches the engine contract that PreHook short-circuit populates StopReason |
| Non-streaming Anthropic (`handleMessages` stream:false) | `runHandle.ShortCircuitResponse()` inside `CollectAnthropicChat` (collect.go:66) | Anthropic routes via `eng.Run` to enable tool_use aggregation (Phase 6 D-07 exception); the chunk-based aggregator would drop the short-circuit payload otherwise |
| Streaming (all three) | `runHandle.ShortCircuitResponse()` on the `eng.Run`-returned handle, BEFORE opening emitter | Chunks channel is empty (closed) on short-circuit per `engine.go:231-239` `newCompletedRun` → `emptyStream`; draining it just to read `FinalResult.StopReason` races the watchdog and emits headers prematurely |

**Pattern rule:** Future hook authors pick discriminator by **code path**, not by **surface**.

### Anti-Patterns to Avoid

- **Don't check `resp.StopReason == canonical.StopError` after a chunk-loop drain on streaming paths.** By the time you've drained the empty chunk channel, you've already (a) opened SSE/NDJSON headers, (b) raced the D-06 watchdog goroutine that fires `Cancel(emptySID)`. Use `ShortCircuitResponse()` BEFORE any header write.
- **Don't push the check into the emitter.** The emitter is stream-only and is invoked AFTER the handler decides to open the stream. The check must precede that decision.
- **Don't make `eng.Run` return an error on short-circuit.** That would change the engine contract and break Anthropic's existing `CollectAnthropicChat`-via-`ShortCircuitResponse` pattern (collect.go:66).
- **Don't add a `StatusCode int` field to `canonical.ChatResponse` to derive the HTTP status.** Speculative (no non-401 producer exists in v1); violates Phase 2 D-11 "canonical types stay narrow."
- **Don't refactor Anthropic's non-streaming path to use `StopReason == StopError`** to "harmonize." Two patterns coexist intentionally; documenting them is the right move.
- **Don't collapse `08.1-PATTERNS.md` into a sentence in some other doc.** Future hook authors need to find this decision at the call site they'd modify, not buried in Phase 8's CONTEXT.md.

## Common Pitfalls

### Pitfall 1: Forgetting to forward `ShortCircuitResponse()` through main.go shims
**What goes wrong:** Adding the method to `ollama.RunHandle` and `openai.RunHandle` interfaces breaks the build at `cmd/otto-gateway/main.go` because `ollamaRunHandleAdapter` (line 928) and `openaiRunHandleAdapter` (line 947) no longer satisfy the interface.
**Why it happens:** Go's structural typing only helps if every member is present.
**How to avoid:** Add the forwarder in the same atomic commit as the interface change. Template (`main.go` already has it for anthropic at lines 861-865):
```go
func (h ollamaRunHandleAdapter) ShortCircuitResponse() *canonical.ChatResponse {
    return h.run.ShortCircuitResponse()
}
func (h openaiRunHandleAdapter) ShortCircuitResponse() *canonical.ChatResponse {
    return h.run.ShortCircuitResponse()
}
```
**Warning signs:** `cannot use … as ollama.RunHandle: missing method ShortCircuitResponse` from `go build ./...`.

### Pitfall 2: Forgetting test-fake propagation (D-08)
**What goes wrong:** Compilation failures in `*_test.go` files because `fakeRunHandle` no longer satisfies the package-private `RunHandle` interface.
**Why it happens:** Whitebox tests sit inside the same package; the interface change ripples to every fake.
**How to avoid:** Mechanical pass — add the stub `func (f *fakeRunHandle) ShortCircuitResponse() *canonical.ChatResponse { return nil }` to every site listed in the "Existing fakeRunHandle / mock enumeration" table below.
**Warning signs:** Package-level test build failure across multiple files. `go test ./internal/adapter/ollama/... ./internal/adapter/openai/...` is the first signal.

### Pitfall 3: Header-leak regression (writing headers before checking)
**What goes wrong:** A future contributor moves the check after `w.Header().Set("Content-Type", "text/event-stream")` — short-circuit then produces an SSE-shaped 401 instead of a JSON-shaped 401, breaking `@anthropic-ai/sdk` HTTP-error handling.
**Why it happens:** SSE/NDJSON header setting is currently encapsulated in `runSSEEmitter`/`runNDJSONEmitter`, but a refactor might inline that for "performance" or "clarity."
**How to avoid:** D-10 E2E rows specifically assert (a) `Content-Type: application/json` (NOT `text/event-stream` or `application/x-ndjson`), (b) zero `data:`, `event:`, or NDJSON `done:true` byte markers in the response body. The assertions are the regression gate.
**Warning signs:** D-10 E2E assertions fail with `Content-Type: text/event-stream` or body containing `data: ` line markers.

### Pitfall 4: Watchdog double-stop or stop-on-nil
**What goes wrong:** `run.StopWatchdog()()` panics with nil dereference when `stopWatchdog` is nil (the short-circuit Run case per `engine.go:150`).
**Why it happens:** The Anthropic `CollectAnthropicChat` already handles this with `if stop := run.StopWatchdog(); stop != nil { stop() }` (collect.go:69-71). Forget to guard, and Slice A's streaming branches will panic on the short-circuit path.
**How to avoid:** Always guard:
```go
if stop := runHandle.StopWatchdog(); stop != nil {
    stop()
}
```
**Warning signs:** Test panic with `nil pointer dereference` inside the new short-circuit branch.

### Pitfall 5: Wrong errorType constant on OpenAI / Anthropic surfaces
**What goes wrong:** Using `errAPI` or `errInvalidRequest` instead of `errAuthentication` produces a 401 with the wrong error envelope.type — `@anthropic-ai/sdk` and Pi-SDK both inspect `error.type` for retry logic.
**Why it happens:** `writeError` signatures differ between adapters:
  - Ollama: `writeError(w, status, msg)` (no type — flat envelope)
  - OpenAI: `writeError(w, status, errorType, message)` (nested envelope, `errAuthentication = "authentication_error"`)
  - Anthropic: `writeError(w, status, errorType, message)` (nested envelope, `errAuthentication = "authentication_error"`)

The non-streaming paths already get this right; the streaming branches must match.
**How to avoid:** Copy the non-streaming line verbatim to the streaming branch. Per-surface call:
  - Ollama: `writeError(w, http.StatusUnauthorized, shortCircuitMessage(sc))`
  - OpenAI: `writeError(w, http.StatusUnauthorized, errAuthentication, shortCircuitMessage(sc))`
  - Anthropic: `writeError(w, http.StatusUnauthorized, errAuthentication, shortCircuitMessage(sc))`
**Warning signs:** D-10 E2E rows fail Anthropic's `env.Error.Type` assertion or OpenAI's `env.Error.Type` assertion.
