## Streaming short-circuit response shape

### Q1: When PreHook short-circuits a stream:true request, where should the 401 be rendered?

| Option | Description | Selected |
|--------|-------------|----------|
| Pre-header 401+JSON | Check ShortCircuitResponse() BEFORE writing SSE/NDJSON headers; render same 401 + native envelope as non-streaming. Symmetric across all 3 surfaces, matches audit prescription, reuses shortCircuitMessage(). | ✓ |
| Anthropic SSE event-error; OpenAI/Ollama 401+JSON | Anthropic emits `event: error` after headers; OpenAI/Ollama use pre-header 401 (no SSE error idiom in their specs). Asymmetric but native to each surface. | |

**Notes:** Symmetry across all three surfaces + reuse of existing shortCircuitMessage() helper. Aligns with the v1.5-MILESTONE-AUDIT.md remediation block (lines 71-74).
