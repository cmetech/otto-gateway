## How It Connects

- Engine-side: `*engine.Run.ShortCircuitResponse()` already exists from Phase 8 SC1 (no engine changes). Anthropic's `RunHandle` already declares the method.
- New seam: `ollama.RunHandle` and `openai.RunHandle` now declare `ShortCircuitResponse() *canonical.ChatResponse`. `*engine.Run` structurally satisfies all three.
- main.go wires `*engine.Run` to the three per-surface `RunHandle` interfaces via shim types (`ollamaRunHandleAdapter`, `openaiRunHandleAdapter`, `anthropicRunHandleAdapter`). All three shim types now forward `ShortCircuitResponse()` to `h.run.ShortCircuitResponse()`.
- Streaming-handler call chain: `eng.Run(streamCtx, req)` → err-guard → **NEW: `if sc := runHandle.ShortCircuitResponse(); sc != nil { ... return }`** → emitter call. The check precedes any SSE/NDJSON header write, so a bad-bearer streaming request returns `Content-Type: application/json` + 401 + native error envelope rather than an empty 200 stream.

## Decisions Made

- Honored plan's **atomic-commit directive**: Patterns A + C + D + B all landed in a single commit. Splitting Pattern A (interface) from Pattern C+D (consumer satisfaction) would have broken `go build ./...` between commits.
- Used hard-coded `http.StatusUnauthorized` per D-04 in every short-circuit block. The dual-discriminator pattern (D-06) plus the deferred non-401 status mapping decision are documented in 08.1-CONTEXT.md.
- Added per-surface attribution comments at each insertion site referencing "Phase 08.1 INTEG-01 D-01..D-04" plus the non-streaming sibling line range, per plan's read_first guidance.
- Did NOT touch `internal/adapter/anthropic/*` interfaces or shims — Anthropic's non-streaming path was already shipping `ShortCircuitResponse()` (Phase 8 SC1). The streaming branch in `handleMessages` was the only Anthropic-side addition.
