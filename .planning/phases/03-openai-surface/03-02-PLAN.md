<objective>
Deliver the thinnest end-to-end OpenAI slice and the load-bearing acceptance bar:
`POST /v1/chat/completions` working both `stream:true` (SSE — what Pi/SC2 drives)
and `stream:false` (JSON — what the SC1 curl drives), reachable over HTTP via the
RegisterRoutes seam from Plan 01 and exercised by an httptest round-trip.

This is a vertical slice: wire decode → engine call → render/SSE emit → reachable
over `r.Route("/v1", …)` in the integration test. After this plan, a real user (or
the integration test standing in for Pi) can POST a chat and receive a correct
streamed OpenAI response — the `/v1/models` and `/v1/completions` endpoints and the
cmd/main.go wiring are subsequent slices (Plans 03, 04).

The OpenAI SSE emitter is STRUCTURALLY SIMPLER than Anthropic's — copy the
single-goroutine writer + header/Flusher pattern but DELETE the block-index state
machine, the ping ticker, and the event:-named framing (Pitfalls 1/6).

Purpose: SC1 (stream:false JSON) and SC2 (Pi stream:true SSE) both pass at the
adapter level, byte-exact via golden fixtures and round-tripped via httptest.
Output: A complete chat-completions handler (stream + non-stream), the SSE emitter,
wire decode, render, error envelope, and the full unit/golden/integration test suite.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |

| T-03-11 | Info Disclosure | engine error path | mitigate | T-02-33: engine.Run/Collect errors logged raw via `slog.Error` then `writeError(500, errAPI, "internal error")` — generic message, never echo err.Error() (copy Anthropic, not Ollama). Asserted in acceptance criteria. |
| T-03-12 | DoS | SSE goroutine on client disconnect | mitigate | Single-goroutine select-loop returns on `ctx.Done`; no spawned writer goroutine, no ticker; `goleak.VerifyTestMain` proves leak-free after the streaming round-trip. |
| T-03-13 | Tampering | SSE header/stream framing | mitigate | Flusher asserted + Content-Type/Cache-Control set BEFORE WriteHeader(200) (Pitfall 2); mid-stream error truncates rather than injecting a JSON body into the stream (Pitfall 3 / A5). |
