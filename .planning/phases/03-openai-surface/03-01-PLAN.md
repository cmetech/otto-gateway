<objective>
Establish the Phase 3 reachability seam and Wave 0 test scaffolding so every
later OpenAI slice has a route to land on and a test gate to verify against.

This plan does the two genuinely-new pieces of structural work and nothing else:
1. The **D-01 SurfaceMount refactor** of `internal/server` — replace the parallel
   `Ollama*`/`Anthropic*` Config fields + two `r.Route`/`r.Mount("/",…)` blocks with
   a generic `Surfaces []SurfaceMount` list grouped by shared prefix, one
   auth-wrapped `r.Route(prefix,…)` per unique prefix. This is what lets OpenAI
   (`/v1`) co-mount with Anthropic (`/v1`) without chi's double-Mount panic.
2. The **config widening (D-05)** so `ENABLED_SURFACES` accepts/defaults to `openai`.
3. The **openai adapter skeleton** (`adapter.go` interfaces + `RegisterRoutes` + a
   verbatim `decode.go` + the goleak `testmain_test.go`) and the `.go-arch-lint.yml`
   boundary — so the package compiles, registers routes, and is boundary-checked
   before any handler logic exists.

Purpose: Make `/v1/chat/completions` reachable (the route exists and is
auth-wrapped) so Plan 02's SSE slice has somewhere to mount. Nothing here streams
yet — this is the foundation the vertical slices plug into.
Output: A compiling `internal/adapter/openai` package with no handlers yet, a
refactored multi-surface server, widened config, and the Wave 0 gates
(server_test SurfaceMount regression, testmain goleak gate, arch-lint entry,
config_test extension).
