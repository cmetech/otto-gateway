<objective>
Complete the OpenAI surface endpoint set: `GET /v1/models` (full, from the pool
catalog — D-04) and `POST /v1/completions` (minimal legacy shim — D-03). These are
the remaining SURF-04 endpoints beyond chat-completions.

This is a vertical slice on top of Plan 02: after this plan a user can list models
(`/v1/models` reflecting the same set as `/api/tags`, satisfying SC3) and hit the
legacy completions endpoint. It shares `handlers.go`/`render.go`/`wire.go` with Plan
02's chat slice, so it is sequenced after (Wave 3) to respect file ownership.

`/v1/models` reuses the Ollama ModelCatalog injection pattern (NOT a static list).
`/v1/completions` is intentionally minimal — accept-and-ignore advanced params,
JSON-only (silently downgrade stream:true), since Pi uses /chat/completions
exclusively and the kiro-cli backend can't honor logprobs/echo/best_of.

Purpose: SC3 (/v1/models and /api/tags reflect the same set; /v1/completions
returns a valid text_completion shape) passes.
Output: handleModels + handleCompletions handlers + their render shapes + unit tests.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → /v1/completions | Untrusted JSON body (polymorphic prompt, advanced params, arbitrary size) crosses here |
| client → /v1/models | Read-only GET; no body, but response must not leak internal model metadata beyond id/owned_by |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-20 | DoS | /v1/completions body | mitigate | `decodeJSONBody` + 4 MiB `MaxBytesReader` → 413 (V5, reused from Plans 01/02). |
| T-03-21 | Info Disclosure | engine error path | mitigate | T-02-33: engine.Collect errors slog'd raw + generic 500 message; never echo err.Error() (copy the Plan 02 chat handler path). |
| T-03-22 | Info Disclosure | /v1/models response | mitigate | modelInfo exposes only id/object/created/owned_by — no internal pool slot detail, no env, no paths. Sourced from the same catalog as /api/tags (no new disclosure surface). |
| T-03-23 | Tampering | advanced-param injection | accept | logprobs/echo/best_of/n decode-and-ignore; the kiro-cli backend never sees them. No behavior change, no risk; documented accept-and-ignore (D-03). |
