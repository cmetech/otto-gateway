### Task 01-02: `make build-fake-kiro` Makefile target (`c45a588`)

**`Makefile`:**

- `.PHONY` line: appended `build-fake-kiro` adjacent to `build` (read order is now `all build build-fake-kiro run ...`).
- New target inserted between `cross-windows-amd64` and the `Distribution packaging` comment block:

  ```makefile
  build-fake-kiro: ## Build deterministic fake-kiro-cli into bin/ for scripts/test-pii.* fake mode
      @mkdir -p $(BUILD_DIR)
      CGO_ENABLED=0 go build -o $(BUILD_DIR)/fake-kiro-cli$$(go env GOEXE) ./tests/e2e/cmd/fake-kiro-cli
  ```

- Preceding comment block documents the T4 mitigation: the target is NOT a dependency of `ci`, `all`, `build`, `package`, or `stage_unix`; operators run it explicitly.
- `make help` lists the new target via the existing `## description` convention.
