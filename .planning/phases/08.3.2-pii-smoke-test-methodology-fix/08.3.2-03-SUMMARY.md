## write_notifs_file body

```sh
write_notifs_file() {
    notifs_path="$1"; shift
    : > "$notifs_path"
    for v in "$@"; do
        printf '{"jsonrpc":"2.0","method":"session/update","params":{"sessionId":"e2e-session-1","update":{"sessionUpdate":"agent_message_chunk","content":{"text":"%s"}}}}\n' "$v" >> "$notifs_path"
    done
}
```

Key choices:

- **POSIX variadic via `shift` + `$@`**: no bash-only arrays. Caller signature: `write_notifs_file "$notifs_path" "$v1" "$v2" ...`.
- **Single-quoted `printf` format**: `"%s"` is literal in the format, so the only interpolation is `$v` in the argument list. This preserves all the JSON-RPC quoting characters as-is — no shell-double-quote escaping mishaps possible.
- **Truncate then append**: `: > "$notifs_path"` zeros the file before the loop appends frames. Avoids any leftover bytes from a prior invocation.
- **No BOM**: `printf` in `/bin/sh` produces clean UTF-8 bytes. POSIX equivalent of AP-1 mitigation.
- **No jq dependency**: values are ASCII fixtures (`corey@cmetech.io`, `192.168.1.42`) so the literal-format approach is safe. If a future fixture needs JSON escaping, the helper grows a jq branch.

## scenario_pii_fake precondition-check + write sequence

In order, the helper:

1. Computes `notifs_path="$TMPDIR_/notifications.ndjson"` — reuses the existing `mktemp -d` + `trap rm -rf` at line 73 (no manual cleanup).
2. Calls `write_notifs_file "$notifs_path" "$PII_EXPECT_EMAIL" "$PII_EXPECT_IPV4"` to emit two NDJSON frames.
3. Counts frames via `wc -l` for the `ok` message.
4. Pre-flight validates: if `jq` is available, loops with `while IFS= read -r line` and pipes each to `jq . >/dev/null`. If `jq` is missing, falls back to a single `python3 -c` loader over the whole file. If neither is available, emits a `warn` and proceeds (graceful degrade — the validators are quality-of-life, not correctness gates). AP-2 mitigation.
5. T2 guard: `curl_auth GET "$BASE/admin/about" -o "$about_body"`. Parses the value from `<dt>KIRO_CMD</dt><dd>VALUE</dd>` via `sed -n 's:...<dt>[[:space:]]*KIRO_CMD[[:space:]]*</dt>[[:space:]]*<dd>\([^<]*\)</dd>.*:\1:p'`. If empty: `fail "could not parse KIRO_CMD value from /admin/about HTML"`. If value does not match `grep -qi 'fake'`: `fail "T2 GUARD: gateway is not pointed at a fake worker..."` and `return`. Same defensive posture as the PowerShell sibling.
6. Informational `info` line telling the operator to set `OTTO_FAKE_KIRO_NOTIFICATIONS_FILE=$notifs_path` in the gateway env if not already wired.
7. Runs the existing `pii_probe` per `$SURFACE` — byte-identical surface dispatch to `scenario_pii_live` and to the pre-Phase-08.3.2 `scenario_pii`.

The script does NOT start/stop the gateway. Operator owns gateway lifecycle (matches the existing scenario contract and the PowerShell sibling).
