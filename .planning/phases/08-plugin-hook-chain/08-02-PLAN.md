<objective>
Slice 2 — AuthHook vertical slice: refactor the bearer-token validation
out of HTTP middleware (`internal/auth/bearer.go`) into a canonical-typed
PreHook that runs on the engine seam. The adapter→hook credential bridge
is the load-bearing connective tissue. Slice does NOT remove the existing
`auth.Bearer` chi middleware — that's slice 5's wiring concern after the
full chain runs in main.go. This slice ships the hook + the
canonical-ctx credential bridge + the per-surface handler stamp.

Purpose: PLUG-03 + the "one place to enforce policy" core value cash-out
— auth lives at the canonical layer, NOT three places (one per adapter).
Closing this slice means a bad bearer token short-circuits the chain
via canonical envelope (T-8-AUTH mitigation) without ever touching
`engine.Run`'s ACP path. Each surface's existing error renderer
(`anthropic/errors.go`, OpenAI's error envelope, Ollama's `{error:"..."}`)
handles the per-surface translation.

Output: `AuthHook` implementing `engine.PreHook`; `canonical.WithBearerToken`
/ `canonical.BearerTokenFromContext`; three adapter handler files stamping
ctx with the bearer credential before `engine.Run`. Co-exists with the
existing `auth.Bearer` chi middleware (defense-in-depth during transition);
slice 5 wires AuthHook into main.go and removes the middleware in one
atomic commit.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client → adapter (HTTP headers) | `Authorization: Bearer <token>` / `x-api-key: <token>` — UNTRUSTED |
| adapter handler → engine.Run (ctx-stamped credential) | Token attached to ctx via canonical helper; canonical-typed boundary |
| AuthHook → engine.Collect (short-circuit) | Bad token produces canonical error response; engine.Collect renders verbatim per Codex H-4 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-8-AUTH | Spoofing / EoP (bearer bypass) | `internal/plugin/auth.go` AuthHook.Before | mitigate | `subtle.ConstantTimeCompare(providedBytes, []byte(valid))` per `internal/auth/bearer.go:51` precedent — equal-length constant-time compare; bad token → canonical short-circuit envelope rendered as native error per surface (OpenAI `{error:{...}}`, Ollama `{error:"..."}`, Anthropic `{type:"error", error:{...}}`). Test asserts: (a) no `==` operator on tokens; (b) bad-token e2e returns 401-shaped envelope for each of three surfaces. |
| T-8-AUTH-2 | Information Disclosure (timing side-channel revealing token count) | AuthHook iteration over `h.Tokens` | accept | Early-exit on match leaks "how many tokens are configured" but NOT token bytes. RESEARCH Pitfall 3 + `bearer.go:48-50` accepted-tradeoff comment. Document in AuthHook code comment matching the precedent. |
| T-8-AUTH-3 | Tampering (bypass via empty-Tokens config) | AuthHook empty-Tokens passthrough | mitigate by test | Mirror `internal/auth/bearer.go:35-38` pattern: empty `Tokens` means auth disabled (Node parity). Test asserts both passthrough behavior AND that bad token IS rejected when Tokens is non-empty. |
| T-8-AUTH-4 | Information Disclosure (header credential logged) | Adapter handlers stamping ctx | mitigate | Adapter handlers MUST NOT log the raw header value. The stamped ctx-credential is consumed by AuthHook and never serialized. Verify by grep: no `logger.*Authorization|logger.*x-api-key|logger.*token` patterns appear in adapter handlers after this slice. |
</threat_model>
