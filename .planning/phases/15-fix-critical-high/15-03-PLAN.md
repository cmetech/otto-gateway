## Icon Strategy (D-14, RESEARCH.md Pitfall 7)

The current template.png/template.ico are monochrome template images. For colored state
icons (green running, yellow warning, red error), PNG assets passed to systray.SetIcon
(NOT SetTemplateIcon) preserve color. For the "running" state, SetTemplateIcon is still
acceptable (the existing green-ish monochrome is fine for "healthy"). For warning/error
states, color IS the signal — use SetIcon with a colored PNG/ICO.

Minimal viable icon approach: create simple solid-color square PNGs (32x32 px) in the
three state colors. Use Go's image/png + image/color packages in a generate script or
provide pre-generated PNG bytes. The assets must be committed as binary files.

If generating icon assets is complex, an alternative is to embed the same template.png
for all three states initially and note in SUMMARY that full-color assets are a v1.10
polish item. The fix's load-bearing property is that applyState CALLS setIconForState —
the icon differentiation is secondary. However, CONTEXT.md D-14 says "add missing 2"
variants, so at minimum add placeholder variants even if they look similar.
