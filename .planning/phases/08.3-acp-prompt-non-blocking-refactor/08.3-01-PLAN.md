## Phase Goal

**As a** loop24-client / GSD Pi / LangFlow / Pi-SDK user, **I want to** receive correct streamed responses from the OTTO Gateway for any prompt — including ones whose worker emits more than 64 session/update chunks before the session/prompt response — **so that** the gateway never wedges into a 100-second STREAM_IDLE_TIMEOUT just because the response is non-trivial (closes the Windows v1.9.2 PII smoke-test regression).

<objective>
Refactor `internal/acp/client.go`:`Prompt()` (lines 696-803) so it returns the `*Stream` as soon as `c.send(...)` accepts the session/prompt request, and moves the response-wait / cancellation / stream-finalization logic into a per-prompt goroutine named `awaitPromptResult`. Add a new `engine.prompt.completed` DEBUG log line, update `Stream`'s docstring to remove the "MUST drain concurrently" foot-gun warning (no API change), and land the load-bearing overflow regression test plus three whitebox close-race tests in the same plan. The plan is intentionally a single atomic vertical slice per RESEARCH.md §Summary "single atomic commit" recommendation — the deadlock fix, the new contract, the regression test, and the observability shift all ship together so an executor cannot partially-land a state where ACP-02 / ACP-03 re-validation is half-met.

Purpose: Close the chunk-buffer-overflow deadlock (Windows v1.9.2 PII smoke-test regression) and re-validate ACP-02 + ACP-03 against the live ACP spec without changing any wire-shape, canonical type, or public API. The fix is a recomposition of primitives that already exist in `internal/acp/` (`c.wg`, `s.closeOnce`, `c.streamMu`, dispatcher register/cancel, `failPending`+`closeSentinelCode`).

Output:
- Modified `internal/acp/client.go` with non-blocking Prompt + `awaitPromptResult` helper + `engine.prompt.completed` emission.
- Modified `internal/acp/stream.go` with updated concurrency-contract godoc (no executable change).
- Modified `internal/acp/integration_test.go` with `TestIntegration_Prompt_OverflowsBuffer_DoesNotDeadlock`.
- Modified `internal/acp/client_test.go` with four new whitebox tests.
- All trust gates (`make ci`) green; `goleak.VerifyTestMain` clean on every termination path.
- Operator-Windows PII smoke-test sign-off (blocking human-verify checkpoint).
</objective>

| Boundary | Description |
