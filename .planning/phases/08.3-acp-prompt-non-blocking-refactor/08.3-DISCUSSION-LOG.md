2. `engine.prompt.sent` log signal semantics

## Decisions

### 1. Cancellation ownership — **Two owners, double-cancel safe**

**Options presented:**
- A. Two owners, double-cancel safe (Recommended) — `engine.Run` keeps its `context.AfterFunc` watchdog; new ACP goroutine is also cancel-aware. Cancel is idempotent.
- B. Single owner — goroutine only (remove engine.Run watchdog)
- C. Single owner — watchdog only (goroutine doesn't send session/cancel)

**Selected:** A.

**Notes:**
- `ACPClient.Cancel(sid)` is documented idempotent at `internal/engine/engine.go:152-158`.
- Smaller engine.go diff; defense-in-depth against races between ctx fire and Prompt return.
- Streaming adapters' existing `runHandle.StopWatchdog()` flow at `internal/adapter/anthropic/handlers.go:194-197` continues to work unchanged.

### 2. `engine.prompt.sent` log signal — **Two log lines**

**Options presented:**
- A. Two log lines (Recommended) — keep `engine.prompt.sent` (meaning shifts to "send accepted") + add `engine.prompt.completed` (response landed) from inside the ACP goroutine.
- B. Rename to clarify timing — single line, no new emission, but breaks external log greps.
- C. Keep semantics by moving emission into goroutine — single line, original timing preserved, but loses the new early signal.

**Selected:** A.

**Notes:**
- `engine.prompt.completed` carries `session_id`, `chunks`, `stop_reason` to match the existing log field shape.
- The new `prompt.sent → prompt.completed` interval would have made the deadlock obvious-on-sight in the original v1.9.2 incident log.
- ChatTrace / external grep compatibility preserved.

### 3. Test strategy — **Add overflow-no-deadlock test + keep existing**

**Options presented:**
- A. Add new `TestIntegration_Prompt_OverflowsBuffer_DoesNotDeadlock` (≥128 chunks emitted before prompt response) + keep existing mixed-variants test as a regression artifact (Recommended).
- B. Modify existing mixed-variants test to encode new contract directly.
- C. Both — add new + simplify existing.

**Selected:** A.

**Notes:**
- Mixed-variants test's "Prompt-on-one-goroutine, drain-on-another" pattern still works under the new contract — kept as backward-compat regression artifact.
- New test asserts: Prompt returns within tight deadline (e.g. 100 ms), all chunks drain, Stream.Result returns expected StopReason, goleak passes.
- Goleak coverage extended to catch the new per-prompt goroutine leaking on any termination path.
