# Phase 8.3 Plan 01: Non-Blocking acp.Client.Prompt() Summary

**Eliminated the chunk-buffer-overflow deadlock in `internal/acp/client.go:Prompt()` by lifting the response-wait into a per-prompt `awaitPromptResult` goroutine (registered with `c.wg`), so `Prompt()` now returns `(*Stream, nil)` within milliseconds of `c.send` acceptance regardless of subsequent chunk volume. Closes the Windows v1.9.2 PII smoke-test regression that wedged the gateway whenever kiro-cli streamed >64 `session/update` chunks before its `session/prompt` response.**

## GREEN — Implementation

**Refactor shape (lift-and-shift; minimum-diff per RESEARCH.md Pattern 1):**

1. **`Prompt()` body** — replaced the blocking `select { case <-ctx.Done(): ... case frame := <-respCh: ... }` (`client.go:756-803` pre-refactor) with:
   - `id := c.nextID.Add(1)` + `respCh := c.disp.register(id)` — kept.
   - DELETED the top-level `defer c.disp.cancel(id)` (moves into the goroutine).
   - `c.streamMu.Lock(); c.activeStream = stream; c.streamMu.Unlock()` — kept.
   - `c.send(...)` — kept; on error path: explicit `c.disp.cancel(id)` BEFORE clearing activeStream + return (Pitfall 4: no goroutine spawned on send-fail).
   - On send success: `c.wg.Add(1)` on its own line (Go memory model — Add must happen-before goroutine start), then `go c.awaitPromptResult(ctx, id, sessionID, stream, respCh)`, then `return stream, nil`.

2. **New `awaitPromptResult(ctx, id, sessionID, stream, respCh)` method** — lifted the select body verbatim from the old synchronous path with these translations:
   - First defer: `defer c.wg.Done()` (LIFO last → runs last so Close()'s c.wg.Wait sees the dispatcher cleanup before unblocking).
   - Second defer: `defer c.disp.cancel(id)` (LIFO first → runs first on return; benign double-delete on the happy path because route() already deleted; required on ctx-cancel + close-sentinel arms).
   - ctx-cancel arm: clear activeStream, send defensive `session/cancel` notification (CONTEXT.md D-01 two-owner pattern — duplicates engine.Run's watchdog harmlessly), `stream.close(nil, fmt.Errorf("acp: prompt: %w", ctx.Err()))`.
   - frame arm — Error != nil: closeSentinelCode → `stream.close(nil, ErrClientClosed)`; else `stream.close(nil, fmt.Errorf("acp: prompt: rpc error %d: %s", code, msg))`.
   - frame arm — happy path: parse `r.StopReason` via existing `parseStopReason` (D-07 forward-compat), read `stream.result.ChunkCount` under `stream.mu.Lock()` (Pitfall 6 defense-in-depth), emit `engine.prompt.completed` Debug log line with `session_id` + `chunks` + `stop_reason` (raw wire string per CONTEXT.md A1/D-02 — canonical.StopReason has no String() method), call `stream.close(&FinalResult{StopReason: stop}, nil)`.

3. **Prompt godoc rewrite** — removed the "Concurrency contract (Phase 1.1 WR-01)" paragraph + the synchronous-drain example; replaced with a Phase 8.3 paragraph that describes the new behavior.

**dispatcher.cancel defer location:** moved from `Prompt()` top level (pre-Phase-8.3 line 729) into `awaitPromptResult` as the second defer. Send-failure path retains an EXPLICIT (non-deferred) call to `c.disp.cancel(id)` because no goroutine is spawned on that path.

**c.wg.Add(1) placement:** on its own line at client.go:758, immediately before `go c.awaitPromptResult(...)` at line 759. Verified via `awk '/c\.wg\.Add\(1\)/{add=NR} /go c\.awaitPromptResult/{ if (add && NR > add) print "OK" }' internal/acp/client.go` → `OK: Add at line 758, go statement at line 759`.
