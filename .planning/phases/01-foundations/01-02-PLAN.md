  newClient(rwc io.ReadWriteCloser, cfg Config) *Client — starts readLoop + writerLoop + pingLoop goroutines (wg.Add(3))
  readLoop(ctx context.Context)  — defer wg.Done(); loops on readFrame → unmarshal → disp.route
  writerLoop(ctx context.Context) — defer wg.Done(); loops on select { case b := <-writeCh: framer.writeFrame(b); case <-ctx.Done(): drain writeCh to zero, return }
  pingLoop(ctx context.Context)  — defer wg.Done(); ticker-based ping
  Close() — closeOnce.Do: cancel() → drain writeCh sentinel (close writeCh) → rwc.Close() (if rwc != nil) OR stdin.Close() (subprocess path) → failPending(ErrClientClosed) → wg.Wait() → cmd.Wait() if owned

  CLOSE ORDER IS MANDATORY (D-07 + Codex HIGH fix):
    1. c.cancel()                          — signal all goroutines to stop
    2. close(c.writeCh)                    — signal writerLoop to drain and exit
    3. if c.cmd != nil { c.stdin.Close() } // subprocess: EOF → kiro-cli → readLoop unblocks
       else { c.rwc.Close() }              // NewWithConn: close injected RWC → scanner unblocks
    4. c.failPending(ErrClientClosed)      — drain pending map, send error to all waiting callers
    5. c.wg.Wait()                         // wait for reader + writer + ping goroutines
    6. if c.cmd != nil { c.cmd.Wait() }    // reap subprocess
