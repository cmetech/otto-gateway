--- Go-novice note: channel semantics ---
A buffered-1 channel can hold exactly one value before the sender blocks. In drainAll,
sending to a full buffered-1 channel while holding the mutex creates a deadlock: nobody
can receive from the channel (caller already walked away) and nobody can acquire the mutex
(drainAll is holding it). The fix is a non-blocking select — if the channel is full, the
frame is already there (caller got it or is gone), so dropping the duplicate is safe.

--- Fix 1: CR-01 — dispatcher.go drainAll (line 86) ---
Change the blocking send inside drainAll to a non-blocking select:

BEFORE (line 86):
    ch <- rpcFrame{Error: &rpcError{Code: -32099, Message: err.Error()}}

AFTER:
    select {
    case ch <- rpcFrame{Error: &rpcError{Code: -32099, Message: err.Error()}}:
    default:
        // Channel already has a frame (route() beat us) or caller is gone — drop safely.
    }

This keeps the delete(d.pending, id) line on the next line unchanged.

sees all goroutines exited. Do NOT reorder these defers.

--- Go-novice note: why a separate wire type? ---
The canonical.Block is an internal representation used throughout the gateway. The ACP
wire format uses a different JSON shape: {"type":"text","content":"..."} for text blocks
and {"type":"resource_link","uri":"...","title":"..."} for resource links. By keeping
the wire type inside package acp, the canonical package stays free of ACP-specific
concerns — if kiro-cli changes its wire format in the future, only translate.go changes.

Add the following to the END of translate.go (after the translateUpdate function):

    // wireBlock is the ACP wire shape for a prompt input block.
    // canonical.Block uses a Go discriminated-union (Kind field + pointer variants);
    // kiro-cli expects a flat JSON object with a "type" string discriminator.
    // translateBlock converts between the two representations.
    type wireBlock struct {
        Type    string `json:"type"`
        Content string `json:"content,omitempty"`    // present for type=="text"
        URI     string `json:"uri,omitempty"`         // present for type=="resource_link"
        Title   string `json:"title,omitempty"`       // present for type=="resource_link"
    }

    // translateBlock converts a canonical.Block to the ACP wire shape.
    // Unknown BlockKind values fall back to an empty text block to avoid data loss.
    func translateBlock(b canonical.Block) wireBlock {
        switch b.Kind {
        case canonical.BlockKindText:
            if b.Text == nil {
                return wireBlock{Type: "text"}
            }
            return wireBlock{Type: "text", Content: b.Text.Content}
        case canonical.BlockKindResourceLink:
            if b.ResourceLink == nil {
                return wireBlock{Type: "resource_link"}
            }
            return wireBlock{
                Type:  "resource_link",
                URI:   b.ResourceLink.URI,
                Title: b.ResourceLink.Title,
            }
        default:
            // Unknown kind — fall back to empty text block to avoid data loss.
            return wireBlock{Type: "text"}
        }
    }

    // translateBlocks converts a slice of canonical.Block values to wire-shape structs.
    func translateBlocks(blocks []canonical.Block) []wireBlock {
        if len(blocks) == 0 {
            return nil
        }
        out := make([]wireBlock, len(blocks))
        for i, b := range blocks {
            out[i] = translateBlock(b)
        }
        return out
    }

No imports need to be added — translate.go already imports "loop24-gateway/internal/canonical".

## CR-04 and CR-06 Deferral Rationale

**CR-04 (permission field silently discarded):** The `permissionParams` struct captures only `RequestID`, losing the `permission` object (scope: shell_exec, file_write, etc.). Phase 1 auto-grants unconditionally per ACP-04 spec. The "one place to enforce policy" value is realised when Phase 8's plugin hook chain adds policy enforcement. Logging and echoing back the permission scope is a Phase 8 concern. Capturing the field now would add dead code (nothing consumes it) which golangci-lint's `unused` linter would flag.

**CR-06 (rpcFrame.ID is *uint64 only):** kiro-cli v2.4.1 (current deployed version) uses numeric IDs exclusively. Adding json.RawMessage ID handling now would require a non-trivial dispatcher rewrite. Defer until kiro-cli actually emits non-numeric IDs, at which point integration tests will surface the regression deterministically.
