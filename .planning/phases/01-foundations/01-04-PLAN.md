    ## Overview
    Brief intro: two wrapper scripts for developer-laptop lifecycle; binary is foreground-only (D-22).

    ## Quick Start (macOS / Linux)
    Code block showing: make build; ./scripts/loop24 start; ./scripts/loop24 status; ./scripts/loop24 stop.
    Or via Makefile: make start; make status; make stop.

    ## Quick Start (Windows)
    Code block showing: make build; .\scripts\loop24.ps1 start; .\scripts\loop24.ps1 status; .\scripts\loop24.ps1 stop.

    ## Subcommands
    Table or list: start (background launch), stop (send SIGTERM, remove PID), status (PID alive check + GET /health), restart (stop + start), logs [-f] (tail log file), run (foreground — same as running binary directly).

    ## File Locations
    Table:
    | File | macOS/Linux default | Windows default |
    |------|--------------------|-----------------|
    | Binary | ./bin/loop24-gateway | .\bin\loop24-gateway.exe |
    | PID file | /tmp/loop24-gateway.pid | %TEMP%\loop24-gateway.pid |
    | Log file (stdout) | /tmp/loop24-gateway.log | %TEMP%\loop24-gateway.log |
    | Log file (stderr) | merged into stdout | %TEMP%\loop24-gateway-err.log |

    ## Environment Variable Overrides
    Table of script-level overrides:
    | Variable | Default | Description |
    |----------|---------|-------------|
    | LOOP24_BIN | ./bin/loop24-gateway (or .exe on Windows) | Path to the gateway binary |
    | LOOP24_PID | /tmp/loop24-gateway.pid (or %TEMP%\... on Windows) | PID file location |
    | LOOP24_LOG | /tmp/loop24-gateway.log (or %TEMP%\... on Windows) | Log file location |
    | LOOP24_ADDR | http://localhost:11434 | Gateway address for health probe in status |

    ## Gateway Environment Variables
    These are set in your shell before calling the wrapper (they pass through to the binary):
    | Variable | Default | Description |
    |----------|---------|-------------|
    | HTTP_ADDR | :11434 | Bind address for the HTTP server |
    | KIRO_CMD | kiro-cli | kiro-cli binary name or full path |
    | KIRO_ARGS | acp | Arguments passed to kiro-cli |
    | KIRO_CWD | (empty) | Default working directory for kiro-cli |
    | DEBUG | false | Enable debug-level logging (1 or true) |
    | PING_INTERVAL | 60000 | ACP ping interval in ms (or Go duration string) |

    ## How status Works
    Explain: (1) checks PID file exists, (2) sends kill -0 to the PID (verifies process alive), (3) GETs /health and pretty-prints JSON response. If PID file is stale (process died unexpectedly), reports "stopped (stale PID)".

    ## Logs
    Explain: logs [-f] tails the log file (stdout+stderr merged on macOS/Linux; stdout only on Windows). For Windows stderr, check %TEMP%\loop24-gateway-err.log separately. Log format is JSON (slog.NewJSONHandler).
