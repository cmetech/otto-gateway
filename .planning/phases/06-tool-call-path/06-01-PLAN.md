    - **D-09 9-step algorithm — IMMUTABLE:**
      1. Defensive nil guards + skip: if `req == nil || resp == nil || len(req.Tools) == 0 || len(resp.Message.ToolCalls) > 0` → return false. (Idempotency invariant D-02 — running twice no-ops on the second call because Message.ToolCalls is now non-empty.)
      2. Extract assistant text: locate the single `ContentKindText` part in `resp.Message.Content`. If absent or empty (after `strings.TrimSpace`), return false.
      3. Try `json.Unmarshal([]byte(text), &parsed)` into `map[string]any`. If success, jump to step 6.
      4. If raw parse failed, run `stripFences(text)` (private helper) to strip ```json or bare ``` fences. If fence-strip returned `(text, false)` (no fence detected), return false. If fence-strip succeeded, retry `json.Unmarshal` on the stripped text.
      5. If still fail after fence-strip → return false. (Text preserved — never mutate response on parse failure.)
      6. `pickBestTool(parsed, req.Tools)` — see scoring rule below. Returns `(*canonical.ToolSpec, int)` — best matching spec + its score.
      7. If `score == 0` → return false. (Text preserved — better than firing the wrong tool. Per D-10 zero-score is no-coerce.)
      8. Rewrite: set the matched ContentKindText part's `Text = ""`; append synthetic `canonical.ToolCall{ID: fmt.Sprintf("call_%d", time.Now().UnixNano()), Name: bestSpec.Name, Arguments: parsed}` to `resp.Message.ToolCalls`.
      9. Return true.
    - **`pickBestTool(parsed map[string]any, tools []canonical.ToolSpec) (*canonical.ToolSpec, int)` — locked rules (D-10, Pitfall 4):**
      - Iterate `tools` in declaration order (slice index, NOT map iteration — Pitfall 4).
      - For each spec: extract `spec.Parameters["properties"]` as `map[string]any`. If absent / not a map / empty, skip this spec (zero score, can't compare).
      - Score = count of keys in `parsed` that appear as top-level keys in `properties`. Top-level only — no recursion (D-10).
      - Track `bestIdx int` and `bestScore int`. On strict `>` update both. On equality (`score == bestScore`), DO NOT update — first-declared wins (deterministic tie-break per D-10).
      - Return `(&tools[bestIdx], bestScore)`. If all specs had zero score, `bestScore == 0`.
    - **`stripFences(text string) (string, bool)` — locked rules (D-10):**
      - After `strings.TrimSpace`, check for `strings.HasPrefix(t, "```json\n")` then `strings.HasSuffix(t, "\n```")`; if both true, return `(trimmed-inner, true)`.
      - Else check `strings.HasPrefix(t, "```\n")` then `strings.HasSuffix(t, "\n```")`; if both true, return `(trimmed-inner, true)`.
      - Tolerate CRLF: also try `\r\n` variants OR run the result through `strings.ReplaceAll(_, "\r\n", "\n")` BEFORE the prefix/suffix check.
      - Returns `(text, false)` if no fence detected — `text` is the trim-spaced original (caller should NOT use the fenced text on a no-fence return; the `false` signal means "give up").
      - Per Pitfall 3: the fence MUST wrap the ENTIRE text (no prose before/after the fence). Two `HasPrefix`/`HasSuffix` chains with no inner search ensures this.
      - **No regex** (Pitfall 3 catastrophic backtracking + Don't-Hand-Roll guidance).
    - **D-12 property tests (testing/quick.Config{MaxCount: 1000}):**
      1. `TestCoerceToolCall_NeverPanics` — random `(text string, toolNames []string)` inputs through CoerceToolCall; defer-recover guard for nil-input panic. Per RESEARCH lines 435-453.
      2. `TestCoerceToolCall_Idempotent` — for arbitrary `(req, resp)`, `Coerce(Coerce(x))` mutation == `Coerce(x)` mutation (re-invocation no-ops because Message.ToolCalls is now non-empty).
      3. `TestCoerceToolCall_NoMatchNoMutation` — when `pickBestTool` scores zero (e.g., parsed JSON has no overlap with any tool's properties), the response is bit-identical to input. Compare `resp.Message.Content` and `resp.Message.ToolCalls` before/after.
      4. `TestCoerceToolCall_TieBreaker` — table-driven: two tools with identical property keys, parsed JSON matches both equally — first-declared wins. Deterministic across 1000 iterations.
