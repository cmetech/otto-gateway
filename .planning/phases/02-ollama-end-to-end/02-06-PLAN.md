## Trust Boundaries

Plan 06 mounts the HTTP server. Every inbound request crosses the network →
chi router → middleware chain → handler. The trust boundary is the HTTP
ingress — Plan 06 is where the auth (Plan 02) is actually enforced and where
the request body's untrusted JSON enters the canonical pipeline.

|-----------|----------|-----------|-------------|-----------------|

| T-02-30 | Tampering | Slowloris (slow-write attacks holding the server open) | mitigate | Phase 1 already sets `http.Server.ReadHeaderTimeout = 10*time.Second` in internal/server/server.go (verified in existing code). Plan 06 preserves this constant in the rewrite. |
| T-02-31 | Spoofing | Auth-bypass via missing AUTH_TOKEN env on a deployed binary | accept | "Empty AUTH_TOKEN = no auth" is the Node default behavior (D-14).
