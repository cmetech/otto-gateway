## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-11 | Tampering | Operator sets ALLOWED_IPS to a malformed CIDR and the gateway silently defaults to allow-all | mitigate | parseCIDRs returns a non-nil error for any malformed entry; Load() appends it to the errs slice; config.Load() then returns a non-nil aggregated error. TestLoad_AllowedIPs_Malformed + TestParseCIDRs_Error assert this contract. Operator gets a startup-time failure rather than a silent open gate. |
| T-02-12 | Tampering | Operator sets POOL_SIZE to a negative or non-integer; gateway proceeds with wrong concurrency | mitigate | getEnvInt errors on non-integer (asserted by TestLoad_PoolSize_Malformed). Plan 04's Pool.applyDefaults floors negative values to 1 (defense in depth at the consumer). |
| T-02-13 | Information Disclosure | AUTH_TOKEN value leaking via accidental log of full config | accept | Config has no String() method; if a future contributor logs the whole struct, the value will appear. The fix is a Stringer that redacts; deferred to a later phase. For Phase 2, only Logger.Info("config loaded", "auth_enabled", len(cfg.AuthToken) > 0)-style logging is sanctioned (Plan 06 enforces). |
| T-02-14 | Tampering | Operator confuses comma-split (AUTH_TOKEN, ALLOWED_IPS) with whitespace-split (KIRO_ARGS) | mitigate | New helper `getEnvStrSliceComma` is distinct from existing `getEnvStrSlice` (whitespace). Doc-comment on each spells out the separator. TestLoad_AuthToken_WithSpaces asserts the whitespace-trim within comma-split behavior. |

Residual exposure: none new at the config layer. AUTH_TOKEN remains an
operator-controlled plain-string secret (acceptable per PROJECT.md context).
</threat_model>
