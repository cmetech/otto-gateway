<objective>
Restore the TRST-04 adapter-over-canonical boundary by relocating the
`ErrPoolExhausted` sentinel from `internal/pool` to `internal/canonical` (per
D-17-01). Today the three adapters (`anthropic`, `ollama`, `openai`)
`errors.Is`-check against `pool.ErrPoolExhausted`, which forces them to import
`internal/pool` — the load-bearing arch-lint violation surfaced by `make ci` at
v1.9 milestone close.

Fix: define `canonical.ErrPoolExhausted` as the authoritative sentinel
(alongside the existing `canonical.ErrStreamIdleTimeout` precedent at
`internal/canonical/errors.go:17`). Re-export it from the `pool` package as
`var ErrPoolExhausted = canonical.ErrPoolExhausted` so existing internal
callers and any external/test code keep compiling. Flip the 7 confirmed
adapter `errors.Is` sites to reference `canonical.ErrPoolExhausted` directly,
and drop the now-unused `"otto-gateway/internal/pool"` import from each
adapter package (canonical is already imported in all three — no new boundary
crossings introduced).

Per D-17-05, scope is strictly bounded to this relocation. If the import-path
flip surfaces a downstream issue (circular import, unexpected consumer that
asserts the literal `pool` package import), STOP and surface as a Phase 17
risk — do NOT expand scope unilaterally.

Purpose: closes the project's oldest architectural invariant (TRST-04
adapter-over-canonical, established Phase 1) which Phase 15-01 broke for
convenience. Without this fix `make ci`'s arch-lint stage cannot exit 0 and
`v1.9.1` cannot ship from a build-green baseline.
Output: One atomic commit (or two — see Task 1's worktree spike) that adds
the canonical sentinel, re-exports from pool, flips all three adapters'
import + errors.Is sites, and produces `total notices: 0` from
`~/go/bin/go-arch-lint check --project-path .`.
</objective>
