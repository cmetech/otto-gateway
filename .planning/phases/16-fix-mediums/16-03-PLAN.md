<objective>
Fix one confirmed Medium finding in the Hooks subsystem:

G-1 (REL-HOOKS-01): Non-streaming aggregation error paths (idle-timeout 504, Result()-error 500) in engine.Collect and anthropic.CollectAnthropicChat return BEFORE running the PostHook chain. This means LoggingHook.startTimes and ChatTraceHook.startTimes sync.Map entries are never reclaimed on error — every failed request leaks one entry, causing unbounded memory growth under a retry storm against a wedged kiro-cli. chat-trace.log also never gets its post_chain_out record for failed requests.

Fix: On every error path in engine.Collect (idle-timeout return at :165, loopErr return at :171, Result() error return at :174) and anthropic.CollectAnthropicChat (idle-timeout return at :177, Result() error return at :184), call the PostHook chain with a nil resp BEFORE returning the error. Update LoggingHook.After() and ChatTraceHook.After() to guard resp == nil and still call startTimes.LoadAndDelete to reclaim the entry.

Purpose: Closes a silent memory leak that makes the gateway unreliable under retry storms; ensures chat-trace.log is complete for observability.
Output: One atomic commit pairing all five file changes with removal of the t.Skip in the Phase 14 regression test.
</objective>
