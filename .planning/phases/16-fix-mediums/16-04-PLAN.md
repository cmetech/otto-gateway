<objective>
Fix four confirmed Medium Tray/wrapper findings. This plan is Wave 3 because T-5 consumes pool.LastProgressAt() (Plan 16-01, Wave 1) and PoolStats.Status (Plan 16-02, Wave 2).

T-4 (REL-TRAY-04): notify() runs a blocking PowerShell MessageBox synchronously on the uiLoop goroutine — calling Stop from the Windows tray stalls uiLoop for up to 30s and may pop a modal. Fix: introduce notifyFn package-level var (testability seam) and dispatch each notify() call in a fire-and-forget goroutine with bounded retry (max 3 attempts, 500ms backoff). Apply to both uihelpers_windows.go and uihelpers_darwin.go for platform symmetry.

T-5 (REL-TRAY-05): makeProbe swallows snapshot errors (snap, _ = client.snapshot()) so a zero-value Snapshot is treated as healthy. The FSM also has no rule for the "busy-but-not-serving" (pool.status == "degraded") case. Fix: surface snapshot errors in makeProbe; update Snapshot.Pool.Status string in status.go; add FSM rule in fsm.go that fires StateDegraded when Pool.Status is "degraded" or "exhausted".

T-6 (REL-TRAY-06): revealBundle passes the full stdout (including wrapper config chatter) to Explorer. Fix: Go-side revealBundle parses the last non-empty stdout line; PS1-side moves informational Write-Host output to stderr so stdout contains only the archive path.

T-7 (REL-TRAY-07): Support bundle in scripts/otto-gw.ps1 does not cap live log copies and has no overall timeout with staging cleanup. Fix: apply --max-mb cap to all collected files including live logs; add 180s verb timeout with staging cleanup on timeout; write progress to stderr (not stdout).

Purpose: Stops uiLoop stalls during Windows Stop, makes the tray honest about wedged-pool state, fixes archive path parse on Windows, and bounds bundle size/time.
Output: Four atomic commits (one per finding), each unskipping its Phase 14 regression test (T-4, T-5, T-7) or updating its permanent-skip message (T-6).
</objective>
