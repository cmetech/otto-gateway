## Accomplishments

- `auth.Bearer` accepts BOTH `Authorization: Bearer` AND `x-api-key` via the new `extractToken` helper. Authorization-wins precedence rejects a bad Bearer even when a valid x-api-key is supplied (downgrade-attack guard).
- `config.Load` reads `ENABLED_SURFACES` (default `ollama,anthropic`) and `ANTHROPIC_PATH_PREFIX` (default `/v1`). `validateEnabledSurfaces` fail-fasts on unknown surface names with a message naming the offender; verified end-to-end (running the binary with `ENABLED_SURFACES=ollama,anthrpic` exits non-zero with a clear diagnostic).
- `server.NewFromConfig` grows `AnthropicPath` + `AnthropicProtectedRouter` Config fields and mounts the anthropic protected router behind the SAME auth chain as Ollama (D-15 propagates dual-header reading to both surfaces with zero per-adapter middleware). Nil-router gate skips cleanly.
- `engine.Collect` adds a second `strings.Builder` for thoughts and the chunk-kind switch emits a `ContentKindThinking` part after the text part when thoughts arrived. The dormant Phase 2 seam is now live — the foundation for ANTH-07 outbound rendering in Plan 03.1-02.
- `engine.buildBlocks` adds `joinThinkingParts` and emits `[Reasoning]\n<thinking>\n\n` AFTER `[Assistant]\n<text>\n\n` so inbound thinking content from Anthropic conversation history reaches kiro-cli as a distinct section (D-11 option a). Text-only assistant turns are regression-guarded against a phantom `[Reasoning]` header.
- `.go-arch-lint.yml` declares all four new components (`adapter_ollama`, `adapter_anthropic`, `engine`, `pool`) plus the missing `auth` component, with TRST-04 mayDependOn lists that constrain adapter packages to `canonical` only. The check exits 0 (was 19 notices on the Phase 1 baseline).

## Decisions Made

1. **D-15 precedence: Authorization wins over x-api-key.** Threat T-3.1-AUTH explicitly requires this guard against a downgrade attack where an attacker supplies `Authorization: Bearer wrong` alongside a stolen `x-api-key: good`. The test case `bearer_wrong_x_api_key_valid_rejects` pins the contract.

2. **D-16 allow-list explicitly REJECTS `openai`.** Phase 3 will widen the allow-list, but until then a premature `ENABLED_SURFACES=openai` MUST fail fast rather than silently disabling all surfaces. Pinned with `TestValidateEnabledSurfaces_OpenAIStillForbidden`.

3. **D-02 follow-on option (a): keep populating Ollama `Thinking` field.** The plan gave latitude; chose option (a) because the existing `omitempty` JSON tag makes the change strictly additive (LangFlow tolerates the field; Node never emits it; we now do when thinking content exists — no regression on the negative side).

4. **D-11 option (a): emit a new `[Reasoning]` bracketed section** (rather than folding thinking text into `[Assistant]`). Keeps `[Assistant]` semantically pure for response text and mirrors the Phase 2 `[System]`/`[User]`/`[Assistant]` convention from the Node reference. Tested ordering: [Assistant] before [Reasoning].

5. **Task 3 placeholder file.** Created `internal/adapter/anthropic/doc.go` as a package-doc-only file so `go-arch-lint` can resolve the `adapter_anthropic` component before Plan 03.1-02 writes any adapter source. Alternative — omitting the component declaration until Plan 03.1-02 — was rejected because the plan's `must_haves.truths` explicitly requires `adapter_anthropic` to be declared in this plan.

6. **Lint-config catch-up.** The Phase 1 `.go-arch-lint.yml` listed only 6 components. Phase 2 added engine, pool, adapter/ollama, and auth in source but never updated the lint. This plan catches up to reality (per the plan's "expectation is that Phase 2 already structured imports correctly and this lint config is catching up"). The server→auth dep was always intended (since Phase 2 D-17 mounted auth.Bearer inside server); the lint file just hadn't been updated.
