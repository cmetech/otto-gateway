# Phase 3.1: Anthropic Surface - Research

**Researched:** 2026-05-23
**Domain:** Go HTTP gateway / Anthropic Messages API server / SSE event stream emission
**Confidence:** HIGH

## Summary

Phase 3.1 builds `internal/adapter/anthropic` ŌĆö a second adapter that mounts under
`/v1` (configurable via `ANTHROPIC_PATH_PREFIX`) and serves `POST /v1/messages`
in both non-streaming and streaming modes. It shares the Phase 2 canonical engine
unchanged: `engine.Collect` for `stream:false`, `engine.Run` + range
`run.Stream().Chunks()` for `stream:true`. Wire shapes are verified against
`@anthropic-ai/sdk@^0.90.0` type definitions in
`~/Projects/repos/local/loop24-client/node_modules/@anthropic-ai/sdk/` and the
official spec at `platform.claude.com/docs/en/api/messages-streaming`.

The two structurally novel pieces vs. Phase 2 are:

1. **SSE event sequencing.** Anthropic uses named SSE events
   (`event: <name>\ndata: <json>\n\n`) in a strict order
   (`message_start` ŌåÆ (`content_block_start` ŌåÆ `content_block_delta`+ ŌåÆ
   `content_block_stop`)+ ŌåÆ `message_delta` ŌåÆ `message_stop`) with `ping`
   keepalives interleaved. A per-request state machine that consumes
   `<-chan canonical.Chunk` and emits the correct named events is the heart
   of the adapter. CONTEXT.md D-04/D-05 lock the design: adapter-local state
   machine, single select-loop, writer touched from one goroutine only.

2. **Multi-surface routing under `/v1`.** Phase 2 owns `/api`; Phase 3.1
   introduces `ENABLED_SURFACES` and `ANTHROPIC_PATH_PREFIX` env vars (both
   default to enabling Anthropic on `/v1`) and grows `server.Config` with
   parallel `AnthropicPath` + `AnthropicProtectedRouter` fields. Phase 3
   inherits a deferred refactor (`Surfaces []SurfaceMount`) when OpenAI
   co-mounts.

The CONTEXT.md D-01..D-20 decisions are dense and largely lock the design.
Research here verifies the wire shapes those decisions assume, extracts SSE
examples from the live SDK + spec, and lays out the validation surface.

**Primary recommendation:** Implement the adapter as a thin translator over
existing engine APIs, with the SSE emitter as the single complexity hotspot.
Drive validation off golden-file event sequences captured from the spec
docs + a live `messages.stream()` integration test with a fake plugin
producing scripted chunks.

## User Constraints (from CONTEXT.md)

### Locked Decisions

> Copied verbatim from CONTEXT.md `<decisions>`. The planner MUST honor these ŌĆö do not propose alternatives.

**Engine / canonical surface (cross-cutting):**

- **D-01:** Anthropic adapter uses `engine.Run()` + ranges `Chunks()` directly. No new engine helper. SSE handler calls `engine.Run(ctx, req)`, gets `*engine.Run`, ranges `run.Stream().Chunks()`. Non-streaming uses `engine.Collect`. Zero engine-package signature churn.
- **D-02:** `engine.Collect` STOPS dropping `ChunkKindThought`; aggregates thoughts parallel to text into a `ContentPart{Kind: ContentKindThinking}` on the assembled `Message.Content`. Phase 2 Ollama's `render.go` updated trivially (planner picks: drop thinking part vs. populate `ollamaChatResponseMessage.Thinking`).
- **D-03:** Any `ChunkKind` change opens a new SSE block in the streaming state machine. Block 0 = text, Block 1 = thinking (when Thought arrives after Text), etc. Tool-call chunks slot in as the next block kind.

**Anthropic adapter:**

- **D-04:** Adapter-local block-index state machine. Per-request `sseEmitter` struct tracks `blockIndex`, `currentKind`, `blockOpen`, `messageID`, writer/flusher. No `BlockIndex` field added to `canonical.Chunk`.
- **D-05:** Single select-loop in the SSE handler goroutine handles ping ticker + chunk channel + ctx.Done. Writer touched from exactly one goroutine. No mutex.
- **D-06:** Phase 3.1 relies on `ctx` propagation for client-disconnect. No explicit `engine.Cancel(sid)`. SHOULD include a debug log line on `ctx.Done`.
- **D-07:** `anthropic-version` header checked per-handler in `handleMessages`. Missing ŌåÆ 400 `invalid_request_error`. Any non-empty value accepted.
- **D-08:** System block normalization ŌĆö string verbatim, array joined with `\n\n`. `cache_control` dropped. `req.System` stays string-typed.
- **D-09:** `tool_result.content` normalization ŌĆö string verbatim, array joined and images dropped with debug log.
- **D-10:** `anthropic-beta` headers and `cache_control` fields accepted-and-ignored; logged at debug. Never persisted to canonical.
- **D-11:** Inbound `thinking` blocks preserved as `ContentPart{Kind: ContentKindThinking, Text: ...}` in canonical. `engine.buildBlocks` extended (planner's discretion: `[Reasoning]` section vs. fold into `[Assistant]`).
- **D-12:** `usage` block returns honest zeros ŌĆö `input_tokens: 0, output_tokens: 0`. `cache_creation_input_tokens` and `cache_read_input_tokens` omitted (zero-value, `omitempty`).
- **D-13:** `redacted_thinking` blocks NEVER emitted outbound in Phase 3.1. Inbound `redacted_thinking` passed through or dropped at wire-decode (planner's call).
- **D-14:** Per-request `cwd` derivation reuses Phase 2 pipeline. `wire.go` populates `canonical.ChatRequest.ResourceLinks` from every `resource_link` content block.

**Auth + routing:**

- **D-15:** `auth.Bearer` extended to accept BOTH `Authorization: Bearer <token>` AND `x-api-key: <token>` globally. Same constant-time-compare; same `AUTH_TOKEN` env var.
- **D-16:** `ENABLED_SURFACES` env var introduced with default `ollama,anthropic`. Unknown surface name ŌåÆ `config.Load` returns error; binary exits non-zero. Phase 3 extends default to include `openai`.
- **D-17:** `server.Config` grows parallel `AnthropicPath` + `AnthropicProtectedRouter` fields. `NewFromConfig` adds `r.Route(cfg.AnthropicPath, ...)` block with auth middleware chain. Phase 3.1 does NOT solve multi-mount-on-`/v1` yet.
- **D-18:** `/v1/messages` is the ONLY `/v1` endpoint Phase 3.1 mounts. `count_tokens` is out of scope.
- **D-19:** `ANTHROPIC_PATH_PREFIX` env var defaults to `/v1`. Both `OPENAI_PATH_PREFIX` and `ANTHROPIC_PATH_PREFIX` default to `/v1` (SURF-08 endpoint-level disambiguation).

**Error handling:**

- **D-20:** Anthropic error envelope emitted by adapter-local helper `writeError(w, status, errorType, message)`. 8 `error_type` values + HTTP status mapping per ANTH-06.

## Project Constraints (from CLAUDE.md)

The planner MUST verify compliance with each of these directives:

- **Go 1.23+, no cgo in main binary** ŌĆö Phase 3.1 introduces no new third-party deps; stays pure-Go. The SSE writer uses stdlib (`net/http`, `time.Ticker`, `encoding/json`).
- **stdlib `net/http` + `chi`** ŌĆö `chi/v5` is already in `go.mod` (v5.3.0). Adapter exposes `ProtectedRouter() chi.Router` like Phase 2 Ollama.

- **Env vars match Node version** ŌĆö `ENABLED_SURFACES`, `ANTHROPIC_PATH_PREFIX` are NEW (no Node equivalent); existing Node vars (`AUTH_TOKEN`, etc.) preserved.
- **`gosec` G204 for subprocess spawn** ŌĆö not directly relevant to Phase 3.1 (no new subprocess paths); inherited from Phase 2 pool.
- **Trust gates non-negotiable from day one** ŌĆö Phase 3.1's new code must pass `golangci-lint` with the existing strict config (`.golangci.yml` already enables `gosec`, `errcheck`, `errorlint`, `wrapcheck`, `revive`/exported, `bodyclose`, `noctx`, etc.). The SSE handler particularly needs `goleak.VerifyNone(t)` coverage (single-goroutine design from D-05 makes this trivial).

| Instead of | Could Use | Tradeoff |
|------------|-----------|----------|
| stdlib SSE writer (`fmt.Fprintf(w, "event: %s\ndata: %s\n\n", ...)` + `Flush`) | A third-party SSE library | Stdlib is fewer lines and zero deps; SDK consumer is strict about framing so explicit control beats a wrapper. |

| `time.Ticker(15s)` for ping | Adaptive ping based on real flow | YAGNI; 15s is the SDK-tested cadence (60s idle timeout in client per CONTEXT.md `<specifics>`). |

### Pattern 1: SSE Emitter State Machine

**What:** Translate a stream of `canonical.Chunk` values into Anthropic SSE
events, where every `ChunkKind` transition opens a new content block.

**When to use:** Inside the `stream:true` branch of `handleMessages` only.
Stateless adapters (non-streaming `engine.Collect`) don't need this.

**Example:**

```go
// Source: This research ŌĆö synthesis of CONTEXT.md D-04 + D-05 + Anthropic spec.
// File: internal/adapter/anthropic/sse.go

// sseEmitter holds the per-request streaming state. Constructed inside the
// SSE handler; touched from exactly one goroutine (the handler) per D-05.
type sseEmitter struct {
	w           http.ResponseWriter
	flusher     http.Flusher
	logger      *slog.Logger
	messageID   string             // generated at message_start
	model       string             // echoed from request; "auto" passes through
	blockIndex  int                // 0 at start; bumped on kind transitions
	currentKind canonical.ChunkKind
	blockOpen   bool
}

// writeEvent emits one named SSE frame: `event: <name>\ndata: <json>\n\n`
// then Flush. Encoder errors after WriteHeader are non-recoverable
// (client has disconnected) ŌĆö log and propagate.
func (e *sseEmitter) writeEvent(eventName string, payload any) error {
	body, err := json.Marshal(payload)
	if err != nil {
		return fmt.Errorf("anthropic: marshal %s: %w", eventName, err)
	}
	if _, err := fmt.Fprintf(e.w, "event: %s\ndata: %s\n\n", eventName, body); err != nil {
		return fmt.Errorf("anthropic: write %s: %w", eventName, err)
	}
	e.flusher.Flush()
	return nil
}

// applyChunk processes one canonical.Chunk and emits the appropriate
// content_block_start/stop and content_block_delta frames.
func (e *sseEmitter) applyChunk(c canonical.Chunk) error {
	if e.blockOpen && c.Kind != e.currentKind {
		if err := e.writeEvent("content_block_stop", contentBlockStop{
			Type: "content_block_stop", Index: e.blockIndex,
		}); err != nil {
			return err
		}
		e.blockIndex++
		e.blockOpen = false
	}

	if !e.blockOpen {
		// Open a new block matching this chunk's kind.
		start := contentBlockStart{
			Type:  "content_block_start",
			Index: e.blockIndex,
		}
		switch c.Kind {
		case canonical.ChunkKindText:
			start.ContentBlock = blockHeader{Type: "text", Text: ""}
		case canonical.ChunkKindThought:
			start.ContentBlock = blockHeader{Type: "thinking", Thinking: ""}
		default:
			// ChunkKindToolCall / ChunkKindPlan dormant in Phase 3.1 ŌĆö
			// drop with debug log; do NOT open a block.
			e.logger.Debug("sse: unsupported chunk kind (Phase 3.1)", "kind", c.Kind)
			return nil
		}
		if err := e.writeEvent("content_block_start", start); err != nil {
			return err
		}
		e.currentKind = c.Kind
		e.blockOpen = true
	}

	// Emit a content_block_delta matching the chunk kind.
	switch c.Kind {
	case canonical.ChunkKindText:
		if c.Text == nil {
			return nil
		}
		return e.writeEvent("content_block_delta", contentBlockDelta{
			Type:  "content_block_delta",
			Index: e.blockIndex,
			Delta: textDelta{Type: "text_delta", Text: c.Text.Content},
		})
	case canonical.ChunkKindThought:
		if c.Thought == nil {
			return nil
		}
		return e.writeEvent("content_block_delta", contentBlockDelta{
			Type:  "content_block_delta",
			Index: e.blockIndex,
			Delta: thinkingDelta{Type: "thinking_delta", Thinking: c.Thought.Content},
		})
	}
	return nil
}
```

### Pattern 2: Run Loop with Ping Ticker

**What:** Single goroutine selects on chunk channel, ping ticker, and
ctx.Done. Writer touched from exactly this goroutine.

**Example:**

```go
// Source: This research ŌĆö synthesis of CONTEXT.md D-05.
// File: internal/adapter/anthropic/sse.go

func runSSEEmitter(ctx context.Context, w http.ResponseWriter, run *engine.Run, model string, logger *slog.Logger) error {
	flusher, ok := w.(http.Flusher)
	if !ok {
		return errors.New("anthropic: response writer is not flusher")
	}

	w.Header().Set("Content-Type", "text/event-stream")
	w.Header().Set("Cache-Control", "no-cache")
	w.Header().Set("Connection", "keep-alive")
	w.WriteHeader(http.StatusOK)

	e := &sseEmitter{
		w:         w,
		flusher:   flusher,
		logger:    logger,
		messageID: genMessageID(),
		model:     model,
	}

	if err := e.writeEvent("message_start", messageStart{
		Type: "message_start",
		Message: anthropicMessage{
			ID:           e.messageID,
			Type:         "message",
			Role:         "assistant",
			Model:        e.model,
			Content:      []contentBlock{},
			StopReason:   nil,
			StopSequence: nil,
			Usage:        usage{InputTokens: 0, OutputTokens: 0},
		},
	}); err != nil {
		return err
	}

	ticker := time.NewTicker(15 * time.Second) // CONTEXT.md <specifics>: ~15s, before SDK 60s idle timeout
	defer ticker.Stop()

	chunks := run.Stream().Chunks()

	for {
		select {
		case <-ctx.Done():
			logger.Debug("sse: client disconnect", "session_id", run.SessionID())
			return ctx.Err()
		case <-ticker.C:
			if err := e.writeEvent("ping", pingEvent{Type: "ping"}); err != nil {
				return err
			}
		case c, ok := <-chunks:
			if !ok {
				// Stream closed.
				if e.blockOpen {
					_ = e.writeEvent("content_block_stop", contentBlockStop{
						Type: "content_block_stop", Index: e.blockIndex,
					})
				}
				final, rerr := run.Stream().Result()
				stopReason := canonical.StopUnknown
				if final != nil {
					stopReason = final.StopReason
				}
				_ = e.writeEvent("message_delta", messageDelta{
					Type: "message_delta",
					Delta: messageDeltaInner{
						StopReason:   mapStopReason(stopReason), // "end_turn" / "max_tokens" / etc.
						StopSequence: nil,
					},
					Usage: messageDeltaUsage{OutputTokens: 0},
				})
				_ = e.writeEvent("message_stop", messageStop{Type: "message_stop"})
				return rerr
			}
			if err := e.applyChunk(c); err != nil {
				return err
			}
		}
	}
}
```

### Pattern 3: Dual-Auth Middleware Extension

**What:** Extend `auth.Bearer` to accept BOTH `Authorization: Bearer <token>`
AND `x-api-key: <token>` headers. D-15 says modify globally (one middleware,
one mental model) ŌĆö applies to all surfaces.

**Example:**

```go
// Source: Extension of internal/auth/bearer.go per CONTEXT.md D-15.
// Pre-condition: existing bearer.go reads only "Authorization: Bearer".
// Post-condition: also reads "x-api-key" when Authorization is absent or non-Bearer.

func Bearer(cfg Config) func(http.Handler) http.Handler {
	return func(next http.Handler) http.Handler {
		return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
			if len(cfg.Tokens) == 0 {
				next.ServeHTTP(w, r)
				return
			}

			provided := extractToken(r)
			if provided == "" {
				writeOllamaError(w, http.StatusUnauthorized, "Invalid or missing API key")
				return
			}

			for _, valid := range cfg.Tokens {
				if subtle.ConstantTimeCompare([]byte(provided), []byte(valid)) == 1 {
					next.ServeHTTP(w, r)
					return
				}
			}
			writeOllamaError(w, http.StatusUnauthorized, "Invalid or missing API key")
		})
	}
}

// extractToken implements the D-15 contract: try Authorization: Bearer
// first, fall back to x-api-key. Constant-time-compare path unchanged.
func extractToken(r *http.Request) string {
	if authHeader := r.Header.Get("Authorization"); strings.HasPrefix(authHeader, "Bearer ") {
		return authHeader[len("Bearer "):]
	}
	return r.Header.Get("x-api-key")
}
```

**Note on error envelope:** `auth.Bearer` lives in `internal/auth` and
currently writes the Ollama-shape error (`{"error": msg}`). The Anthropic
adapter's error envelope is different (`{"type":"error","error":{...}}`).
Two acceptable resolutions, planner picks:

1. **Per-surface error rendering in middleware** ŌĆö auth middleware writes
   a generic envelope; both adapter packages parse it on the way out.
   Over-engineered for Phase 3.1.
2. **Accept the Ollama-shape error from `auth.Bearer` even on `/v1/messages`** ŌĆö
   the SDK doesn't crash on it (it raises a generic `APIError` with the
   raw body). Pragmatic and acceptable per discuss-log's "auth not
   adapter-aware" decision.
3. **Lift the auth error into a chi error handler hook** ŌĆö chi supports
   per-route renderers. More machinery but cleaner long-term.

Recommend **option 2 for Phase 3.1** (simplest, consistent with D-15 "one
middleware"); revisit when Phase 8 adds the hook chain.

### Anti-Patterns to Avoid

- **Sharing the SSE writer across goroutines.** D-05 locks single-goroutine writer access. If a hook ever wants to inject content mid-stream, the hook returns a `canonical.Chunk` on a channel; the SSE goroutine consumes it. Never write directly to `w` from a hook.
- **Promoting `BlockIndex` into `canonical.Chunk`.** D-04 explicitly forbids this. Canonical stays wire-agnostic; the adapter owns the per-request block-index state machine.
- **Pre-buffering chunks to figure out block transitions.** Defeats the streaming benefit; D-03 says any kind change immediately opens a new block. The state machine is forward-only.
- **Re-rolling SSE framing.** Use the canonical `event: <name>\ndata: <json>\n\n` format from `fmt.Fprintf` + `Flush`. SDK parser is strict (`@anthropic-ai/sdk` `core/streaming.js`); custom framing breaks it.
- **Forwarding `anthropic-beta` to kiro-cli.** D-10 says accept-and-ignore. kiro-cli has no concept of Anthropic betas.
- **Letting `auth.Bearer` write the Anthropic error envelope.** The middleware is surface-agnostic; the Ollama-shape error body from `auth.Bearer` is acceptable on `/v1/messages` (loop24-client raises a generic APIError on 401, body shape secondary). Better: emit the canonical auth error from the hook chain in Phase 8.
- **Calling `os.Getenv` inside handlers.** All env reads stay in `config.Load`. Surface enablement is decided at startup.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| SSE frame parsing | A custom parser for inbound SSE | N/A in Phase 3.1 (we EMIT only) | Just `Flush` after every `Fprintf`. |
| JSON streaming | Custom incremental JSON encoder | `encoding/json` Marshal per-event | One event = one JSON object; we don't stream partial JSON inside an event. |
| Time-based ping scheduler | `for { time.Sleep(15s); ŌĆ” }` | `time.NewTicker(15 * time.Second)` + `defer ticker.Stop()` | Tickers integrate with `select`; Sleeps don't and leak goroutines on disconnect. |
| Goroutine-leak detection | Manual goroutine-count assertion | `goleak.VerifyNone(t)` per Phase 1/2 pattern | Trust gate; already wired in `testmain_test.go` for other packages. |
| Bearer token compare | `if provided == valid` | `subtle.ConstantTimeCompare` (existing in `auth.Bearer`) | Timing side-channel; existing code already correct. |
| Body-cap | Manual byte counting | `http.MaxBytesReader` via the existing `decodeJSONBody` helper pattern (Phase 2 Ollama `helpers.go`) | Codex M-5; same 4 MiB cap. |
| Chi sub-router composition | A bespoke router | `r.Route(prefix, ...)` + `r.Use(mw)` + `r.Mount("/", adapter.ProtectedRouter())` | Phase 2 pattern; same chi `Router` type used. |

**Key insight:** Phase 3.1 is mostly translation glue. The only place a
sophisticated mistake can hide is the SSE state machine ŌĆö and even there,
the design is constrained to a single goroutine with a `select` on three
channels (chunks, ticker, ctx). Property-test the state machine, golden-
test the event sequence, integration-test against real kiro-cli + real
loop24-client. That's the whole validation surface.

## Common Pitfalls

### Pitfall 1: SSE framing drift breaks SDK silently
**What goes wrong:** Emit `event:<name>\n` (no space after colon, or
extra whitespace) ŌĆö SDK parser accepts but downstream `data:` line
misses parsing as expected event.
**Why it happens:** Free-text `fmt.Fprintf` makes framing easy to typo.
**How to avoid:** A single `writeEvent(name, payload)` helper that
`fmt.Fprintf(w, "event: %s\ndata: %s\n\n", name, json)` and `Flush`. All
callers use it. Golden-test the literal bytes of a known short stream
against the Anthropic spec example (lines 647-671 of the spec doc).
**Warning signs:** SDK emits a generic "event without data" warning, or
the client never sees the response (events parse as comments / unknown
type).

### Pitfall 2: Forgetting Flusher.Flush() after every write
**What goes wrong:** Chunks arrive in real-time but the client sees
nothing until the response buffer fills (`http.ResponseWriter` default
behavior ŌĆö chunked encoding doesn't auto-flush per write).
**Why it happens:** `Flush` is on `http.Flusher`, not `http.ResponseWriter`
ŌĆö a type-assertion miss silently disables streaming.
**How to avoid:** Type-assert `w.(http.Flusher)` ONCE in `runSSEEmitter`,
return 500 if it fails. Call `e.flusher.Flush()` after EVERY
`fmt.Fprintf(e.w, ...)`. Property-test: count flushes >= count writes
under a fake writer.
**Warning signs:** Streaming test hangs until kiro-cli finishes (defeats
the purpose).

### Pitfall 3: Ping ticker interleaves mid-block-delta
**What goes wrong:** Without a single select-loop owner, a goroutine
emits `ping` between two `content_block_delta` lines of a partially-
formed event. SDK rejects.
**Why it happens:** Multi-goroutine writer access (D-05's anti-pattern).
**How to avoid:** D-05 locked design ŌĆö single goroutine, three-case
select, writer never escapes. Each `writeEvent` call is atomic from
the writer's perspective.
**Warning signs:** SDK errors with "unexpected token" or test golden
file mid-file content corruption.

### Pitfall 4: Block kind transition mid-content-block-delta
**What goes wrong:** Text chunk ŌåÆ Thought chunk ŌåÆ Text chunk produces
indices 0, 1, 2 but the state machine forgets to emit `content_block_stop`
between transitions.
**Why it happens:** Naive "just emit deltas" loop.
**How to avoid:** D-03 / D-04 design: on EVERY new chunk, check
`blockOpen && chunk.Kind != currentKind`. If true: emit `content_block_stop`,
bump index, emit `content_block_start`. Then emit the delta.
**Warning signs:** SDK throws "content block ended without start"
or content blocks at the wrong index.

### Pitfall 5: Anthropic `usage` shape on message_delta is cumulative
**What goes wrong:** Emit `output_tokens` per-delta, not cumulative
(per spec, "token counts shown in the usage field of the message_delta
event are cumulative").
**Why it happens:** Misread the spec.
**How to avoid:** Phase 3.1 emits `output_tokens: 0` in `message_delta`
(D-12 honest zeros). Single value, no per-delta accumulation. Phase 7+
real counting must implement cumulative semantics.
**Warning signs:** N/A in Phase 3.1; future regression risk.

### Pitfall 6: `stop_reason` mapping mismatches Anthropic enum
**What goes wrong:** Canonical `StopEndTurn` ŌåÆ emit `"stop"` (Ollama
convention) instead of `"end_turn"`.
**Why it happens:** Reuse Phase 2 `mapStopReason` from Ollama's
`render.go`.
**How to avoid:** Anthropic adapter MUST have its own `mapStopReason`
emitting `"end_turn"` | `"max_tokens"` | `"stop_sequence"` | `"tool_use"`
| `"pause_turn"` | `"refusal"` (or null when StopUnknown).
**Warning signs:** SDK's `stream.finalMessage()` reports the wrong
stop_reason; loop24-client switching logic breaks.

### Pitfall 7: Missing `anthropic-version` returns wrong shape
**What goes wrong:** Default 400 from chi binding emits text/plain
"missing header" instead of the Anthropic error envelope.
**Why it happens:** Treat the header check as decode-time validation
not body-time validation.
**How to avoid:** D-07 ŌĆö first line of `handleMessages` checks
`r.Header.Get("anthropic-version") == ""` and calls
`writeError(w, 400, "invalid_request_error", "anthropic-version header required")`.
**Warning signs:** Integration test against the SDK fails with raw text
body 400.

### Pitfall 8: `tool_use.input` rendered as a JSON string
**What goes wrong:** Adapter does `json.Marshal(input)` then puts the
resulting string in the wire shape (OpenAI convention).
**Why it happens:** Confusion with OpenAI's `arguments` field.
**How to avoid:** ANTH-03 requires `input` as a plain object ŌĆö
serialize `map[string]any` directly into the JSON output, not as a
nested string. Property-test round-trip: decode a known input map,
re-encode through adapter, parse, assert object equality (not string).
**Warning signs:** `@anthropic-ai/sdk` `ToolUseBlock.input` parser
treats the string as a primitive; downstream code reads
`block.input` as `string` instead of `object`.

### Pitfall 9: `ResourceLink` blocks lose their `name` field
**What goes wrong:** Phase 1.1 D-04 added a `name` field that wire decoders
must populate. Anthropic `resource_link` blocks have a `name` field
(per spec); failing to surface it breaks the engine's `pickCwd` chain when
deriving cwd from longest-common-parent.
**Why it happens:** Skipping required wire fields during normalization.
**How to avoid:** `wire.go` populates `canonical.ResourceLinkBlock.Name`
from the wire JSON's `name` field (NOT just `uri`). Test the round-trip.
**Warning signs:** kiro-cli rejects `resource_link` blocks (per ACP spec,
`name` is REQUIRED).

### Pitfall 10: `slices.Contains(cfg.EnabledSurfaces, "anthropic")` typo-tolerant
**What goes wrong:** `ENABLED_SURFACES=anthrpic` (typo) ŌåÆ contains check
returns false ŌåÆ Anthropic surface disabled silently ŌåÆ user sees 404 on
`/v1/messages` and assumes the code is broken.
**Why it happens:** Loose env var validation.
**How to avoid:** D-16 ŌĆö `config.Load` validates against an allow-list
`{"ollama","anthropic"}` (Phase 3 adds `"openai"`). Unknown name returns
a clear error and the binary exits non-zero.
**Warning signs:** Operator reports "404 on /v1/messages despite
ENABLED_SURFACES set" ŌĆö config validation prevents the silent failure.

```go
	Name     string         `json:"name,omitempty"`     // for type="tool_use"
	Input    map[string]any `json:"input,omitempty"`    // for type="tool_use" ŌĆö OBJECT per ANTH-03
}

type usage struct {
	InputTokens              int  `json:"input_tokens"`
	OutputTokens             int  `json:"output_tokens"`
	CacheCreationInputTokens *int `json:"cache_creation_input_tokens,omitempty"`
	CacheReadInputTokens     *int `json:"cache_read_input_tokens,omitempty"`
}

// chatResponseToMessage converts a canonical.ChatResponse (assembled by
// engine.Collect with D-02 thought aggregation) into the Anthropic
// non-streaming Message wire shape. Honest zeros per D-12.
func chatResponseToMessage(resp *canonical.ChatResponse, model string) *anthropicMessage {
	out := &anthropicMessage{
		ID:    resp.ID, // adapter chooses generator (D-15-discretion); reuse Phase 2 chatcmpl- prefix or new uuid
		Type:  "message",
		Role:  "assistant",
		Model: model,    // echo requested model (kiro-cli may have "auto"; loop24-client treats as opaque)
		Usage: usage{InputTokens: 0, OutputTokens: 0},
	}

	stopWire := mapStopReason(resp.StopReason) // returns "end_turn" / "max_tokens" / ... or nil
	out.StopReason = stopWire
	// StopSequence stays nil (no custom stop_sequences in Phase 3.1)

	for _, part := range resp.Message.Content {
		switch part.Kind {
		case canonical.ContentKindText:
			out.Content = append(out.Content, contentBlock{Type: "text", Text: part.Text})
		case canonical.ContentKindThinking:
			out.Content = append(out.Content, contentBlock{Type: "thinking", Thinking: part.Text})
		case canonical.ContentKindToolUse:
			if part.ToolUse != nil {
				out.Content = append(out.Content, contentBlock{
					Type:  "tool_use",
					ID:    part.ToolUse.ID,
					Name:  part.ToolUse.Name,
					Input: part.ToolUse.Input,
				})
			}
		// ContentKindImage and ContentKindToolResult are inbound-only on Anthropic
		// (assistant never emits these); skip.
		}
	}

	// Defensive: if no content parts produced (edge case ŌĆö empty stream),
	// emit a single empty text block so SDK doesn't see content: [].
	if len(out.Content) == 0 {
		out.Content = []contentBlock{{Type: "text", Text: ""}}
	}

	return out
}

func mapStopReason(s canonical.StopReason) *string {
	var r string
	switch s {
	case canonical.StopEndTurn:
		r = "end_turn"
	case canonical.StopMaxTokens:
		r = "max_tokens"
	case canonical.StopRefusal:
		r = "refusal"
	case canonical.StopCancelled:
		r = "end_turn" // closest semantically; SDK has no "cancelled"
	case canonical.StopMaxTurnRequests:
		r = "max_tokens" // closest
	default: // StopUnknown
		return nil
	}
	return &r
}
```

```go
type errorEnvelope struct {
	Type  string     `json:"type"` // always "error"
	Error errorInner `json:"error"`
}

type errorInner struct {
	Type    string `json:"type"`    // one of the 8 error_type values
	Message string `json:"message"`
}

const (
	errInvalidRequest  = "invalid_request_error"
	errAuthentication  = "authentication_error"
	errPermission      = "permission_error"
	errNotFound        = "not_found_error"
	errRequestTooLarge = "request_too_large"
	errRateLimit       = "rate_limit_error"
	errAPI             = "api_error"
	errOverloaded      = "overloaded_error"
)

func writeError(w http.ResponseWriter, status int, errorType, message string) {
	w.Header().Set("Content-Type", "application/json")
	w.WriteHeader(status)
	_ = json.NewEncoder(w).Encode(errorEnvelope{
		Type:  "error",
		Error: errorInner{Type: errorType, Message: message},
	})
}

// Mid-stream SSE error: emit as event: error frame, then close stream.
func writeSSEError(w http.ResponseWriter, flusher http.Flusher, errorType, message string) {
	env := errorEnvelope{
		Type:  "error",
		Error: errorInner{Type: errorType, Message: message},
	}
	body, _ := json.Marshal(env)
	_, _ = fmt.Fprintf(w, "event: error\ndata: %s\n\n", body)
	flusher.Flush()
}
```

### Header + body validation

```go
// Source: CONTEXT.md D-07 + D-08.
// File: internal/adapter/anthropic/handlers.go

const messagesBodyCap int64 = 4 << 20 // 4 MiB ŌĆö same as Phase 2 chat body

func (a *Adapter) handleMessages(w http.ResponseWriter, r *http.Request) {
	if a.cfg.Engine == nil {
		writeError(w, http.StatusServiceUnavailable, errAPI, "kiro-cli not configured (set KIRO_CMD)")
		return
	}

	// D-07: anthropic-version is REQUIRED on input.
	if r.Header.Get("anthropic-version") == "" {
		writeError(w, http.StatusBadRequest, errInvalidRequest, "anthropic-version header is required")
		return
	}

	// D-10: accept-and-ignore anthropic-beta + cache_control fields; log at debug.
	if beta := r.Header.Get("anthropic-beta"); beta != "" {
		a.cfg.Logger.Debug("anthropic: accepted-and-ignored anthropic-beta", "value", beta)
	}

	var wire anthropicMessagesRequest
	if err := decodeJSONBody(w, r, messagesBodyCap, &wire); err != nil {
		if isMaxBytesError(err) {
			writeError(w, http.StatusRequestEntityTooLarge, errRequestTooLarge, "request body too large")
			return
		}
		writeError(w, http.StatusBadRequest, errInvalidRequest, "invalid JSON: "+err.Error())
		return
	}

	// Per Anthropic spec: max_tokens REQUIRED on input; model REQUIRED;
	// messages REQUIRED non-empty.
	if wire.Model == "" {
		writeError(w, http.StatusBadRequest, errInvalidRequest, "`model` is required")
		return
	}
	if wire.MaxTokens <= 0 {
		writeError(w, http.StatusBadRequest, errInvalidRequest, "`max_tokens` is required and must be > 0")
		return
	}
	if len(wire.Messages) == 0 {
		writeError(w, http.StatusBadRequest, errInvalidRequest, "`messages` is required and must be a non-empty array")
		return
	}

	req := wireToChatRequest(&wire, r)
	stream := wire.Stream

	if !stream {
		resp, err := a.cfg.Engine.Collect(r.Context(), req)
		if err != nil {
			writeError(w, http.StatusInternalServerError, errAPI, err.Error())
			return
		}
		writeJSON(w, chatResponseToMessage(resp, wire.Model))
		return
	}

	// Streaming path ŌĆö engine.Run + range Chunks (D-01).
	run, err := a.cfg.Engine.Run(r.Context(), req)
	if err != nil {
		writeError(w, http.StatusInternalServerError, errAPI, err.Error())
		return
	}
	if err := runSSEEmitter(r.Context(), w, run, wire.Model, a.cfg.Logger); err != nil {
		// At this point the response is committed; can't write a JSON error.
		// Emit mid-stream error event if possible, then return.
		a.cfg.Logger.Debug("anthropic: sse handler exit", "err", err)
	}
}
```
