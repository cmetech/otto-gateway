# Phase 03.1 Plan 06: Case-Insensitive Bearer Scheme Match (CR-02 Fix) Summary

Closed VERIFICATION.md Gap 2 / REVIEW CR-02 by replacing the exact-case `strings.HasPrefix(authHeader, "Bearer ")` check in `internal/auth/bearer.go::extractToken` with a length-guarded `strings.EqualFold` match, restoring RFC 7235 §2.1 / RFC 6750 case-insensitive scheme semantics and preserving the D-15 Authorization-wins downgrade-attack guard for ALL RFC-valid spellings of the Bearer scheme.

## What changed

### `internal/auth/bearer.go::extractToken`

**Before** (lines 77-83, exact-case match — defective):

```go
func extractToken(r *http.Request) string {
    const bearerPrefix = "Bearer "
    if authHeader := r.Header.Get("Authorization"); strings.HasPrefix(authHeader, bearerPrefix) {
        return authHeader[len(bearerPrefix):]
    }
    return r.Header.Get("x-api-key")
}
```

**After** (case-insensitive match with length guard — fixed):

```go
func extractToken(r *http.Request) string {
    const bearerPrefix = "Bearer "
    authHeader := r.Header.Get("Authorization")
    // RFC 7235 §2.1 + RFC 6750: the auth-scheme token is case-insensitive.
    // The len(authHeader) >= len(bearerPrefix) guard prevents an
    // out-of-bounds slice when the header is shorter than the prefix
    // (including the empty / absent case — r.Header.Get returns "").
    if len(authHeader) >= len(bearerPrefix) &&
        strings.EqualFold(authHeader[:len(bearerPrefix)], bearerPrefix) {
        return authHeader[len(bearerPrefix):]
    }
    return r.Header.Get("x-api-key")
}
```

Doc-comment on `extractToken` was extended with one sentence noting that the scheme token is matched case-insensitively per RFC 7235 §2.1 (so `Bearer`, `bearer`, `BEARER`, `BeArEr` are all accepted), and the Authorization-wins precedence paragraph was extended with a note that the precedence holds across ALL RFC-valid scheme spellings — closing the downgrade vector described in CR-02.
