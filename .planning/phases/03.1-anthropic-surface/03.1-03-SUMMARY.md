## Decisions Made

1. **Test-injectable inner loop (runSSEEmitterLoop) separate from production entry (runSSEEmitter).** The production path constructs the real `time.NewTicker(PingInterval)` per W7; the inner loop accepts a `<-chan time.Time` so tests can fire pings on demand without waiting 15 real seconds. Single-goroutine invariant (D-05) is preserved in both paths. Inner loop is package-private (lowercase) — the only export is runSSEEmitter.

2. **applyChunk reorder: identify kind support BEFORE state mutation.** Discovered during Task 1 RED phase when the unsupported-kind drop test failed. The naive close-then-open flow emitted content_block_stop + bumped blockIndex on a dropped chunk, forcing the next supported chunk into a new block — but the plan's behavior contract requires "text + drop + text = single text block content 'ok' then 'more'". The fix moves the kind-dispatch switch to step 1 of applyChunk; unsupported kinds short-circuit BEFORE any state change (no close, no bump, no log-as-open). RESEARCH.md §Pattern 1 lines 372-378 default branch confirms this is the intended semantics.

3. **Error-frame is terminal — no message_stop after `event: error`.** When Stream().Result() returns an error mid-stream, finalizeStream emits `event: error` via Plan 02's writeSSEError helper and returns WITHOUT message_delta/message_stop. The @anthropic-ai/sdk MessageStream parser treats `event: error` as the terminal frame; emitting message_stop after it would race with the SDK's error dispatch and produce a confusing user-visible state. Verified by `TestRunSSEEmitterLoop_ResultError` which asserts no message_delta / message_stop follows the error frame.

4. **content_block payload type — distinct textBlockHeader + thinkingBlockHeader structs.** Plan 03 deliberated between (a) one polymorphic blockHeader struct with omitempty Text/Thinking fields and (b) distinct kind-specific types using `any` for `contentBlockStart.ContentBlock`. Chose (b) because Anthropic's spec requires the empty `text` or `thinking` field to be PRESENT on content_block_start (not omitted) — option (a) would have required custom MarshalJSON to control omission per-kind. Distinct types are cleaner and produce the right wire bytes via the default encoder. Same trick for textDelta / thinkingDelta payloads inside contentBlockDelta.

5. **Trailing-newline reconciliation on golden compare.** Discovered when the pre-commit `end-of-file-fixer` hook collapsed the SSE goldens' trailing `\n\n` to a single `\n`. The wire output retains the full `\n\n` per spec. compareGolden trims trailing newlines from both actual and want before byte-equality so the goldens can live cleanly under the pre-commit hook without mutating wire output or disabling the hook. The trim is symmetric so any non-newline drift still fails the test loudly.

6. **`<name>.golden.actual` mismatch artifact + .gitignore exclusion.** On golden mismatch, compareGolden writes the actual normalized output to `testdata/<name>.actual` so a developer can `diff` against the expected. `.gitignore` excludes `**/testdata/*.actual` so the artifact never accidentally lands in a commit (the source `.golden` files ARE tracked). This is the standard golang golden-file workflow.

7. **Comment block in handlers_test.go uses descriptive language, not the deleted test name.** Initial draft of the replacement comment block named TestHandleMessages_StreamingBranchPlaceholder directly. The orchestrator's deletion guard greps the literal name; the comment would have made the guard fail. Rewrote the comment to describe the deleted test functionally without using its literal name.

8. **Field declaration order in payload structs matches the Anthropic spec's wire order.** encoding/json walks struct fields in declaration order, so the byte-exact golden tests double as field-order pins. `anthropicMessage`: `id, type, role, model, content, stop_reason, stop_sequence, usage`. `messageDelta`: `type, delta, usage`. `messageDeltaInner`: `stop_reason, stop_sequence`. `messageDeltaUsage`: `output_tokens`. Any reorder is caught immediately by the golden tests.
