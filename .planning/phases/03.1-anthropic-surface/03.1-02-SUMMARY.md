## Decisions Made

1. **Local-only Engine + RunHandle + Stream interface mirror (TRST-04 preservation).** adapter.go declares all three interfaces locally so the package never imports internal/engine. `*engine.Engine.Run()` returns concrete `*engine.Run`, which has `Stream() engine.Stream` and `SessionID() string` — these structurally satisfy both the local RunHandle and the local Stream. The structural-typing trick is what makes the Phase 1.1 / Phase 2 single-engine architecture extensible to N adapters without ever touching internal/engine's API surface.

2. **Forward-compatible streaming-stub pattern (B4 part 1).** runSSEEmitterStub emits ONLY `event: message_start\ndata: <minimal-payload>\n\n` plus `Content-Type: text/event-stream` + cache headers, then drains the stream and returns. The corresponding test (TestHandleMessages_StreamingBranchPlaceholder) asserts ONLY the Content-Type header and the first SSE line — both properties Plan 03.1-03's real emitter will preserve byte-for-byte. The stub is intentionally inert beyond that (no message_stop, no content_block_* frames) because Plan 03's emitter owns the full sequence and may differ in payload details.

3. **T-02-33 leak guard enforced at every error path.** handleMessages converts every engine error into `writeError(w, 500, errAPI, "internal error")` AFTER logging the raw err via `a.cfg.Logger.Error`. err.Error() NEVER reaches the response. Three tests assert no leak: TestHandleMessages_EngineError_500 (Collect path), TestHandleMessages_StreamingEngineRunError_500 (Run path), and the body-content checks in TestHandleMessages_BadJSON_400 + TestHandleMessages_BodyTooLarge_413.

4. **Auth 401 envelope ships as documented Ollama exception (W6).** auth.Bearer middleware (modified in Plan 03.1-01 for D-15 dual-header reading) emits `{"error":"Invalid or missing API key"}` on 401 — the Ollama shape. The Anthropic adapter does NOT intercept the middleware. TestAuth_401_OnMessages pins this contract with an inline comment naming Phase 8 as the lift site. Acceptable per RESEARCH.md §Pattern 3 option 2: the @anthropic-ai/sdk raises a generic APIError on 401 regardless of envelope shape; the wire-shape mismatch is an annotation issue for SDK-level debugging, not a behavior break.

5. **document and search_result content blocks: accept-and-drop with debug log.** The plan's `<scope_note>` permits this. ANTH-01..07 do not exercise either type and loop24-client does not send them. The wire decoder's default case (unknown block types) also debug-logs and drops — covered by the same code path. If a future loop24-client variant starts sending document blocks, the adapter behavior is debug-log + drop (request continues with the rest of the content blocks intact); the symptom would be "kiro-cli doesn't see the document content", which is the correct cue to revisit.

6. **doc.go deletion.** Plan 03.1-01 documented that doc.go was a forward-declaration placeholder — Plan 03.1-02 is the plan that supersedes it. Deleting cleanly avoids a duplicate package-doc-comment lint warning (gocritic / staticcheck). The package directory continues to satisfy the .go-arch-lint.yml adapter_anthropic component declaration because adapter.go is now in place.
