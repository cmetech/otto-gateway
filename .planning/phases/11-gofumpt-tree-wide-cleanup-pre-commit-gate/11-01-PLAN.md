## D-11-01: govulncheck carve-out for FMT-02

**Decision:** FMT-02's "`make ci` exits 0 end-to-end" requirement is
**partly satisfied** for v1.6 — the brief §3.12 sequence steps
`fmt-check → vet → build → lint → test-race → arch-lint → examples` all
exit 0 on `main` at commit `eaf87f7` and at phase close. The terminal
`govulncheck ./...` step fails on multiple Go stdlib CVEs
(GO-2026-5039, -5037, -4982, -4980, -4971, -4947, -4946, -4870, …).

**Rationale:** These vulnerabilities pre-existed v1.6 (visible on `main`
before Phase 10 started) and are explicitly **routed to v1.7** per the
Phase 10 closure decision recorded in
`.planning/phases/10-golangci-lint-v2-cleanup-re-gate/10-04-SUMMARY.md`
(Wave 4 deviations chain). Addressing them requires a Go toolchain
upgrade, which is out of v1.6's "narrow-scope, debt-reduction" envelope
per `REQUIREMENTS.md` Out of Scope §1.

**Audit trail:** This carve-out is recorded here, in
`11-01-SUMMARY.md` at phase close, and in `REQUIREMENTS.md`'s
Traceability table (FMT-02 row will read `Phase 11 | 11-01 | Complete
(govulncheck routed to v1.7)`).

## D-11-02: pre-commit hook over `make pre-commit` target

**Decision:** Satisfy CI-01 by **adding a gofumpt hook to the existing
`.pre-commit-config.yaml`**, not by introducing a new `make pre-commit`
target.

**Rationale:**
- `.pre-commit-config.yaml` already exists at repo root and is wired
  with the `pre-commit` framework (`golangci-lint`, `shellcheck`,
  `gitleaks`, `end-of-file-fixer`, `go-mod-tidy`, etc.).
- Lowest friction path: one YAML stanza vs. duplicating gate logic in
  Make. The framework handles staged-file targeting natively, which is
  the precise behavior CI-01 requires.
- Contributor portability: `pre-commit` is the de facto standard for
  Python/Go/JS mixed repos; pinning hooks gives reproducible local gates
  across macOS/Linux/Windows.
- Fewest moving parts: no new Makefile target, no duplicate
  `golangci-lint` invocation logic, no divergence risk between Make and
  the framework over time.

`make pre-commit` as a fallback target is **not in scope** for v1.6 —
operators who decline to install the framework can still run `make
fmt-check lint` manually; the framework gives the same coverage with
staged-file scoping that Make cannot replicate without significant
shell plumbing.

</scope_decisions>

      stated explicitly for clarity.
