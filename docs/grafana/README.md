# Grafana dashboards

## OTTO Gateway — Developer Observability

`otto-gateway-dashboard.json` visualizes everything the gateway exposes at
`/metrics` (and that the tray remote-writes to Grafana Cloud): usage attribution,
HTTP traffic & latency, kiro engine usage, pool/session health, and per-gateway +
per-worker resource usage.

### Import

1. Grafana → **Dashboards → New → Import**.
2. Upload `otto-gateway-dashboard.json` (or paste its contents).
3. When prompted for **Data source**, select your Prometheus source
   (e.g. `grafanacloud-<org>-prom`).
4. **Import**.

### Template variables (top pulldowns)

`Data source` · `Gateway` (instance) · `Surface` · `Skill` · `Client` · `Model`
— all multi-select with **All**. Every panel filters off these, so you can slice
by one gateway (≈ one user's machine), one API surface, one invoking skill, etc.

### Sections

| Row | Contents |
|-----|----------|
| Fleet Overview | headline stat tiles (gateways, requests, sessions, error rate, credits/turns, unhealthy count) |
| Usage — Who & How | requests by surface / skill / client / model, requests-by-gateway, top-skills & top-clients tables |
| HTTP Traffic & Latency | rate by route/status, p50/p95/p99, latency heatmap |
| Kiro Engine Usage | credits & turns rate, turn-duration quantiles, context-window usage gauge + trend, MCP init outcomes |
| Pool & Session Health | slots alive/busy, active sessions, session lifecycle, respawns/ping escalations, per-gateway health matrix |
| Resource Usage | gateway process CPU%/RSS, per-worker CPU%/RSS, open FDs |
| Fleet Inventory | per-gateway version / commit / gateway_id table |

### Notes

- Usage panels populate **as traffic flows** — a fresh gateway with no requests
  shows "No data" until it serves something.
- The "Goroutines" panel only fills if `GW_METRICS_SERIES_PREFIXES` includes
  `go_` (the default is `gw_,process_`).
- All series carry `job="otto-gateway"` and `instance=<gateway_id>` external
  labels (added by the tray remote-writer); the dashboard filters on `instance`.

### Regenerating

The JSON is generated by `scripts/gen_grafana_dashboard.py` — edit that and
re-run it rather than hand-editing the JSON, so panel structure stays consistent.
